# インストールパッケージのアップロード

OAT上にOceanBaseプロダクトをインストールしたり、コンポーネントを作成したりするには、各プロダクトのインストールパッケージ（イメージ）をOATサーバーにアップロードするか、リモートでイメージに接続する必要があります。そのため、OATを使用してOCPをデプロイする前に、まずOCPのインストールに必要なパッケージを、OATがインストールされているサーバーにアップロードする必要があります。

本記事では、OCPに必要なインストールパッケージをOATサーバーにアップロードする方法について説明します。

## 操作手順

### ステップ1：OCPのインストールパッケージをOATサーバーにアップロード

OATをインストールしてから、インストールパッケージを `/oat/data_dir/images` ディレクトリにアップロードする必要があります。

OCPのデプロイメントでは、OATをインストールしたサーバーに、OCP dockerイメージとMetaDB dockerイメージの2つのイメージをアップロードする必要があります。

<main id="notice" type='notice'>
<h4>注意</h4>
<p>oat-all-in-oneインストールパッケージを使用してOATをインストールした場合は、OCP dockerイメージのみをアップロードする必要があります。</p>
</main>

例：

```shell
[root@hostname /]# scp ./ocp.tar <oat_server>:/oat/data_dir/images/ocp.tar
[root@hostname /]# scp ./meta_xxx.tar <oat_server>:/oat/data_dir/images/meta_xxx.tar
```

ここで `oat_server` は、OATがデプロイされているサーバーのIPアドレスを表します。

### ステップ2：インストールパッケージをOATサービスに追加

1. OATにログインします。

2. 左側のナビゲーションバーで、**システムとセキュリティ \> パッケージ** をクリックして **パッケージ管理** ページに移動し、右上の **イメージの追加** をクリックして、**ローカルスキャン** を選択します。
