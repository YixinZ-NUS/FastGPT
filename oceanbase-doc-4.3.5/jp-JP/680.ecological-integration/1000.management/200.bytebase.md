# Bytebaseを使用したOceanBaseインスタンスの管理

Bytebaseは、オープンソースのデータベース変更管理およびコラボレーションツールであり、SaaSサービスとオンプレミス版を提供しています。本記事では、OceanBaseデータベースのMySQLモードを例に、Bytebaseを使用してOceanBaseインスタンスを管理する操作を紹介します。

## 前提条件

1. Bytebaseのデプロイを完了します。具体的な操作手順については、[GitHub](https://github.com/bytebase/bytebase)を参照してください。

2. OceanBaseクラスタの接続情報を取得します。これには、インスタンス名、IPアドレス、ポート、接続に使用するユーザー、パスワードが含まれます。新しいユーザーを作成する必要がある場合は、Bytebaseの権限要件と実際のビジネスニーズに基づいてアカウントに権限を付与してください。詳細については、[ユーザーの作成](https://en.oceanbase.com/docs/common-oceanbase-database-10000000001974752)を参照してください。

## 操作手順

1. Bytebaseプラットフォームにログインし、指示に従って新しいワークスペースを作成し、ご自身のワークスペースにアクセスします。
   <!-- ![](https://obbusiness-private.oss-cn-shanghai.aliyuncs.com/doc/img/cloud/420/zh/20240325.png) -->

2. **インスタンスの追加** をクリックし、ポップアップウィンドウで **OceanBase (MySQL)** を選択します。
    <!-- ![](https://obbusiness-private.oss-cn-shanghai.aliyuncs.com/doc/img/cloud/420/zh/2024032501.png) -->

    ここでは、以下の接続文字列を例に、パラメータの説明を行います。

    ```shell
    obclient -h10.10.10.1 -uusername@obtenant#obdemo -P2883 -p****** -c -A oceanbase
    ```

    |パラメータ|説明|
    |-----|-----|
    |インスタンス名前| インスタンス名前をカスタマイズします。例：obdemo。|
    |環境|実際の状況に応じて、Test環境またはPro環境を選択します。例：Test環境。|
    |HostまたはSocket|接続文字列の `-h` の後のアドレス。例：10.10.10.1。|
    |ポート|接続文字列の `-P` の後のポート番号。例：2883。|
    |ユーザー名|接続文字列の `-u` の後のユーザー名。例：username@obtenant。|
    |パスワード|接続文字列の `-p` の後のユーザーパスワード。例：******。|

3. **接続テスト** をクリックして、接続が成功した後に **作成** をクリックします。

4. 作成が完了した後、GitOps統合を実行できます。
   - GitOpsワークフローを有効にする必要がある場合は、BytebaseにGitHub、GitLab、またはBitbucket Cloudなどのコードリポジトリサービスを統合できます。
   - データベース変更スクリプトを保存するためのGitリポジトリを設定し、変更を対応するブランチにコミットします。Bytebaseは自動的に変更をプルし、承認とリリースのプロセスを実行します。

5. データベースの変更を管理します。
   - SQL移行スクリプトを作成し、Bytebaseのインターフェースを通じで変更申請をコミットします。
   - レビュープロセスに参加し、他のチームメンバーと協力してSQL変更をレビューします。
   - 承認された変更を実行すると、Bytebaseは事前に設定された時間ウィンドウ内でターゲットデータベースに自動的に適用します。
