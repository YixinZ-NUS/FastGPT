# SET NAMES

## Purpose

This statement sets the character set and collation for the current session.

## Syntax

```sql
SET NAMES 'charset_name' [COLLATE 'collation_name']
```

## Parameters

|       Parameter       |                Description                |
|----------------|----------------------------------|
| charset_name   | The specified character set.                           |
| collation_name | The specified collation for the character set. If this parameter is not specified, the default collation of the character set is used. |

  <main id="notice" type='explain'>
    <h4>Note</h4>
    <p>You can execute the <code>SHOW CHARSET;</code> statement to query the character sets supported by the system, and execute the <code>SHOW COLLATION;</code> statement to query the collations supported by the system. </p>
  </main>

## Examples

* Set the character set of the current session to `gbk` and the collation to the default collation of the character set.

  ```sql
  obclient> SET NAMES 'gbk';

  obclient> SHOW VARIABLES LIKE 'character_set_c%';

  obclient> SHOW VARIABLES LIKE 'collation_connection';
  ```

* Set the character set of the current session to `utf8mb4` and the collation to `utf8mb4_general_ci`.

  ```sql
  obclient> SET NAMES 'utf8mb4' COLLATE 'utf8mb4_general_ci';

  obclient> SHOW VARIABLES LIKE 'character_set_c%';

  obclient> SHOW VARIABLES LIKE 'collation_connection';
  ```
