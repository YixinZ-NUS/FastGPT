# ADD RESTORE SOURCE

## Purpose

You can use this statement to load the path of the data to be restored.

## Required privileges

To execute this statement, you must have the `ALTER SYSTEM` privilege.

## Syntax

```sql
ALTER SYSTEM ADD RESTORE SOURCE 'restore_path';
```

## Parameters

|              **Parameter**       |                     **Description**                       |
|-----------------------------|----------------------------------------------------|
| restore_path                | The path of the data backup set or archivelog piece to be loaded. You can specify one or more paths. <main id="notice" type='notice'><h4>Notice</h4><p><ul><li>If multiple backup sets or archivelog pieces to load are stored in an object storage service, values of the <code>host</code>, <code>access_id</code>, and <code>access_key</code> parameters in their paths must be the same. </li><li>You can execute this statement multiple consecutive times in the same session. The system incrementally adds the execution results to the restore source list. </li></ul></p></main> |

## Examples

1. Load the backup set to restore from.

    ```shell
    obclient [oceanbase]> ALTER SYSTEM ADD RESTORE SOURCE 'file:///ob_backup/data/backup_set_1_full';
    ```

2. Load the archivelog piece to restore from.

    ```shell
    obclient [oceanbase]> ALTER SYSTEM ADD RESTORE SOURCE 'file:///ob_backup/archive/piece_d1002r1p1,file:///ob_backup/archive/piece_d1002r1p2';
    ```

    ```shell
    obclient [oceanbase]> ALTER SYSTEM ADD RESTORE SOURCE 'oss://test_bucket/archive/piece_1?host=***&access_id=x****&access_key=****';
    ```
