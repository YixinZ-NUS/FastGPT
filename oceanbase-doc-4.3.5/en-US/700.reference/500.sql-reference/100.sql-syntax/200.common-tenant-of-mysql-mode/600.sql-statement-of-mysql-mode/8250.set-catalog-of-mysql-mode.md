# SET CATALOG

## Purpose

You can execute this statement to switch catalogs.

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>For OceanBase Database V4.3.5, the <code>SET CATALOG</code> statement is supported starting from V4.3.5 BP2. </p>
</main>

## Privilege requirements

To execute the <code>SET CATALOG</code> statement, the current user must have the <code>USE CATALOG</code> privilege. For more information about the privileges in OceanBase Database, see [Privilege types in MySQL-compatible mode](../../../../../600.manage/500.security-and-permissions/300.access-control/200.user-and-permission/200.permission-of-mysql-mode/100.permission-classification-of-mysql.md).

## Syntax

```sql
SET CATALOG {INTERNAL | external_catalog_name};
```

## Parameters

|        **Parameter**       | **Description** |
|-----------------------|----------|
| INTERNAL              | Indicates switching to the system's internal catalog. |
| external_catalog_name | The name of the external catalog to switch to. |

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>You can run the <code>use internal.test</code> or <code>use odps_catalog.default</code> statement to switch catalogs and databases at the same time. </p>
</main>

## Examples

* Switch to the external data catalog named `test_odps_catalog`.

    ```sql
    SET CATALOG test_odps_catalog;
    ```

* Switch to the internal catalog.

    ```sql
    SET CATALOG INTERNAL;
    ```
