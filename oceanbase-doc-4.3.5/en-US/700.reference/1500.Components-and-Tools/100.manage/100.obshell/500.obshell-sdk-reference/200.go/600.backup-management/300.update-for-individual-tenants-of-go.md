# Update the backup configuration for a tenant

This topic describes how to update the backup configuration for a tenant by using obshell-sdk-go.

The difference between updating the backup configuration for a tenant and [setting the backup configuration for a tenant](100.backup-configuration-for-individual-tenants-of-go.md) is that you do not need to set the `data_base_uri` and `archive_base_uri` parameters.

<main id="notice" type='notice'>
  <h4>Notice</h4>
  <p>If you want to set the <code>binding</code> and <code>piece_switch_interval</code> parameters, you must set the <code>archive_base_uri</code> parameter, because the three parameters are bound and cannot be set individually. </p>
</main>

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>It is recommended that you first go to <a href='../100.quickstart-of-go.md'>Get started with obshell-sdk-go</a> to learn how to use obshell-sdk-go. </p>
</main>

## Considerations

* Make sure that obshell is running.

* The node corresponding to the client must be in an initialized cluster.

## Sample code

```go
package main

import (
  "github.com/oceanbase/obshell-sdk-go/services"
)

func main() {
  // Create a client instance with the node address set to '10.10.10.1' and the port set to 2886.
  client, err := services.NewClientWithPassword("10.10.10.1", 2886, "****")
  if err != nil {
    panic(err)
  }

  request := client.V1().NewTenantBackupConfigPatchRequest("tenant2")
  request.SetDeletePolicy("", "3d")
  _, err = client.V1().TenantBackupConfigSyncWithRequest(request)
  if err != nil {
    panic(err)
  }
}
```
