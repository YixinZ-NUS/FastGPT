# Modify the backup status of a tenant

This topic describes how to modify the backup status of a tenant by using the API.

## Request parameters

| Parameter | Type | Required | Example | Description |
|------|-------|------|------|------|
| status | string | No | "canceled" | The backup status. Valid values include `canceled`. The default value is `canceled`. |

### Request

PATCH /api/v1/tenant/:name/backup

### Response

| Parameter | Type | Description |
|-----|------|-------|
| successful | bool | Indicates whether the request is successful. |
| timestamp | time.Time | The timestamp when the server received the request. |
| duration | int | The duration of processing the request in milliseconds. |
| status | int | The HTTP status code. |
| traceId | string | The Trace ID of the request. |
| error | ApiError | The error information returned by the server, which includes the following fields:<ul><li><code>code</code>: the error code. </li><li><code>message</code>: the error message. </li><li><code>subErrors</code>: the suberror information. </li></ul> |

## Examples

### Request example

PATCH 10.10.10.1:2886/api/v1/tenant/t1/backup

```json
{
  "status":"canceled",
}
```

### Response example

```json
{
  "successful": true,
  "timestamp": "2024-01-09T17:11:43.385889278+08:00",
  "duration": 1,
  "status": 200,
  "traceId": "877fe32b83634153",
}
```
