# multiblock_read_gap_size

<main id="notice" type='explain'>
<h4>Note</h4>
<p>Due to the storage architecture change of OceanBase Database V4.x, this parameter is no longer applicable and has been deprecated since OceanBase Database V4.0.0. </p>
</main>

## Description

`multiblock_read_gap_size` specifies the size of multiple block caches from which data can be read in one I/O operation.

## Attributes

| **Attribute** | **Description** |
|------------------|------------|
| Type | Capacity |
| Default value | 0K |
| Value range | \[0K, 2M\] |
| Effective upon OBServer node restart | No |
