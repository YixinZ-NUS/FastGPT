# location_fetch_concurrency

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V1.4.</p>
</main>

## Description

`location_fetch_concurrency` specifies the maximum number of concurrent threads for refreshing the `location_cache` of a single node.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Type    | Int      |
| Default value      | 20          |
| Value range    | [1, 1000] |
| Modifiable | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | No. The setting takes effect immediately.  |

## Examples

Set the maximum number of concurrent threads for refreshing the `location_cache` of a single node to 20.

```shell
obclient> ALTER SYSTEM SET location_fetch_concurrency = 20;
```
