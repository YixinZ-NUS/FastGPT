# tcp_keepcnt

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V2.2.77. </p>
</main>

## Description

`tcp_keepcnt` specifies the maximum number of retries before closing an inactive connection.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can query this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

| **Attribute** | **Description** |

## Examples

Set the maximum number of retries before closing an inactive connection to 10.

```shell
obclient> ALTER SYSTEM SET tcp_keepcnt = 10;
```
