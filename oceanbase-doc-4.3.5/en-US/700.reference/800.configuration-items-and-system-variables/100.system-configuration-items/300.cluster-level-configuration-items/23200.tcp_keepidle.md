# tcp_keepidle

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V2.2.77. </p>
</main>

## Description

`tcp_keepidle` specifies the threshold for triggering keepalive probes for TCP connections in an idle state.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

| **Attribute** | **Description** |

## Examples

Set the threshold for triggering keepalive probes for TCP connections in an idle state to 7200s.

```shell
obclient> ALTER SYSTEM SET tcp_keepidle = '7200s';
```
