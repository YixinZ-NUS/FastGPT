# tenant_cpu_variation_per_server

<main id="notice" type='explain'>
<h4>Note</h4>
<p>In OceanBase Database V2.x, the corresponding logic is changed, and the corresponding feature is no longer available. This parameter has been deprecated since OceanBase Database V2.2.77. </p>
</main>

## Description

`tenant_cpu_variation_per_server` specifies the allowed variation in percentage of CPU quota allocated to any tenant on each OBServer node.

## Attributes

| Attribute | Description |
|------------|------------|
| Type | Double |
| Default value | 50 |
| Value range | [0, 100] |
| Effective upon OBServer node restart | No. The setting takes effect immediately.          |

## Examples

Set the allowed variation in percentage of CPU quota allocated to any tenant on each OBServer node to 50%.

```shell
obclient> ALTER SYSTEM SET tenant_cpu_variation_per_server=50;
```
