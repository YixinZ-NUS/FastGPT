# sys_bkgd_migration_change_member_list_timeout

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V2.2.50. </p>
</main>

## Description

`sys_bkgd_migration_change_member_list_timeout` specifies the timeout period for changing the Paxos member list during a replica migration.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Type             | Time         |
| Default value               | 20s <main id="notice" type='explain'><h4>Note</h4><p>For V4.0.0, the default value is changed from 1h to 20s. </p></main>          |
| Value range             | [0s, 24 h] |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | No. The setting takes effect immediately. |

## Examples

Set the timeout period for changing the Paxos member list during a replica migration to 20s.

```shell
obclient> ALTER SYSTEM SET sys_bkgd_migration_change_member_list_timeout = '20s';
```
