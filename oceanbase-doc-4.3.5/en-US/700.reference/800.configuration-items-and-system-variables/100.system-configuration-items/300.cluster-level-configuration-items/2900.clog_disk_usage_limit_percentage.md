# clog_disk_usage_limit_percentage

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>In OceanBase Database V4.x, log disk space is managed at the tenant level. This parameter is no longer applicable and has been deprecated since OceanBase Database V4.0.0. </li>
<li>In OceanBase Database V4.x, you can use the tenant-level parameter <code>log_disk_utilization_limit_threshold</code> to specify the threshold for stopping writes to log disk space. For more information about the <code>log_disk_utilization_limit_threshold</code> parameter, see <a href="../400.tenant-level-configuration-items/4600.log_disk_utilization_limit_threshold.md">log_disk_utilization_limit_threshold</a>. </li></ul>
</main>

`clog_disk_usage_limit_percentage` specifies the maximum percentage of disk space available for transaction logs.

| **Attribute** | **Description** |
|------------------|-------------|
| Type | Integer |
| Default value | 95 |
| Value range | \[80, 100\] |
| Effective upon OBServer node restart | No |

<main id="notice" type='notice'>
  <h4>Notice</h4>
  <p>When the disk space used for storing transaction logs reaches the value of this parameter, disk writes are stopped and no more logs are accepted.  </p>
</main>
