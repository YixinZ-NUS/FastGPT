# bf_cache_miss_count_threshold

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V1.4. </p>
</main>

## Description

`bf_cache_miss_count_threshold` specifies the threshold for triggering BloomFilter cache. A BloomFilter cache is created when the number of macroblock misses reaches this threshold.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can query this parameter by using the `SHOW PARAMETERS` statement.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

|      **Attribute**    |  **Description**  |
|------------------|------------|
| Type             | Int       |
| Default value               | 100      |
| Value range             | [0, +âˆž) <br> If the value is 0, BloomFilter cache creation is disabled. |

| Effective upon OBServer node restart | No. The setting takes effect immediately.   |

## Examples

Create a BloomFilter cache when the number of macroblock misses reaches 100.

```shell
obclient> ALTER SYSTEM SET bf_cache_miss_count_threshold=100;
```
