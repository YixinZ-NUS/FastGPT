# migrate_concurrency

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>OceanBase Database V4.x does not support configuration of thread resources by task type due to limited thread resources. This parameter has been deprecated since OceanBase Database V4.0.0. </li>
<li>In OceanBase Database V4.x, you can use the <code>ha_mid_thread_score</code> parameter to specify the number of worker threads for high-availability medium-priority tasks such as migration tasks. For more information about the tenant-level parameter <code>ha_mid_thread_score</code>, see <a href="../400.tenant-level-configuration-items/3600.ha_mid_thread_score.md">ha_mid_thread_score</a>. </li></ul>
</main>

## Description

`migrate_concurrency` specifies the maximum concurrency for internal data migration.

## Attributes

| **Attribute** | **Description** |
|------------------|-----------|
| Type | Integer |
| Default value | 10 |
| Value range | \[0, 64\] |
| Effective upon OBServer node restart | No |
