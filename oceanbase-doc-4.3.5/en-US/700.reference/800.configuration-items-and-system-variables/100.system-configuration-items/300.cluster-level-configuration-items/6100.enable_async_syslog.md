# enable_async_syslog

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V2.2.50.</p>
</main>

## Description

`enable_async_syslog` specifies whether to enable asynchronous writing of system logs.

## Privilege requirements

* Query the parameter

  `sys` and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

|      **Attribute**    |   **Description**   |
|------------------|--------------|
| Type             | Bool                |
| Default value               | True                |
| Value range             | <li> True: enabled  <li> False: disabled    |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately.    |

## Examples

Enable asynchronous writing of system logs.

```shell
obclient> ALTER SYSTEM SET enable_async_syslog = True;
```
