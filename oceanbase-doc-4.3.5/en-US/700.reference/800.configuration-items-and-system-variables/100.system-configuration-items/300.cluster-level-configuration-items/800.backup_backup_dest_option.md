# backup_backup_dest_option

<main id="notice" type='explain'>
<h4>Note</h4>
<p>Since V4.x no longer supports the backup of backed-up data (secondary backup) feature, this parameter is removed starting from V4.0.0. </p>
</main>

## Description

`backup_backup_dest_option` specifies parameters related to backup backup.

## Attributes

|        Attribute        |  Description   |
|------------------|-------|
| Type             | String  |
| Default value              | Empty string  |
| Effective upon OBServer node restart | No     |

## Considerations

`backup_backup_dest_option` is a subset of the `backup_dest_option` parameter. The `backup_backup_dest_option` parameter does not support the `backup_copies` and `log_archive_piece_switch_interval` parameters.
