# balancer_tolerance_percentage

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V1.4. </p>
</main>

## Description

`balancer_tolerance_percentage` specifies the tolerance of disk imbalance among multiple units of a tenant in the load balancing strategy. If the disk imbalance is within the range of the mean value plus or minus the tolerance, no balancing action is triggered.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement to query this parameter.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

|      **Attribute**    |  **Description** |
|------------------|-----------|
| Type            | Int        |
| Default value              | 10        |
| Value range            | [1, 100) |
| Modifiable          | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.    |
| Effective upon OBServer node restart | No. The setting takes effect immediately.   |

## Examples

Set the tolerance of disk imbalance among multiple units of a tenant to 10.

```shell
obclient> ALTER SYSTEM SET balancer_tolerance_percentage=10;
```
