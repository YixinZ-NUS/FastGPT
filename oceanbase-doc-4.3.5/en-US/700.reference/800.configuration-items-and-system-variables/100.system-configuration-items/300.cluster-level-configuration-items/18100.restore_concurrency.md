# restore_concurrency

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>OceanBase Database V4.x does not support configuration of thread resources by task type due to limited thread resources. This parameter has been deprecated since OceanBase Database V4.0.0. </li>
<li>In OceanBase Database V4.x, you can use the <code>ha_high_thread_score</code> parameter to specify the number of worker threads for high-availability high-priority tasks such as replication, rebuild, and restore tasks. For more information about the tenant-level parameter <code>ha_high_thread_score</code>, see <a href="../400.tenant-level-configuration-items/3400.ha_high_thread_score.md">ha_high_thread_score</a>. </li></ul>
</main>

## Description

`restore_concurrency` specifies the maximum concurrency of tenant data restore from backups.

## Attributes

| **Attribute** | **Description** |
|------------------|------------|
| Type | Integer |
| Default value | 0 |
| Value range | \[0, 512\] |
| Effective upon OBServer node restart | No |
