# enable_syslog_wf

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V1.4. </p>
</main>

## Description

`enable_syslog_wf` specifies whether to print system logs of the WARNING level and above to a separate log file.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

|      **Attribute**    |   **Description**   |
|------------------|--------------|
| Type             | Bool                |
| Default value               | True                |
| Value range             | <ul><li>True: enabled</li> <li>False: disabled</li></ul>     |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately.   |

## Considerations

This feature takes effect only when the value of the [max_syslog_file_count](13300.max_syslog_file_count.md) parameter is a non-zero positive number.

## Examples

Set the system to print system logs of the WARNING level and above to a separate log file.

```shell
obclient> ALTER SYSTEM SET enable_syslog_wf = True;
```
