# migration_disable_time

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V1.4. </p>
</main>

## Description

`migration_disable_time` specifies the cooldown period during which new data migration operations are prohibited after a data migration fails due to a resource issue, such as disk space exhaustion.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can query this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Examples

Set the cooldown period during which new data migration operations are prohibited after a data migration fails due to a resource issue, such as disk space exhaustion, to 3600s.

```shell
obclient> ALTER SYSTEM SET migration_disable_time = 3600s;
```
