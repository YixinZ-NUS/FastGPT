# rpc_timeout

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>For V2.1, this parameter is available starting with V2.1. </p>
</main>

## Description

`rpc_timeout` specifies the timeout period for internal cluster requests.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Examples

Set the timeout period for internal cluster requests to 2s.

```shell
obclient> ALTER SYSTEM SET rpc_timeout = '2s';
```
