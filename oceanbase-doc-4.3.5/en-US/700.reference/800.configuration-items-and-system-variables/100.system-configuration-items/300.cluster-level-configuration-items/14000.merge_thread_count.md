# merge_thread_count

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>Due to limited thread resources, the V4.x version does not support configuring thread resources based on specific task types. This parameter was removed in V4.0.0.</li>
<li>In the V4.x version, the <code>compaction_low_thread_score</code> parameter is used to set the number of worker threads for low-priority compactions. For more information about the tenant-level parameter <code>compaction_low_thread_score</code>, see <a href="../400.tenant-level-configuration-items/1100.compaction_low_thread_score.md">compaction_low_thread_score</a>. </li></ul>
</main>

## Description

The `merge_thread_count` parameter specifies the number of threads for daily major compactions.

## Attributes

|      **Attribute**      |   **Description**   |
|------------------|------------|
| Type             | Int         |
| Default value               | 0          |
| Value range             | [0, 256] |
| Effective upon OBServer node restart | No          |

## Considerations

If the value of this parameter is `0`, the number of major compaction worker threads is calculated as `min{10,cpu_cnt}`, where `cpu_cnt` is the number of system CPUs.
