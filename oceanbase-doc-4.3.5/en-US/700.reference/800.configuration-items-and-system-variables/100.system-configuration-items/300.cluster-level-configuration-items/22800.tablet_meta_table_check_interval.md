# tablet_meta_table_check_interval

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V4.0.0. </p>
</main>

## Description

`tablet_meta_table_check_interval` specifies the interval at which the background inspection thread inspects the `DBA_OB_TABLET_REPLICAS`/`CDB_OB_TABLET_REPLICAS` view.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Type             | Time         |
| Default value               | 30m |
| Value range             | [1m, +âˆž)    |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | No. The setting takes effect immediately. |

## Examples

Set the interval at which the background inspection thread inspects the `DBA_OB_TABLET_REPLICAS`/`CDB_OB_TABLET_REPLICAS` view to 30m.

```shell
obclient> ALTER SYSTEM SET tablet_meta_table_check_interval = '30m';
```
