# trx_2pc_retry_interval

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V1.4. </p>
</main>

## Description

`trx_2pc_retry_interval` specifies the interval at which two-phase commit (2PC) retries are performed when a 2PC operation fails.

## Privilege requirements

* Query the parameter

  You can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter in the `sys` tenant and all user tenants.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Type             | Time            |
| Default value               | 100ms           |
| Value range             | [1ms, 5000ms] |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | No. The setting takes effect immediately.  |

## Examples

Set the interval at which 2PC retries are performed when a 2PC operation fails to 100ms.

```shell
obclient> ALTER SYSTEM SET trx_2pc_retry_interval = '100ms';
```
