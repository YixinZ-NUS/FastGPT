# weak_read_version_refresh_interval

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V1.4.</p>
</main>

## Description

`weak_read_version_refresh_interval` specifies the refresh cycle of the weak consistency read version number, which affects the latency of weak consistency read data.

## Privilege requirements

* Query the parameter

  `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Type             | Time      |
| Default value               | 100ms <main id="notice" type='explain'><h4>Note</h4><p>For V4.0.0, the default value is changed from 50ms to 100ms.</p></main>         |
| Value range             | [50ms, +∞)<main id="notice" type='explain'><h4>Note</h4><p>For V4.0.0, the value range is changed from [0ms, +∞) to [50ms, +∞).</p></main> |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately. |

## Examples

Set the refresh cycle of the weak consistency read version number to 100ms.

```shell
obclient> ALTER SYSTEM SET weak_read_version_refresh_interval ='100ms';
```
