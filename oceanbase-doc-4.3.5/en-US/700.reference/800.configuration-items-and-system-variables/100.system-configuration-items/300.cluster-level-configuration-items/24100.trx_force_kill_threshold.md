# trx_force_kill_threshold

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>In OceanBase Database V4.x, the corresponding logic is changed, and the corresponding feature is no longer available. This parameter has been deprecated since OceanBase Database V4.0.0. </p>
</main>

## Description

`trx_force_kill_threshold` specifies the maximum amount of time that the system waits before killing transactions for a freeze or leader switchover.

## Attributes

| **Attribute** | **Description** |
|------------------|--------------|
| Type | Time |
| Default value | 100ms |
| Value range | \[1ms, 10s\] |
| Effective upon OBServer node restart | No |
