# sys_cpu_limit_trigger

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>Due to the storage architecture change of OceanBase Database V4.x, this parameter is no longer applicable and has been deprecated since OceanBase Database V4.0.0. </li>
<li>OceanBase Database V4.x does not support resource control by parameter. You can call the <code>DBMS_RESOURCE_MANAGER</code> system package to isolate and control resources. For more information about the <code>DBMS_RESOURCE_MANAGER</code> system package, see <a href="../../../../600.manage/200.tenant-management/600.common-tenant-operations/300.resource-isolation/100.resource-isolation-overview.md">Overview</a>. </li></ul>
</main>

## Description

`sys_cpu_limit_trigger` specifies the CPU utilization threshold. When CPU utilization reaches the threshold, background tasks in the system will be suspended.
