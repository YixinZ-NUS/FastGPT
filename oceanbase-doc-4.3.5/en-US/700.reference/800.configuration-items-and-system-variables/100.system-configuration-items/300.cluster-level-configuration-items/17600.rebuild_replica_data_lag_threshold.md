# rebuild_replica_data_lag_threshold

<main id="notice" type='explain'>
<h4>Note</h4>
<p>Due to the architecture upgrade of OceanBase Database V4.x, this parameter is no longer applicable and has been deprecated since OceanBase Database V4.0.0. </p>
</main>

## Description

`rebuild_replica_data_lag_threshold` specifies the threshold of the size difference of transaction logs between the leader and a follower. When the difference reaches the threshold, replica rebuilding is triggered.

| **Attribute** | **Description** |

| Type | Capacity |

| Effective upon OBServer node restart | No |

## Considerations

If this parameter is set to `0`, replica rebuilding will not be triggered.
