# resource_soft_limit

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>Due to the storage architecture upgrade of OceanBase Database V4.x, the meaning of this parameter is no longer applicable, and it has been deprecated since OceanBase Database V4.0.0. </li>
<li>In OceanBase Database V4.x, you can only use the tenant-level parameter <code>enable_rebalance</code> to specify whether to enable unit balancing. For more information about the parameter, see <a href="../400.tenant-level-configuration-items/2800.enable_rebalance.md">enable_rebalance</a>. </li></ul>
</main>

## Description

`resource_soft_limit` specifies whether to enable unit balancing.

## Attributes

| **Attribute** | **Description** |
|------------------|--------------|
| Type | Integer |
| Default value | 50 |
| Value range | \[1, 10000\] |
| Effective upon OBServer node restart | No |

## Considerations

If this parameter is set to a value less than `100`, unit balancing is enabled. If this parameter is set to a value greater than or equal to `100`, unit balancing is disabled.
