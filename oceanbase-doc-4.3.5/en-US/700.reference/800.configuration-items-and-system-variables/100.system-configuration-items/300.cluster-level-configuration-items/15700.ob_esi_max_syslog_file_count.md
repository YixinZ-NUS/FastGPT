# ob_esi_max_syslog_file_count

<main id="notice" type='explain'>
<h4>Note</h4>
<p>Due to the architecture optimization of the physical backup and restore feature in OceanBase Database V4.x, the feature corresponding to this parameter is no longer supported and this parameter has been deprecated since OceanBase Database V4.0.0. </p>
</main>

## Description

`ob_esi_max_syslog_file_count` specifies the maximum number of log files that can be retained for the obesi process.

## Attributes

| **Attribute** | **Description** |
| --- | --- |
| Type | Integer |
| Default value | 0 |
| Value range | [0, +âˆž) |
| Effective upon OBServer node restart | No |

## Considerations

Each log file can occupy at most 256 MB of disk space. If this parameter is set to `0`, no log files are deleted.
