# log_storage_warning_tolerance_time

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>For V4.1.0, this parameter is available starting with V4.1.0.</p>
</main>

## Description

`log_storage_warning_tolerance_time` specifies the time from when I/O fails on a log disk to when the disk is marked as a bad disk.

## Privilege requirements

* Query the parameter

  The sys tenant and all user tenants can query this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view.

* Modify the parameter

  Only the sys tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Type     | Time |
| Default value       | 5s |
| Value range     | [1s,300s] |
| Modifiable | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | No. The setting takes effect immediately. <main id="notice" type='explain'><h4>Note</h4><p>Starting from V4.2.5 BP2, this parameter takes effect dynamically in V4.2.5. </p></main> |

## Examples

Set the time from when I/O fails on a log disk to when the disk is marked as a bad disk to 5s.

```shell
obclient> ALTER SYSTEM SET log_storage_warning_tolerance_time = '5s';
```
