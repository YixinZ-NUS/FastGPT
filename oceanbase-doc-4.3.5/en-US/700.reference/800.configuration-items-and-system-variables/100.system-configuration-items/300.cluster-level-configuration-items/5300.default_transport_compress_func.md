# default_transport_compress_func

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>Due to the architecture optimization of OceanBase Database V4.x, each module selects its own compression algorithm instead of using the unified default compression algorithm, and this parameter has been deprecated since OceanBase Database V4.0.0. </li>
<li>For OceanBase Database V4.x: </li>
  <ul><li>In OceanBase Database V4.1.0 and later, you can use the tenant-level parameter <code>log_transport_compress_func</code> to specify the algorithm for compressing logs for transmission. For more information about the <code>log_transport_compress_func</code> parameter, see <a href="../400.tenant-level-configuration-items/5000.log_transport_compress_func.md">log_transport_compress_func</a>. </li>
  <li>In OceanBase Database V4.2.1 and later versions of V4.2.x, you can use the tenant-level parameter <code>kv_transport_compress_func</code> to specify the algorithm for compressing OBKV-Table query results for transmission. In OceanBase Database V4.3.x, you can use the cluster-level parameter <code>tableapi_transport_compress_func</code> to specify the algorithm for compressing OBKV-Table query results for transmission. For more information about the <code>tableapi_transport_compress_func</code> parameter, see <a href="22600.tableapi_transport_compress_func.md">tableapi_transport_compress_func</a>. </li></ul>
</main>

## Description

`default_transport_compress_func` specifies the remote procedure call (RPC) compression algorithm for the entire cluster.

## Attributes

| **Attribute** | **Description** |
|------------------------------|----------------------------------------|
| Type | String |
| Default value | none |
| Value range | </li><li> none   </li><li> lz4_1.0   </li><li> snappy_1.0   </li><li> zlib_1.0   </li><li> zstd_1.0   </li><li> zstd_1.3.8 |
| Modifiable | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | No. The setting takes effect immediately.                       |

## Examples

Set the RPC compression algorithm for the entire cluster to lz4_1.0.

```shell
obclient> ALTER SYSTEM SET default_transport_compress_func = 'lz4_1.0';
```
