# location_cache_refresh_sql_timeout

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V2.2.77. </p>
</main>

## Description

`location_cache_refresh_sql_timeout` specifies the timeout period for refreshing the `location_cache` by using an SQL statement.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Type    | Time      |
| Default value      | 1s        |
| Value range    | [1ms,+âˆž) |
| Modifiable | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | No. The setting takes effect immediately.  |

## Examples

Set the timeout period for refreshing the `location_cache` by using an SQL statement to 1s.

```shell
obclient> ALTER SYSTEM SET location_cache_refresh_sql_timeout = '1s';
```
