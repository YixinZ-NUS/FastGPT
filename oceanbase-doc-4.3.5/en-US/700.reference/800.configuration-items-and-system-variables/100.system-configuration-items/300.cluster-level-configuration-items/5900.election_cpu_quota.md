# election_cpu_quota

<main id="notice" type='explain'>
<h4>Note</h4>
<p>Due to the system architecture optimization of OceanBase Database V4.x, the number of threads for scheduled election tasks varies with the number of log streams. Only one thread is required, and the CPU quota no longer needs to be specified. This parameter has been deprecated since OceanBase Database V4.0.0. </p>
</main>

`election_cpu_quota` specifies the CPU quota allocated to background tasks related to replica election.

| **Attribute** | **Description** |
|------------------|-----------|
| Type | Double |
| Default value | 3 |
| Value range | \[0, 10\] |
| Effective upon OBServer node restart | No |
