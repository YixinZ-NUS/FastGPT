# enable_cgroup

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This parameter was introduced in V3.2.4. </p>
</main>

## Description

The `enable_cgroup` parameter specifies whether to enable the cgroup feature for an OBServer node.

## Privilege requirements

* Query the parameter

  Both the sys tenant and user tenants can execute the `SHOW PARAMETERS` statement or query the `GV$OB_PARAMETERS` view to obtain the value of this parameter.

* Modify the parameter

  Only the sys tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

| **Attribute** | **Description** |
| ------ | ------ |
| Type | Bool |
| Default value | True |
| Value range |<ul><li>True: If the system has mounted the cgroup directory, the cgroup feature is enabled. Otherwise, a warning is returned. </li><li>False: The cgroup feature is disabled. </li></ul>|
| Modifiable | Yes. You can execute the `ALTER SYSTEM SET` statement to modify this parameter. |
| Effective upon OBServer node restart| No. The setting takes effect immediately. <main id="notice" type='explain'><h4>Note</h4><p>In V4.3.5 and later, this parameter takes effect dynamically. </p></main> |

## Example

The following example shows how to enable the cgroup feature.

```shell
obclient> ALTER SYSTEM SET enable_cgroup = true;
```
