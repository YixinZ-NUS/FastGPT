# minor_freeze_times

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>Due to the storage architecture upgrade of OceanBase Database V4.x, and because this parameter serves the same purpose as the <code>major_compact_trigger</code> parameter, this parameter has been deprecated since OceanBase Database V4.0.0. </li>
<li>In OceanBase Database V4.x, you can only use the tenant-level parameter <code>major_compact_trigger</code> to specify the number of minor compactions that triggers a major compaction. For more information about the <code>major_compact_trigger</code> parameter, see <a href="../400.tenant-level-configuration-items/5200.major_compact_trigger.md">major_compact_trigger</a>. </li></ul>
</main>

## Description

`minor_freeze_times` specifies the number of freezes that triggers a major compaction.

## Attributes

| **Attribute** | **Description** |
|------------------|--------------|
| Type | Integer |
| Default value | 100 |
| Value range | \[0, 65535\] |
| Effective upon OBServer node restart | No |

<main id="notice" type='explain'>
  <h4>Note</h4>
  <ul>
  <li>If this parameter is set to <code>0</code>, a major compaction is not triggered regardless of the number of freezes. </li>
  <li>This parameter is equivalent to the <code>major_compact_trigger</code> parameter. </li>
  </ul>
</main>
