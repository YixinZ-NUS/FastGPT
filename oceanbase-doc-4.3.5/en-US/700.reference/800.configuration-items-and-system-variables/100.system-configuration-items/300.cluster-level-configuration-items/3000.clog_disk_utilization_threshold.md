# clog_disk_utilization_threshold

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>In OceanBase Database V4.x, log disk space is managed at the tenant level. This parameter is no longer applicable and has been deprecated since OceanBase Database V4.0.0. </li>
<li>In OceanBase Database V4.x, you can use the tenant-level parameter <code>log_disk_utilization_threshold</code> to specify the threshold for the reuse of log disk space. For more information about the <code>log_disk_utilization_threshold</code> parameter, see <a href="../400.tenant-level-configuration-items/4700.log_disk_utilization_threshold.md">log_disk_utilization_threshold</a>. </li></ul>
</main>

`clog_disk_utilization_threshold` specifies the threshold for the reuse of clog or ilog disk space.

| Attribute | Description |
|------------------|------------|
| Type | Integer |
| Default value | 80 |
| Value range | \[10, 100) |
| Effective upon OBServer node restart | No |

**Examples**

```sql
obclient> ALTER SYSTEM SET clog_disk_utilization_threshold=80;
```
