# token_reserved_percentage

<main id="notice" type='explain'>
<h4>Note</h4>
<p>In OceanBase Database V4.x, the logic of CPU scheduling among tenants is changed, and the corresponding feature is no longer available. This parameter has been deprecated since OceanBase Database V4.2.0. </p>
</main>

## Description

`token_reserved_percentage` specifies the percentage of idle tokens reserved for tenants during scheduling of CPU resources for the tenants.

## Attributes

| **Attribute** | **Description** |
|------------------|------------|
| Type | Double |
| Default value | 30 |
| Value range | [0, 100] |
| Modifiable | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | No. The setting takes effect immediately.          |

## Examples

Set the percentage of idle tokens reserved for tenants to 30%.

```shell
obclient> ALTER SYSTEM SET token_reserved_percentage=30;
```
