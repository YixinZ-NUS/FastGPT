# enable_rich_error_msg

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V1.4. </p>
</main>

## Description

`enable_rich_error_msg` specifies whether to add the server address, time, and trace ID to the client message for debugging.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can query this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

|      **Attribute**    |   **Description**   |
|------------------|--------------|
| Type             | Bool                |
| Default value               | False                |
| Value range             | <ul><li>True: The `ip:port` of the OBServer node, error time, and log Trace ID are added to the error message. </li> <li>False: The text error message remains unchanged. </li></ul>     |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | No. The setting takes effect immediately.    |

## Examples

Do not add the server address, time, and trace ID to the client message.

```shell
obclient> ALTER SYSTEM SET enable_rich_error_msg = False;
```
