# zone

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V2.2.50. </p>
</main>

## Description

`zone` specifies the name of the zone where the node resides.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can query this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view.

* Modify the parameter

  This parameter is not usually configured. Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

|      **Attribute**      |    **Description**    |
|------------------|--------------|
| Type             | String  |
| Default value               | "" indicates an empty string.     |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately.            |

## Examples

Set the name of the zone where the node resides to `zone1`.

```shell
obclient> ALTER SYSTEM SET `zone` = 'zone1';
```
