# ob_java_connector_path

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>For V4.3.5 BP1, this parameter is available starting with V4.3.5 BP1. </p>
</main>

## Description

`ob_java_connector_path` specifies the path of the executable dependency jar package that can be loaded by the JVM of an OBServer node.

## Privilege requirements

* Query the parameter

    The `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

    Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

|Attribute|Description|
|---|---|
|Parameter type|String|
|Default value|NULL|
|Modifiable|Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
|Effective upon OBServer node restart|No. The setting takes effect immediately. |

## Examples

Set the path of the executable dependency jar package that can be loaded by the JVM of an OBServer node to `/home/admin/jni_packages/`.

```shell
ALTER SYSTEM SET ob_java_connector_path="/home/admin/jni_packages/";
```
