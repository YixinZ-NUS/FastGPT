# server_cpu_quota_min

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V2.2.50. </p>
</main>

## Description

`server_cpu_quota_min` specifies the minimum CPU quota that can be used by the system. The system automatically reserves this quota.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can query this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

|      **Attribute**      |  **Description**   |
|------------------|-----------|
| Type             | Double    |
| Default value               | 0, indicating adaptive adjustment<main id="notice" type='explain'><h4>Note</h4><ul><li>For V4.0.0, the default value is changed from 2.5 to 1.  </li><li>For V4.2.1, the default value is changed from 1 to 0. </li></ul></main>  |
| Value range             | [0, 16]<main id="notice" type='explain'><h4>Note</h4><ul><li>For V4.0.0, the value range is changed from [0, 16] to [1, 16].</li><li>For V4.2.1, the value range is changed from [1, 16] to [0, 16]. </li></ul></main>       |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately.<main id="notice" type='explain'><h4>Note</h4><p>For V4.1.0, the setting takes effect immediately instead of after the OBServer node restarts. </p></main>             |

## Considerations

If the value is 0, the system adaptively adjusts the minimum CPU quota based on the value of the `cpu_count` parameter:

1. If `cpu_count` is less than 8, the minimum CPU quota is adaptively adjusted to 1.
2. If 8 ≤ `cpu_count` < 16, the minimum CPU quota is adaptively adjusted to 2.
3. If 16 ≤ `cpu_count` < 32, the minimum CPU quota is adaptively adjusted to 3.
4. If `cpu_count` is 32 or more, the minimum CPU quota is adaptively adjusted to 4.

## Examples

Adaptively adjust the maximum CPU quota.

```shell
obclient> ALTER SYSTEM SET server_cpu_quota_max=0;
```
