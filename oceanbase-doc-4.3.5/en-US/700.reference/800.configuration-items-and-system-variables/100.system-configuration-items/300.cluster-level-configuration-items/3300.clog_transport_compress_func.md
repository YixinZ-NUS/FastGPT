# clog_transport_compress_func

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>This parameter has been deprecated since OceanBase Database V4.0.0 because this version does not support compression of transaction logs. </li>
<li>OceanBase Database V4.1.0 and later support compression of transaction logs. In OceanBase Database V4.1.0 and later, you can use the tenant-level parameter <code>log_transport_compress_func</code> to specify the algorithm for compressing transaction logs for internal transmission. For more information about the <code>log_transport_compress_func</code> parameter, see <a href="../400.tenant-level-configuration-items/5000.log_transport_compress_func.md">log_transport_compress_func</a>. </li></ul>
</main>

## Description

`clog_transport_compress_func` specifies the algorithm for compressing transaction logs for internal transmission.

## Attributes

| **Attribute** | **Description** |
|------------------|------------------------------------------------------------------------------|
| Type | String |
| Default value | lz4_1.0 |
| Value range | </li><li> none   </li><li> lz4_1.0   </li><li> snappy_1.0   </li><li> zlib_1.0   </li><li> zstd_1.0   </li><li> zstd_1.3.8 |
| Effective upon OBServer node restart | No |
