# use_large_pages

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V2.2.50. </p>
</main>

## Description

`use_large_pages` specifies whether to use huge pages for the database.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can query the value of this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Type             | String      |
| Default value       | false       |
| Value range             | <ul><li>false: Disable huge pages. </li> <li>true: Allow the mixed use of huge pages and regular pages. </li> <li>only: Force the system to use only huge pages. </li> </ul>    |
| Modifiable | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | Yes. The setting takes effect only after the node is restarted. |

## Examples

Disable huge pages for the database.

```shell
obclient> ALTER SYSTEM SET use_large_pages = 'false';
```
