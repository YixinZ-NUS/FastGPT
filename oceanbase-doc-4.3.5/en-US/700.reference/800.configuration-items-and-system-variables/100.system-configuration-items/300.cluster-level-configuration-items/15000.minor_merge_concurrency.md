# minor_merge_concurrency

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>Due to limited thread resources, the V4.x version no longer supports configuring thread resources by specifying a specific task type. This parameter was removed in V4.0.0. </li>
<li>In the V4.x version, you can set the number of worker threads for major compaction at the tenant level by specifying the <code>compaction_mid_thread_score</code> parameter. For more information about the <code>compaction_mid_thread_score</code> parameter, see <a href="../400.tenant-level-configuration-items/1200.compaction_mid_thread_score.md">compaction_mid_thread_score</a>. </li></ul>
</main>

## Description

The `minor_merge_concurrency` parameter specifies the number of concurrent threads for minor compaction.

## Attributes

|      **Attribute**      |  **Description**   |
|------------------|-----------|
| Type             | Int        |
| Default value              | 0, indicating that the number of concurrent threads is min[10, cpu_cnt]        |
| Valid values             | [0, 64] |
| Effective upon OBServer node restart | No         |
