# plan_cache_evict_interval

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V1.4.</p>
</main>

## Description

`plan_cache_evict_interval` specifies the eviction interval of the execution plan cache.

## Privilege requirements

* Query the parameter

  You can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter in the `sys` tenant and all user tenants.

* Modify the parameter

  Only the `sys` tenant can modify this parameter. User tenants cannot modify this parameter.

## Attributes

|      **Attribute**      |    **Description**    |
|------------------|--------------|
| Type             | Time      |
| Default value               | 5s <main id="notice" type='explain'><h4>Note</h4><p>For V4.2.0, the default value of this parameter is changed from 1s to 5s. </p></main>      |
| Value range             | [0s, +âˆž) |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately.   |

## Examples

Set the eviction interval of the execution plan cache to 5s.

```shell
obclient> ALTER SYSTEM SET plan_cache_evict_interval ='5s';
```
