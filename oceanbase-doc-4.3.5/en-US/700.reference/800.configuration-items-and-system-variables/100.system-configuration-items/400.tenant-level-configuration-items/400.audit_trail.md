# audit_trail

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V2.2.50. </p>
</main>

## Description

`audit_trail` specifies whether to enable database auditing.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  The `sys` tenant and user tenants can modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Parameter type           | String     |
| Default value             | NONE        |
| Value range           | <ul><li>NONE: disables auditing.</li><li>OS: writes audit records to local files.</li><li>DB: writes audit records to internal tables.</li><li>DB,EXTENDED: writes audit records to internal tables and records include the executed SQL statements.</li><li>DB_EXTENDED: writes audit records to internal tables and records include the executed SQL statements.</li></ul>  |
| Modifiable         | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | No. The setting takes effect immediately.   |

## Examples

Disable database auditing.

```shell
obclient> ALTER SYSTEM SET audit_trail ='NONE';
```
