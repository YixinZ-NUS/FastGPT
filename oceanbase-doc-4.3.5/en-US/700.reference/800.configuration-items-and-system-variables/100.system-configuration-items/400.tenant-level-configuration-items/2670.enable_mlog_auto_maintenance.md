# enable_mlog_auto_maintenance

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>For OceanBase Database V4.3.5, this parameter is available starting with V4.3.5 BP4.</p>
</main>

## Description

`enable_mlog_auto_maintenance` specifies whether to enable the automatic management feature of materialized view logs (mlogs).

## Privilege requirements

* Query the parameter

  You can execute the `SHOW PARAMETERS` statement or query the `GV$OB_PARAMETERS` view to query this parameter in the `sys` tenant and all user tenants.

* Modify the parameter

  You can modify this parameter in the `sys` tenant and all user tenants.

## Attributes

| **Attribute** | **Description** |
|----------|----------|
| Parameter type  | Boolean |
| Default value    | True <main id="notice" type='notice'><h4>Notice</h4><p>In OceanBase Database V4.3.5 BP4, the default value of this parameter is <code>True</code> for new tenants and <code>False</code> for tenants upgraded from earlier versions of OceanBase Database. </p></main>|
| Value range  | <ul><li>True: indicates that the automatic management feature of mlogs is enabled. </li><li>False: indicates that the automatic management feature of mlogs is disabled. </li></ul> |
| Modifiable | Yes. You can execute the `ALTER SYSTEM SET` statement to modify this parameter. |
| Effective only after OBServer restart | No. The setting takes effect immediately. |

## Examples

* Disable the automatic management feature of mlogs.

    ```shell
    obclient> ALTER SYSTEM SET enable_mlog_auto_maintenance = False;
    ```

* Enable the automatic management feature of mlogs.

    ```shell
    obclient> ALTER SYSTEM SET enable_mlog_auto_maintenance = True;
    ```
