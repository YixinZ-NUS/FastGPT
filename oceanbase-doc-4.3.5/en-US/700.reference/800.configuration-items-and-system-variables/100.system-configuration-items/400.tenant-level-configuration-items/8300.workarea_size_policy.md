# workarea_size_policy

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V2.2.50. </p>
</main>

## Description

`workarea_size_policy` specifies the strategy for manually or automatically adjusting the size of the SQL workarea.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  The `sys` tenant and user tenants can modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Parameter type             | String                 |
| Default value               | AUTO                               |
| Value range             | <ul><li>MANUAL: specifies manual adjustment. </li><li>AUTO: specifies automatic adjustment. </li></ul>   |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately. |

## Examples

Set the strategy for manually or automatically adjusting the size of the SQL workarea to AUTO.

```shell
obclient> ALTER SYSTEM SET workarea_size_policy= 'AUTO';
```
