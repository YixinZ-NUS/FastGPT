# audit_log_compression

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p><ul><li>For V4.3.x, this parameter is available starting with V4.3.3. </li> <li>For V4.2.x, this parameter is available starting with V4.2.4. </li></ul></p>
</main>

## Description

`audit_log_compression` specifies the compression algorithm used for audit logs.

<main id="notice" >
  <h4>Applicability</h4>
  <p>This topic applies only to OceanBase Database Enterprise Edition. OceanBase Database Community Edition does not support audit features. </p>
</main>

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  The `sys` tenant and user tenants can use the `ALTER SYSTEM SET` statement to modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Type   | String |
| Default value     | NONE, which indicates that compression is not enabled. |
| Value range   | <ul><li>NONE</li> <li>ZSTD</li></ul>|
| Modifiable | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | No. The setting takes effect immediately.   |

## Considerations

This parameter specifies whether to compress the archived files when the file size exceeds the value specified by the `audit_log_rotate_on_size` parameter.

## Examples

Set the compression algorithm for audit logs to ZSTD.

```shell
obclient> ALTER SYSTEM SET audit_log_compression='ZSTD';
```
