# clog_persistence_compress_func

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>Due to the storage architecture change of OceanBase Database V4.x, OceanBase Database V4.0.0 does not support log compression for storage. This parameter has been deprecated since OceanBase Database V4.0.0. </li>
<li>OceanBase Database V4.2.1 and later support log compression for storage to adapt to business needs. In OceanBase Database V4.2.1 and later except V4.3.0, you can use the tenant-level parameter <code>log_storage_compress_func</code> to specify the algorithm for compressing logs for storage. </li></ul><p></p>
</main>

## Description

`clog_persistence_compress_func` specifies the algorithm for compressing transaction logs for storage.

## Attributes

| **Attribute** | **Description** |
|------------------|--------------------------------------------------------------------------|
| Type | String |
| Default value | lz4_1.0 |
| Value range | </li><li> none   </li><li> lz4_1.0   </li><li> zstd_1.0   </li><li> zstd_1.3.8    |
| Effective upon OBServer node restart | No |
