# tenant_sql_net_thread_count

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>For V4.1.0, this parameter is available starting with V4.1.0. </p>
</main>

## Description

`tenant_sql_net_thread_count` specifies the number of I/O threads for MySQL tenants, that is, the number of `sql_nio_server` threads.

## Privilege requirements

* Query the parameter

  `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  `sys` tenant and user tenants can modify this parameter.

## Attributes

|        Attribute      |  Description  |
|------------------|-------|
| Type            | Int |
| Default value              | 0, which indicates the value of `unit_min_cpu`. |
| Value range             | [0, 64] |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately.  |

## Examples

Set the number of I/O threads for MySQL tenants to 0.

```shell
obclient> ALTER SYSTEM SET tenant_sql_net_thread_count= 0;
```
