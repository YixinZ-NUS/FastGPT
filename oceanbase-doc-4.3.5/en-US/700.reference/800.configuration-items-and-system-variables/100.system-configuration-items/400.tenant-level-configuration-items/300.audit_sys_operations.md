# audit_sys_operations

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V2.2.50. </p>
</main>

## Description

`audit_sys_operations` specifies whether to track the operations of system users.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can query this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view.

* Modify the parameter

  The `sys` tenant and user tenants can modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Parameter type           | Bool                |
| Default value             | False               |
| Value range           | <ul><li>False</li><li>True</li></ul>   |
| Modifiable         | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately.  |

## Examples

Set the `audit_sys_operations` parameter to `False` to stop tracking the operations of system users.

```shell
obclient> ALTER SYSTEM SET audit_sys_operations =False;
```
