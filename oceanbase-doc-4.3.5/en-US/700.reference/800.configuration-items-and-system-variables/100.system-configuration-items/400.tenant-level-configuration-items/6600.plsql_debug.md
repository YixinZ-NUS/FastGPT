# plsql_debug

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V2.2.77. </p>
</main>

## Description

`plsql_debug` specifies whether to compile the PL/SQL code into a debug version.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can query the value of this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view.

* Modify the parameter

  The `sys` tenant and user tenants can modify this parameter.

## Attributes

|        Attribute      |  Description  |
|------------------|-------|
| Type             | Bool                    |
| Default value               | False                   |
| Value range             | <ul><li>True: Yes. </li><li>False: No. </li></ul>   |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately.   |

## Examples

Set the parameter to disable the debug version.

```shell
obclient> ALTER SYSTEM SET plsql_debug=false;
```
