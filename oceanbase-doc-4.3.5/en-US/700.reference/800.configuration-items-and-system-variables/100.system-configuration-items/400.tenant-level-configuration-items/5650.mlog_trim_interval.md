# mlog_trim_interval

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>For OceanBase Database V4.3.5, this parameter is available starting with V4.3.5 BP4.</p>
</main>

## Description

`mlog_trim_interval` specifies the interval for the scheduled automatic trimming task of materialized view logs (mlogs).

## Privilege requirements

* Querying the parameter

  You can run the `SHOW PARAMETERS` statement or query the `GV$OB_PARAMETERS` view in the `sys` tenant and all user tenants to query this parameter.

* Modifying the parameter

  You can modify this parameter in the `sys` tenant and user tenants.

## Attribute description

| **Attribute** | **Description** |
|----------|----------|
| Parameter type  | Time |
| Default value    | 1d |
| Value range  | [5s, 30d] |
| Modifiability | This parameter can be modified by using the `ALTER SYSTEM SET` statement. |
| Effective only after OBServer restart | No, the setting takes effect immediately. |

## Examples

* Set the interval for the mlog trimming task to `5s`.

    ```shell
    obclient> ALTER SYSTEM SET mlog_trim_interval = '5s';
    ```

* Set the interval for the mlog trimming task to `1d`.

    ```shell
    obclient> ALTER SYSTEM SET mlog_trim_interval = '1d';
    ```
