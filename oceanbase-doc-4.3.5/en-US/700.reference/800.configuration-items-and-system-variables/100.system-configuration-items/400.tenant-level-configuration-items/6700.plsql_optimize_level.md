# plsql_optimize_level

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is removed in V4.2.3 BP1 and replaced with the system variable <a href="../../200.system-variable/300.global-system-variable/10910.plsql_optimize_level-global.md">plsql_optimize_level (Global)</a>. </p>
</main>

## Description

`plsql_optimize_level` specifies the optimization level when PL/SQL code is compiled.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can query this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view.

* Modify the parameter

  The `sys` tenant and user tenants can modify this parameter.

## Attributes

|        Attribute        |                 Description                 |
|------------------|------------------------------------|
| Type             | Int                               |
| Default value               | 1                                  |
| Value range             | [0,3]<ul><li>0: Disable both frontend and backend optimizations. This option minimizes compilation overhead and time but may increase execution time. </li><li>1: Disable frontend optimization and enable backend optimization. This option significantly reduces compilation overhead and time but may slightly increase execution time. </li><li>2: Enable both frontend and backend optimizations. This option provides a balanced compilation and execution performance, which is the default optimization behavior. </li><li>3: Attempt to enable more optimization strategies, which may bring additional performance improvements.  |
| Effective upon OBServer node restart | No. The setting takes effect immediately.                      |

## Examples

Enable optimization when PL/SQL code is compiled.

```shell
obclient> ALTER SYSTEM SET plsql_optimize_level=1;
```
