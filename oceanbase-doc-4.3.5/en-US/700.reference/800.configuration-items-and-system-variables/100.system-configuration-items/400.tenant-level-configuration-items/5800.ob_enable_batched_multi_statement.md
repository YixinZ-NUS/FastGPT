# ob_enable_batched_multi_statement

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V2.2.50. </p>
</main>

## Description

`ob_enable_batched_multi_statement` specifies whether to enable the optimization for grouped execution of batched multi-statement operations.

## Privilege requirements

* Query the parameter

  `sys` tenant and all user tenants can query this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view.

* Modify the parameter

  `sys` tenant and user tenants can modify this parameter.

## Attributes

|      **Attribute**      |    **Description**    |
|------------------|--------------|
| Type             | Bool                       |
| Default value               | False                      |
| Value range             | <ul><li>False: disable the optimization. </li><li>True: enable the optimization.  </li></ul> |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately.   |

## Examples

Disable the optimization for grouped execution of batched multi-statement operations.

```shell
obclient> ALTER SYSTEM SET ob_enable_batched_multi_statement= False;
```
