# enable_manual_storage_cache_policy

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This parameter is introduced in BP2 of V4.3.5. </p>
</main>

## Description

`enable_manual_storage_cache_policy` specifies whether to enable the hot data cache strategy manually specified by users.

<main id="notice">
<h4>Applicability</h4>
<p>This parameter applies only to Shared-Storage (SS) mode in OceanBase Database. </p>
</main>

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can execute the `SHOW PARAMETERS` statement or query the parameter in the `GV$OB_PARAMETERS` view.

* Modify the parameter

  The `sys` tenant and user tenants can modify the parameter.

  <main id="notice" type='notice'>
  <h4>Notice</h4>
  <p>The hot data cache strategy specified by users takes effect only in user tenants in MySQL mode. If you modify the parameter in an Oracle tenant or the <code>sys</code> tenant for a user tenant or a non-user tenant such as the <code>sys</code> tenant or Meta tenant, the modification succeeds but the feature does not take effect. </p>
  </main>

## Attributes

| **Attribute** | **Description** |
|----------|----------|
| Parameter type  | Bool  |
| Default value    | True |
| Value range   | <ul><li>`True`: enables the hot data cache strategy specified by all users and uses the value of the `storage_cache_policy` parameter specified in tables, partitions, or indexes. </li> <li>`False`: disables the hot data cache strategy specified by all users and uses the value of the `storage_cache_policy` parameter specified in tables, partitions, or indexes. If no value is specified for tables, partitions, or indexes, the value specified for this parameter is used as the hot data cache strategy for all tables, partitions, and indexes. </li></ul> |
| Modifiability | Yes. You can modify the parameter by using the `ALTER SYSTEM SET` statement. |
| Effective only after OBServer restart | No, the setting takes effect immediately. |

## Examples

* Enable the hot data cache strategy in a user tenant.

  ```shell
  obclient> ALTER SYSTEM SET enable_manual_storage_cache_policy= True;
  ```

* Enable the hot data cache strategy for the `mysqltenant` tenant in the `sys` tenant.

  ```shell
  obclient> ALTER SYSTEM SET enable_manual_storage_cache_policy= True TENANT = mysqltenant;
  ```
