# sql_work_area

  <main id="notice" type='explain'>
    <h4>Note</h4>
    <p>This parameter is no longer available in the current version. </p>
  </main>

## Description

`sql_work_area` specifies the size of the work area for a tenant.

## Privilege requirements

* Query the parameter

  You can query this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view in the `sys` tenant and all user tenants.

* Modify the parameter

  You can modify this parameter in the `sys` tenant and user tenants.

## Attributes

|        Attribute      |  Description  |
|------------------|-------|
| Type             | Capacity       |
| Default value               | 1G. The default unit is MB. You must specify the unit when you configure this parameter.      |
| Value range             | [10M, +âˆž) |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately.   |

## Examples

Set the size of the work area for a tenant to 1G.

```shell
obclient> ALTER SYSTEM SET sql_work_area='1G';
```
