# rpc_memory_limit_percentage

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>For V4.1.0, this parameter is available starting with V4.1.0. </p>
</main>

## Description

`rpc_memory_limit_percentage` specifies the maximum percentage of the total memory of a tenant that can be used for RPC.

## Privilege requirements

* Query the parameter

  You can query this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view in the `sys` tenant and all user tenants.

* Modify the parameter

  You can modify this parameter in the `sys` tenant and user tenants.

## Attributes

|        Attribute      |  Description  |
|------------------|-------|
| Type             | Int |
| Default value               | 0, which indicates that no limit is set for the memory used for RPC. |
| Value range             | [0,100] |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately.    |

## Examples

Set the maximum percentage of the total memory of a tenant that can be used for RPC to 10.

```shell
obclient> ALTER SYSTEM SET rpc_memory_limit_percentage=10;
```
