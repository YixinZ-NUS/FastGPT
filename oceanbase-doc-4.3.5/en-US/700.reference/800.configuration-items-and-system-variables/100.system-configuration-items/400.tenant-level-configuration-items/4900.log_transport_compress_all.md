# log_transport_compress_all

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>For V4.1.0, this parameter is available starting with V4.1.0.</p>
</main>

## Description

`log_transport_compress_all` specifies whether to enable compression for log transmission.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can query this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view.

* Modify the parameter

  The `sys` tenant and user tenants can modify this parameter.

## Attributes

|      **Attribute**      |    **Description**    |
|------------------|--------------|
| Type             | Bool |
| Default value               | False |
| Value range             |<ul><li>False: Disable compression for log transmission.  </li><li>True: Enable compression for log transmission.</li></ul>|
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter.|
| Effective upon OBServer node restart | No. The setting takes effect immediately.    |

## Examples

Disable compression for log transmission.

```shell
obclient> ALTER SYSTEM SET log_transport_compress_all= False;
```
