# default_auto_increment_mode

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is available starting with V4.0.0.</p>
</main>

## Description

`default_auto_increment_mode` specifies the default auto-increment mode for auto-increment columns.

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can use the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view to query this parameter.

* Modify the parameter

  The `sys` tenant and user tenants can modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Type             | String  |
| Default value               | order |
| Value range             | <ul><li>order: The data of the auto-increment column remains increasing. </li> <li>noorder: The auto-increment column only guarantees the uniqueness of the auto-increment value. </li></ul> |
| Modifiable           | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | No. The setting takes effect immediately. |

## Examples

Set the default auto-increment mode for auto-increment columns to order.

```shell
obclient> ALTER SYSTEM SET default_auto_increment_mode = 'order';
```
