# suspend_storage_cache_task

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This parameter is introduced in BP2 of V4.3.5. </p>
</main>

## Description

`suspend_storage_cache_task` specifies whether to suspend background cache tasks.

<main id="notice">
<h4>Applicability</h4>
<p>This parameter applies only to Shared-Storage (SS) mode in OceanBase Database. </p>
</main>

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can execute the `SHOW PARAMETERS` statement or query the parameter in the `GV$OB_PARAMETERS` view.

* Modify the parameter

  The `sys` tenant and user tenants can modify the parameter.

  <main id="notice" type='notice'>
  <h4>Notice</h4>
  <p>Due to the fact that the manual hot data cache strategy only takes effect in MySQL mode, if you modify this parameter in Oracle mode or if the <code>sys</code> tenant modifies this parameter for a user tenant in MySQL mode or another non-user tenant (including the <code>sys</code> tenant and the Meta tenant), the modification succeeds but the feature does not take effect. </p>
  </main>

## Attributes

| **Attribute** | **Description** |
|----------|----------|
| Parameter type  | Bool  |
| Default value    | False |
| Value range   | <ul><li>`True`: Suspends the current cache tasks and newly initiated cache tasks.</li> <li>`False`: Does not suspend background cache tasks, that is, allows cache tasks to proceed normally.</li></ul> |
| Modifiability | Yes. You can modify the parameter by using the `ALTER SYSTEM SET` statement. |
| OBServer node restart required for take effect | No. The setting takes effect immediately. |

## Examples

* User tenant

  1. Suspend cache tasks in the current tenant.

     ```shell
     obclient> ALTER SYSTEM SET suspend_storage_cache_task = True;
     ```

  2. Resume cache tasks in the current tenant.

     ```shell
     obclient> ALTER SYSTEM SET suspend_storage_cache_task = False;
     ```

* sys tenant

  1. Suspend cache tasks in the `mysqltenant` tenant.

     ```shell
     obclient> ALTER SYSTEM SET suspend_storage_cache_task = True TENANT = mysqltenant;
     ```

  2. Resume cache tasks in the `mysqltenant` tenant.

      ```shell
      obclient> ALTER SYSTEM SET suspend_storage_cache_task = False TENANT = mysqltenant;
      ```
