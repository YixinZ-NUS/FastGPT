# enable_adaptive_plan_cache

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This parameter is introduced in OceanBase Database V4.3.5 BP1. </p>
</main>

## Description

`enable_adaptive_plan_cache` specifies whether to enable adaptive plan cache.

<main id="notice">
  <h4>Applicability</h4>
  <p>This parameter applies to OceanBase Database in MySQL mode and Oracle mode. </p>
</main>

## Privilege requirements

* Querying the parameter

  You can execute the `SHOW PARAMETERS` statement or query the `GV$OB_PARAMETERS` view to query the value of this parameter in the `sys` tenant and all user tenants.

* Modifying the parameter

  You can modify this parameter only in the `sys` tenant. User tenants cannot modify this parameter. / You can modify this parameter in both the `sys` tenant and user tenants.

## Attribute description

| **Attribute**  |    **Description**  |
|-----------|---------------|
| Parameter type    |    BOOL              |
| Default value      |  False  |
| Value range    |      <li>True: enables the feature.</li><li>False: disables the feature.</li>          |
| Modifiability | Yes. You can execute the `ALTER SYSTEM SET` statement to modify this parameter. |
| Effective only after OBServer restart | No |

## Considerations

After you enable the adaptive plan cache feature, the system automatically enables or disables the plan cache based on the SQL characteristics.

## Examples

Run the following statement to enable the adaptive plan cache feature:

```shell
obclient> ALTER SYSTEM SET enable_adaptive_plan_cache = true;
```
