# audit_log_query_sql

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p><ul><li>For V4.3.x, this parameter is available starting with V4.3.3. </li> <li>For V4.2.x, this parameter is available starting with V4.2.4. </li></ul></p>
</main>

## Description

`audit_log_query_sql` specifies how to record SQL statements in audit logs.

<main id="notice" >
  <h4>Applicability</h4>
  <p>This topic applies only to OceanBase Database Enterprise Edition. OceanBase Database Community Edition does not support audit logs. </p>
</main>

## Privilege requirements

* Query the parameter

  The `sys` tenant and all user tenants can query this parameter by using the `SHOW PARAMETERS` statement or the `GV$OB_PARAMETERS` view.

* Modify the parameter

  The `sys` tenant and user tenants can use the `ALTER SYSTEM SET` statement to modify this parameter.

## Attributes

| **Attribute** | **Description** |
| -------- | -------- |
| Type   | String |
| Default value     | ALL |
| Value range   | <ul><li>ALL: records all Query SQL statements. </li><li>NONE: does not record Query SQL statements. </li></ul>|
| Modifiable | Yes. You can use the `ALTER SYSTEM SET` statement to modify the parameter. |
| Effective upon OBServer node restart | No. The setting takes effect immediately.   |

## Examples

Set the parameter to disable the recording of Query SQL statements in audit logs.

```shell
obclient> ALTER SYSTEM SET audit_log_query_sql='NONE';
```
