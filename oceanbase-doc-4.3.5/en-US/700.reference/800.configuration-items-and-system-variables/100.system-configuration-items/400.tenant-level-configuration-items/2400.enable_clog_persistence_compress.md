# enable_clog_persistence_compress

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>Due to the storage architecture change of OceanBase Database V4.x, OceanBase Database V4.0.0 does not support log compression for storage. This parameter has been deprecated since OceanBase Database V4.0.0. </li>
<li>OceanBase Database V4.2.1 and later support log compression for storage to adapt to business needs. In OceanBase Database V4.2.1 and later except V4.3.0, you can use the tenant-level parameter <code>log_storage_compress_all</code> to specify whether to compress logs for storage. </li></ul>
</main>

## Description

`enable_clog_persistence_compress` specifies whether to compress transaction logs for storage.

## Attributes

| **Attribute** | **Description** |
|------------------|--------|
| Type | Boolean |
| Default value | False |
| Value range | </li><li>True</li><li>False |
| Effective upon OBServer node restart | No |
