# ob_sparse_drop_ratio_search

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This variable is introduced in V4.3.5 BP2. </p>
</main>

## Description

`ob_sparse_drop_ratio_search` is used during sparse vector queries to set the smallest portion of dimension values in the query vector to 0. This reduces the dimensionality and accelerates the query process.

<main id="notice" type='explain'>
  <h4>Applicability</h4>
  <p>This variable applies only to the MySQL-compatible mode of OceanBase Database. </p>
</main>

## Privilege requirements

* Query variables

  * Both the `sys` tenant and all user tenants can use the `SHOW VARIABLES` statement or the `information_schema.SESSION_VARIABLES` view to check the values of session system variables.

* Modify variables

  * Both the `sys` tenant and all MySQL-compatible tenants can directly modify the values of session system variables for their respective tenants.

## Attributes

| **Attribute**  | **Description**  |
|---------|---------|
| Parameter type    | Uint |
| Default value     | 0    |
| Value range    | [0,100]    |
| Effect scope    | SESSION |
| Modifiable? | Yes. You can use the `SET` or `ALTER SESSION SET` statement to modify it. |

## Examples

When configuring sparse vector queries, set the smallest 30% of dimension values in the query vector to `0`.

```shell
obclient> SET ob_sparse_drop_ratio_search = 30;
```
