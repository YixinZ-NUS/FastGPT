# event_scheduler

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>For OceanBase Database V4.3.5, this variable takes effect starting from V4.3.5 BP2.</p>
</main>

## Description

`event_scheduler` specifies whether to enable and run the event scheduler on the server.

<main id="notice">
  <h4>Applicability</h4>
  <p>This variable applies only to the MySQL mode of OceanBase Database. </p>
</main>

## Privileges

* Query variables

  The `sys` tenant and all user tenants can use the `SHOW VARIABLES` statement or the `SYS.TENANT_VIRTUAL_GLOBAL_VARIABLE` view (Oracle mode) and the `information_schema.GLOBAL_VARIABLES` view (MySQL mode) to check the values of global system variables.

* Modify variables

  * The `sys` tenant can directly modify the value of a global system variable.

  * MySQL tenants need to have the `SUPER` or `ALTER SYSTEM` privilege to modify the value of a global system variable.

  * Oracle tenants need to have the `ALTER SYSTEM` privilege to modify the value of a global system variable.

## Attributes

| **Attribute** | **Description** |
| --- | --- |
| Parameter type | Bool |
| Default value | OFF |
| Value range | <ul><li> OFF: the event scheduler is stopped. </li><li> ON: the event scheduler is started; the event scheduler thread is running and all scheduled events are executed. </li></ul>|
| Scope | Global |
|Modifiability|Yes|

## Examples

Execute the following statement to enable the event scheduler:

```sql
obclient> SET GLOBAL event_scheduler=ON;
```
