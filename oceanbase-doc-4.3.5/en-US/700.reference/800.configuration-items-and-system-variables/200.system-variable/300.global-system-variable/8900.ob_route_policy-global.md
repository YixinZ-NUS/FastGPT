# ob_route_policy

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This variable was introduced in V1.4. </p>
</main>

## Description

The `ob_route_policy` variable specifies the routing strategy for the OBServer node to select a data replica.

## Privilege requirements

* Query variables

  * Global level

    The `sys` tenant and all user tenants can execute the `SHOW VARIABLES` statement or the `SYS.TENANT_VIRTUAL_GLOBAL_VARIABLE` view (in Oracle mode) or the `information_schema.GLOBAL_VARIABLES` view (in MySQL mode) to query the value of a global system variable.

  * Session level

    The `sys` tenant and all user tenants can execute the `SHOW VARIABLES` statement or the `SYS.TENANT_VIRTUAL_SESSION_VARIABLE` view (in Oracle mode) or the `information_schema.SESSION_VARIABLES` view (in MySQL mode) to query the value of a session system variable.

* Modify variables

  * Set the variable at the global level

    * The `sys` tenant can directly modify the value of a global system variable.

    * MySQL user tenants must have the `SUPER` or `ALTER SYSTEM` privilege to modify the value of a global system variable.

    * Oracle user tenants must have the `ALTER SYSTEM` privilege to modify the value of a global system variable.

  * Set the variable at the session level

    The `sys` tenant and all user tenants can directly modify the value of a session system variable in the current tenant.

## Attribute descriptions

|  **Attribute**  |             **Description**       |
|------------|----------------------------|
| Parameter type | Enum  |
| Default value       | 1     |
| Value range | <ul><li> 1 (`READONLY_ZONE_FIRST`): Zones with read-only replicas are prioritized. </li>  <li> 2 (`ONLY_READONLY_ZONE`): Requests are sent only to zones with read-only replicas. </li> <li> 3 (`UNMERGE_ZONE_FIRST`): Zones with replicas not in the major compaction state are prioritized. </li> <li> 4 (`UNMERGE_FOLLOWER_FIRST`): Follower replicas not in the merge state are prioritized. </li>  <li> 5 (`FORCE_READONLY_ZONE`): All read-only requests are not forwarded to non-read-only replicas.  <main id="notice" type='explain'>  <h4>Note</h4>  <p>For V4.3.x, this value was introduced starting from V4.3.5 BP1.</p> </main> </li> </ul> |
| Effective scope | <li> Global   <li> Session                            |
| Modifiable     | Yes. You can execute the `SET` statement to modify the value.|

## Note

For V4.3.5 and later, the value of the variable `ob_route_policy` has been changed as follows:

* FORCE_READONLY_ZONE: After the parameter value is set, strong-consistency and DML requests are executed normally without errors.
* COLUMN_STORE_ONLY: After the parameter value is set, strong-consistency and DML requests are executed normally without errors.

## Examples

Set the routing policy for selecting data replicas inside the OBServer node to prioritize the Zone where the read-only replica is located.

* Set the variable at the session level

  ```shell
  obclient> SET ob_route_policy = 'READONLY_ZONE_FIRST';
  ```

  or

  ```shell
  obclient> SET ob_route_policy = 1;
  ```

* Set the variable at the global level

  ```shell
  obclient> SET GLOBAL ob_route_policy = 'READONLY_ZONE_FIRST';
  ```

  or

  ```shell
  obclient> SET GLOBAL ob_route_policy = 1;
  ```
