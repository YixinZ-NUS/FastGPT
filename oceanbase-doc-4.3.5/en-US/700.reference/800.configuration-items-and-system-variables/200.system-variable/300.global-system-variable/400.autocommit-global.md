# autocommit

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This variable is available starting with V1.4. </p>
</main>

## Description

`autocommit` specifies whether to automatically commit transactions.

## Privilege requirements

* Query variables

  * Global level

    `sys` tenants and all user tenants can execute the `SHOW VARIABLES` statement or query the `SYS.TENANT_VIRTUAL_GLOBAL_VARIABLE` view (Oracle mode) or the `information_schema.GLOBAL_VARIABLES` view (MySQL mode) to view the values of Global system variables.

  * Session level

    `sys` tenants and all user tenants can execute the `SHOW VARIABLES` statement or query the `SYS.TENANT_VIRTUAL_SESSION_VARIABLE` view (Oracle mode) or the `information_schema.SESSION_VARIABLES` view (MySQL mode) to view the values of Session system variables.

* Modify variables

  * Global level

    * `sys` tenants can directly modify the values of Global system variables.

    * MySQL user tenants must have the `SUPER` or `ALTER SYSTEM` privilege to modify the values of Global system variables.

    * Oracle user tenants must have the `ALTER SYSTEM` privilege to modify the values of Global system variables.

  * Session level

    `sys` tenants and all user tenants can directly modify the values of Session system variables.

## Attributes

| **Attribute**  |    **Description**  |
|-----------|---------------|
| Type    | Bool                    |
| Default value      | 1                       |
| Value range    | <ul><li>1: Automatic commit</li><li>0: No automatic commit</li></ul>   |
| Scope    |  <ul><li>Global</li><li>Session</li></ul>  |
| Modifiable  | Yes. You can execute the `SET` statement to modify the value.|

## Examples

Set whether to automatically commit transactions.

* Global level

   ```shell
   obclient> SET GLOBAL autocommit=1;
   ```

* Session level

   ```shell
   obclient> SET autocommit=1;
   ```
