# plsql_optimize_level

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This variable is available starting with V4.2.3 BP1. </p>
</main>

## Description

`plsql_optimize_level` specifies the optimization level when PL/SQL code is compiled.

## Privilege requirements

* Query variables

    The `sys` tenant and all user tenants can execute the `SHOW VARIABLES` statement or query the `SYS.TENANT_VIRTUAL_SESSION_VARIABLE` view (Oracle mode) or the `information_schema.SESSION_VARIABLES` view (MySQL mode) to view the values of session system variables.

* Modify variables

    The `sys` tenant and all user tenants can directly modify the values of session system variables in their own tenants.

## Description

|        Attribute        |                 Description                 |
|------------------|------------------------------------|
| Parameter type             | Int                               |
| Default value              | 2                                  |
| Value range             | [0,3]<ul><li>0: Disables both frontend and backend optimizations. This minimizes compilation overhead and time, but may increase execution time. </li><li>1: Disables frontend optimization and enables backend optimization. This significantly reduces compilation overhead and time, but may slightly increase execution time. </li><li>2: Enables both frontend and backend optimizations, providing a balanced compilation and execution performance. This is the default optimization behavior. </li><li>3: Attempts to enable more optimization strategies, which may result in additional performance improvements. |
| Scope |  <ul><li>Global  </li><li>Session </li></ul>|
| Modifiable          | Yes. You can execute the `SET` or `ALTER SYSTEM SET` statement to modify the value.          |

## Considerations

A higher optimization level means that the compiler performs more optimization operations. We recommend that you set `plsql_optimize_level` to 2.

## Examples

* MySQL mode

  * Set the PL optimization level to `1` globally to disable frontend optimization and enable backend optimization.

    ```shell
    obclient> SET GLOBAL plsql_optimize_level = 1;
    ```

  * Set the PL optimization level to `2` globally to enable frontend and backend optimizations.

    ```shell
    obclient> SET GLOBAL plsql_optimize_level = 2;
    ```

* Oracle mode

  * Set the PL optimization level to `1` globally to disable frontend optimization and enable backend optimization.

    ```shell
    obclient> ALTER SYSTEM SET plsql_optimize_level = 1;
    ```

  * Set the PL optimization level to `2` globally to enable frontend and backend optimizations.

    ```shell
    obclient> ALTER SYSTEM SET plsql_optimize_level = 2;
    ```
