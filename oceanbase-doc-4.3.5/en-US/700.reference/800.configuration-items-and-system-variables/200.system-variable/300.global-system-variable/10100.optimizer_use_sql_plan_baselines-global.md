# optimizer_use_sql_plan_baselines

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This variable is available starting with V2.1. </p>
</main>

## Description

The `optimizer_use_sql_plan_baselines` variable controls whether the optimizer uses a plan baseline.

## Required privileges

* Query variables

  * Global level

    `sys` tenants and user tenants can execute the `SHOW VARIABLES` statement or view `SYS.TENANT_VIRTUAL_GLOBAL_VARIABLE` (Oracle mode) or `information_schema.GLOBAL_VARIABLES` (MySQL mode) to query the value of a global system variable.

  * Session level

    `sys` tenants and user tenants can execute the `SHOW VARIABLES` statement or view `SYS.TENANT_VIRTUAL_SESSION_VARIABLE` (Oracle mode) or `information_schema.SESSION_VARIABLES` (MySQL mode) to query the value of a session system variable.

* Modify variables

  * Set the variable at the global level

    * `sys` tenants can directly modify the value of a global system variable.

    * MySQL user tenants must have the `SUPER` or `ALTER SYSTEM` privilege to modify the value of a global system variable.

    * Oracle user tenants must have the `ALTER SYSTEM` privilege to modify the value of a global system variable.

  * Set the variable at the session level

    `sys` tenants and user tenants can directly modify the value of a session system variable in their own tenant.

## Attributes

| **Attribute**  |               **Description**                |
|---------|--------------------------------------------------------------------------------|
| Type    | bool                                |
| Default value     | 0                                   |
| Value range    | <li> 0: indicates that the optimizer does not consider plans in plan baselines when generating new plans. It directly generates and executes new plans by using the optimizer.   <li> 1: indicates that the optimizer prefers to use plans in plan baselines when generating new plans. For new plans that are different from those in plan baselines, the optimizer will use them only after they are verified.    |
| Effect scope    | <li> Global   <li> Session                |
| Modifiable  | Yes. You can modify this variable by using the `SET` statement. |

## Examples

Set the optimizer to not use plans in plan baselines.

* Global level

  ```shell
  obclient> SET GLOBAL optimizer_use_sql_plan_baselines = 0;
  ```

* Session level

  ```shell
  obclient> SET optimizer_use_sql_plan_baselines = 0;
  ```
