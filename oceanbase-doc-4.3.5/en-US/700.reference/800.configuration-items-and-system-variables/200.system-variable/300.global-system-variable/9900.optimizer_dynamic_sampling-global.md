# optimizer_dynamic_sampling

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This variable is avaiable starting with V4.2.0.</p>
</main>

## Description

`optimizer_dynamic_sampling` specifies the level of dynamic sampling.

## Privilege requirements

* Query variables

  * Global level

    `sys` tenant and all user tenants can execute the `SHOW VARIABLES` statement or query the `SYS.TENANT_VIRTUAL_GLOBAL_VARIABLE` view (Oracle mode) or `information_schema.GLOBAL_VARIABLES` view (MySQL mode) to obtain the value of a global system variable.

  * Session level

    `sys` tenant and all user tenants can execute the `SHOW VARIABLES` statement or query the `SYS.TENANT_VIRTUAL_SESSION_VARIABLE` view (Oracle mode) or `information_schema.SESSION_VARIABLES` view (MySQL mode) to obtain the value of a session system variable.

* Modify variables

  * Global level

    * `sys` tenant can directly modify the value of a global system variable.

    * MySQL user tenants must have the `SUPER` or `ALTER SYSTEM` privilege to modify the value of a global system variable.

    * Oracle user tenants must have the `ALTER SYSTEM` privilege to modify the value of a global system variable.

  * Session level

    `sys` tenant and all user tenants can directly modify the value of a session system variable.

## Attributes

| **Attribute**  |    **Description**  |
|-----------|---------------|
| Type    | Int |
| Default value      | 1 |
| Value range    | [0, 1] |
| Scope    | <ul><li>Global</li><li>Session</li></ul>  |
| Modifiable  | Yes. You can execute the `SET` statement to modify the value. |

## Examples

Set the level of dynamic sampling to 1.

* Global level

  ```shell
  obclient> SET GLOBAL optimizer_dynamic_sampling=1;
  ```

* Session level
  ```shell
  obclient> SET optimizer_dynamic_sampling=1;
  ```
