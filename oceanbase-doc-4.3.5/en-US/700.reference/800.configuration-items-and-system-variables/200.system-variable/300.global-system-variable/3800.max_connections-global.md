# max_connections

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This variable is available starting with V2.2.77. </p>
</main>

## Description

The `max_connections` variable specifies the maximum number of connections allowed for the entire tenant.

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This variable is effective only in MySQL mode. </p>
</main>

## Privilege requirements

* Query a variable

  The `sys` tenant and all user tenants can execute the `SHOW VARIABLES` statement or view the `information_schema.GLOBAL_VARIABLES` view (MySQL mode) to query the value of a global system variable.

* Modify a variable

  * The `sys` tenant can directly modify the value of a global system variable.

  * A MySQL user tenant needs the `SUPER` or `ALTER SYSTEM` privilege to modify the value of a global system variable.

## Examples

Set the maximum number of connections for the entire tenant to 2147483647.

```shell
obclient> SET GLOBAL max_connections=2147483647;
```
