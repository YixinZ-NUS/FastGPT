# ob_timestamp_service

<main id="notice" type='explain'>
<h4>Note</h4>
<p>In OceanBase Database V4.x, the system uses the Global Timestamp Service (GTS) by default, and you do not need to specify the timestamp service. This variable has been deprecated since OceanBase Database V4.0.0. </p>
</main>

## Description

`ob_timestamp_service` specifies the timestamp service to be used.

## Attributes

| **Attribute**  |           **Description**                         |
|---------|----------------------------------------------|
| Type    | Enum                         |
| Default value     | GTS                            |
| Valid values    | <li> LTS: The server's local timestamp is used as the transaction snapshot during transaction execution. This value does not support cross-partition access.   <li> GTS: The global timestamp from the GTS service is used as the transaction snapshot and the commit version number during transaction execution. There are no limitations on this value.   <li> (Not supported) HA_GTS    |
| Scope    | Global            |
