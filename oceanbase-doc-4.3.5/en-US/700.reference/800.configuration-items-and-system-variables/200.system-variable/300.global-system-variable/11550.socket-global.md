# socket

<main id="notice" type='explain'>
  <h4>Note</h4>
  <ul><li>For V4.3.x, this variable was introduced starting from V4.3.5 BP1.</li><li>For other versions of V4.2.x, this variable was introduced starting from V4.2.5 BP1. For V4.2.1, it was introduced starting from V4.2.1 BP10.</li></ul>
</main>

## Description

The `socket` variable specifies the name of the `Socket` file connected to the local client.

<main id="notice">
  <h4>Applicability</h4>
  <p> This variable applies only to the MySQL-compatible mode of OceanBase Database. </p>
</main>

## Privilege requirement

* Query variables

  The `sys` tenant and all user tenants can use the `SHOW VARIABLES` statement or the `information_schema.GLOBAL_VARIABLES` view (MySQL-compatible mode) to view the values of global system variables.

* Modify variables

  You cannot modify this variable.

## Attributes

| **Attribute**  |    **Description**  |
|-----------|---------------|
| Parameter type    | Varchar              |
| Default value      | /run/sql.sock   |
| Scope    |  GLOBAL  |
| Modifiable  | No       |

## Examples

Query the name of the `socket` file connected to the local client.

```shell
obclient> SHOW VARIABLES LIKE 'socket';
```

The query result is as follows:

```shell
+---------------+------------------------------------+
| Variable_name | Value                              |
+---------------+------------------------------------+
| socket        | /home/admin/oceanbase/run/sql.sock |
+---------------+------------------------------------+
1 row in set (0.003 sec)
```
