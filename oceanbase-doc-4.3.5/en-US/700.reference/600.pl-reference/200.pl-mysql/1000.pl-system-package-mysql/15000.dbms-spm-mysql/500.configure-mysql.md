# CONFIGURE

The `CONFIGURE` procedure is used to set some parameters of SPM.

## Syntax

```sql
DBMS_SPM.CONFIGURE (
    database_name     VARCHAR(65535),
    parameter_name    VARCHAR(65535),
    parameter_value   DECIMAL
  );
```

## Parameters

| **Parameter** | **Description** |
| --- | --- |
| database_name | The name of the database. This parameter is required only in MySQL-compatible mode. |
| parameter_name | The name of the parameter to be set. For more information, see **SPM parameters**. |
| parameter_value | The value to which the parameter is to be set. For more information, see **SPM parameters**. |

## SPM parameters

| SPM parameter | **Description** | **Possible values** | **Default value** |
| --- | --- | --- | --- |
| space_budget_percent | The maximum percentage of SYSAUX space allowed for SPM. | 1, 2, ..., 50 | 10 |
| plan_retention_weeks | The number of weeks after which a plan is automatically discarded if it is not used. | 5, 6, ..., 523 | 53 |

## Examples

Set the plan to be automatically purged if not used within 20 weeks.

```sql
BEGIN
  DBMS_SPM.CONFIGURE('plan_retention_weeks', 20);
END;
```
