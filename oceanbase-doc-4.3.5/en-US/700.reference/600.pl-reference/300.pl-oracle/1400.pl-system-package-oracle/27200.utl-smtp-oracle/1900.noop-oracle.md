# NOOP

Verifies whether the server is still connected and listening.

<main id="notice" >
    <h4>Applicability</h4>
    <p>This content applies only to the Oracle-compatible mode of OceanBase Database. </p>
</main>

## Syntax

```sql
UTL_SMTP.NOOP (
   c  IN OUT NOCOPY connection)
RETURN reply;

UTL_SMTP.NOOP (
   c  IN OUT NOCOPY connection);
```

## Parameters

| **Parameter** |         **Description**         |
|--------|------------------------|
|c|The SMTP connection. |

## Return value

| **Return value** |          **Description**          |
|---------|--------------------------|
|reply|The reply to the command. If multiple replies are returned, the last reply is returned. |

## Considerations

* It is used to trigger a successful response from the server. After `open_connect`, `noop` can be used to verify whether the server is still connected and properly listening.
* The expected response is a message starting with `250`, indicating that the server is connected and properly listening.
