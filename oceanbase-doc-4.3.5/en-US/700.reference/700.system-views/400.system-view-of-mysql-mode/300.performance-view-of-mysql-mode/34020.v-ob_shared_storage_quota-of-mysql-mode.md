# V$OB_SHARED_STORAGE_QUOTA

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view was introduced in OceanBase Database V4.3.4. </p>
</main>

## Purpose

This view displays the I/O access resource information on the current OBServer node.

## Columns

| Column     | Type          | Nullable? | Description                                                        |
|-------------|---------------|----------------|-------------------------------------------------------------|
| SVR_IP      | varchar(46)   | NO   | The IP address of the OBServer node.  |
| SVR_PORT    | bigint(20)    | NO   | The port of the OBServer node.     |
| ENDPOINT    | varchar(256)  | NO   | The endpoint of shared storage. If the shared-nothing (SN) mode is used, this field is empty.     |
| PATH        | varchar(1024) | NO   | The path in object storage. If the SN mode is used, this field displays the local path. </br>Commonly used path types are: `local://`, `oss://`, `file://`, and `s3://`.   |
| CLASS_ID    | bigint(20)    | NO   | The ID for labeling. The meaning of this field varies based on the purpose of the bucket in object storage:<ul><li>If the bucket is used for clog storage or data storage, this field is meaningless and has a value of `0`. </li> <li>If the bucket is used for backup and restore, data import, or data export, this field indicates the tenant ID. </li></ul>      |
| TYPE        | varchar(32)   | NO   | The metric type, which can be one of the following types:<ul><li>`IPS`: the number of instructions per second</li> <li>`OPS`: the number of operations per second</li> <li>`IOPS`: the number of input or output operations per second</li> <li>`IBW`: the input bandwidth</li> <li>`OBW`: the output bandwidth</li> <li>`IOBW`: the input or output bandwidth</li> <li>`TAG`: user-defined tag information</li></ul>     |
| REQUIREMENT | bigint(20)    | NO   | The server resource requirements    |
| ASSIGN      | bigint(20)    | NO   | The upper limit of the server resources specified by the throttling algorithm     |

## Sample query

In a user tenant, query the applied and allocated quotas for each I/O access metric on the current OBServer node for clog storage and data storage scenarios. For backup and restore, data import, and data export scenarios, query the applied and allocated quotas for each I/O access metric on the current OBServer node for the tenant.

```shell
obclient> SELECT * FROM oceanbase.V$OB_SHARED_STORAGE_QUOTA;
```
