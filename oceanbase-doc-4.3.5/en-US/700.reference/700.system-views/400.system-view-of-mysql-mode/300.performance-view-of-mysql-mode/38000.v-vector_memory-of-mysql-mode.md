# V$VECTOR_MEMORY

<main id="notice" type='explain'>
  <h4>Note</h4>
  <ul><li>This view is introduced in V4.3.5 BP3. </li></ul>
</main>

## Purpose

This view shows the detailed statistics about the memory usage of vector index data on the current OBServer node for the current tenant.

## Columns

| Column | Type | Nullable? | Description |
| ---- | ---- | ---- | ---- |
| TENANT_ID         | bigint(20)          | NO   |  Tenant ID: <ul><li>1: the ID of the sys tenant.  </li><li>Other values: the ID of a user tenant or Meta tenant.  </li></ul>   |
| SVR_IP            | varchar(46)         | NO   |  The IP address of the node.   |
| SVR_PORT          | bigint(20)          | NO   |  The port number of the node.   |
| VECTOR_MEM_HOLD   | bigint(20)          | NO   |  The size of memory occupied by vector index data, in bytes.   |
| VECTOR_MEM_LIMIT  | bigint(20)          | NO   |  The maximum size of memory allowed for vector index data, in bytes.   |
| VECTOR_MEM_USED   | bigint(20)          | NO   |  The actual memory used by vector index data, in bytes.   |

## Examples

The following example shows how to query the detailed statistics about the memory usage of vector index data for the current tenant.

```shell
OceanBase(root@test)>SELECT * FROM oceanbase.V$VECTOR_MEMORY  \G
```

The query result is as follows:

```shell
*************************** 1. row ***************************
                svr_ip: 11.158.31.35
              svr_port: 46002
             tenant_id: 1004
       vector_mem_hold: 82231296
       vector_mem_used: 77876328
      vector_mem_limit: 429496720
1 row in set
```
