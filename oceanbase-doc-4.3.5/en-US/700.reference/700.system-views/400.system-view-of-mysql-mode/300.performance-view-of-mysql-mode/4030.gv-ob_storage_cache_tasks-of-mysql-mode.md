# GV$OB_STORAGE_CACHE_TASKS

<main id="notice" type='explain'>
<h4>Note</h4>
<p>In V4.3.5, this view is introduced in V4.3.5 BP2. </p>
</main>

## Purpose

This view displays the cache tasks of the current tenant on all OBServer nodes.

<main id="notice" type='explain'>
<h4>Note</h4>
<p> This view applies only to Shared-Storage (SS) mode. In Shared-Nothing (SN) mode, the query result of this view is empty. </p>
</main>

## Columns

|      Column      |         Type          | Nullable? |                       Description      |
|-----------------|-----------------------|------------------------|---------------------------------------|
| SVR_IP          | varchar(46)           | NO                     | The IP address of the OBServer node.    |
| SVR_PORT        | bigint(20)            | NO                     | The port number of the OBServer node.    |
| TENANT_ID       | bigint(20)            | NO                     | The ID of the tenant to which the tablet belongs. |
| TABLET_ID       | bigint(20)            | NO                     | The tablet ID.                          |
| STATUS          | varchar(64)           | NO                     | The status of the cache task. Valid values: <ul><li>`WAITING`: The task is being initialized. </li> <li>`DOING`: The task is being executed. </li> <li>`FINISHED`: The task is completed. </li> <li>`SUSPENDED`: The task is suspended. </li> <li>`CANCELED`: The task is canceled. </li> <li>`FAILED`: The task execution failed. </li></ul> |
| SPEED           | varchar(29)           | NO                     | The pull speed of the tablet.           |
| START_TIME      | timestamp(6)          | NO                     | The start time of the cache task.       |
| COMPLETE_TIME   | timestamp(6)          | NO                     | The end time of the cache task.         |
| RESULT          | bigint(20)            | NO                     | The error code if the task execution fails. |
| COMMENT         | varchar(4096)         | NO                     | The detailed error description when the task fails, which contains the `trace_id`. |
