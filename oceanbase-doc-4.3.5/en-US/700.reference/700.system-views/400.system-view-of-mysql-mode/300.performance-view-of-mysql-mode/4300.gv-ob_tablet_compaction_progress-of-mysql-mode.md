# GV$OB_TABLET_COMPACTION_PROGRESS

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is introduced in V4.0.0. </p>
</main>

## Purpose

This view displays the progress information of tablet-level compactions in the current tenant across all nodes.

## Columns

| Column | Type | Nullable? | Description |
| --- | --- | --- | --- |
| SVR_IP                       | varchar(46)  | NO         | The IP address of the server.                                                                                |
| SVR_PORT                     | bigint(20)   | NO         | The port number of the server. |
| TENANT_ID                    | bigint(20)   | NO         | The ID of the tenant. |
| TYPE                         | varchar(64)  | NO         | The type of compaction. Valid values: <ul><li>`MDS_TABLE_MERGE`: persists the system metadata to disk in the format of a SSTable. </li> <li>`MAJOR_MERGE`: performs major compaction for a tenant. </li> <li>`MEDIUM_MERGE`: performs major compaction for a partition. </li> <li>`MINI_MERGE`: converts a MemTable into a Mini SSTable. </li> <li>`MINOR_MERGE`: merges multiple Mini SSTables into a new Mini SSTable or merges multiple Mini SSTables and a Minor SSTable into a new Minor SSTable. </li> <li>`META_MAJOR_MERGE`: a special type of compaction that synthesizes data before a specified point in time into a Meta Major SSTable. The data in the Meta Major SSTable is stored in the same format as that in a Major SSTable, but does not contain multi-version data or uncommitted transaction data. </li></ul>    |
| LS_ID                        | bigint(20)   | NO         | The ID of the log stream. |
| TABLET_ID                    | bigint(20)   | NO         | The ID of the data shard.                                                                                  |
| COMPACTION_SCN                      | bigint(20) unsigned   | NO         | The version information of the compaction. |
| TASK_ID                      | varchar(64)  | NO         | The ID of the executed trace.                                                                                |
| STATUS                       | varchar(15)  | NO         | The status of the task.  |
| DATA_SIZE                    | bigint(20)   | NO         | The total amount of data to be scanned.                                                                                |
| UNFINISHED_DATA_SIZE         | bigint(20)   | NO         | The amount of data that has not been scanned.                                                                                 |
| PROGRESSIVE_COMPACTION_ROUND | bigint(20)   | NO         | The number of progressive compaction rounds. |
| CREATE_TIME                  | timestamp(6) | NO         | The creation time of the task. |
| START_TIME                   | timestamp(6) | NO         | The start time.  |
| ESTIMATED_FINISH_TIME        | timestamp(6) | NO         | The estimated completion time. |
| START_CG_ID                  | bigint(20)   | NO         | The start ID of the column group. <main id="notice" type='explain'><h4>Note</h4><p>This field is introduced in V4.3.0. </p></main>  |
| END_CG_ID                    | bigint(20)   | NO         | The end ID of the column group. <main id="notice" type='explain'><h4>Note</h4><p>This field is introduced in V4.3.0. </p></main>

## Sample query

A user queries the progress of major compactions on all nodes of the current tenant.

```shell
obclient [oceanbase]> SELECT * FROM oceanbase.GV$OB_TABLET_COMPACTION_PROGRESS WHERE TYPE='MAJOR_MERGE'\G
```
