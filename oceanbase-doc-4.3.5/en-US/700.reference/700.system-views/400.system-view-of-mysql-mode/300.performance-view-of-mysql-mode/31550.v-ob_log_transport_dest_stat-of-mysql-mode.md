# V$OB_LOG_TRANSPORT_DEST_STAT

<main id="notice" type='explain'>
  <h4>Note</h4>
  <ul><li>The V$OB_LOG_TRANSPORT_DEST_STAT view was introduced in V4.3.5 for V4.3.x. </li> <li>The V$OB_LOG_TRANSPORT_DEST_STAT view was introduced in V4.2.5 for V4.2.x. </li></ul>
</main>

## Purpose

The `V$OB_LOG_TRANSPORT_DEST_STAT` view displays information about downstream log synchronization links (OBCDCs or network standby databases) of the current tenant on the current OBServer node.

## Columns

| **Column** | **Type** | **Nullable?** | **Description** |
| -------- | -------- | ------------------ | -------- |
| TENANT_ID                   | bigint(20)          | NO   | Tenant ID     |
| SVR_IP                      | varchar(46)         | NO   | Server IP     |
| SVR_PORT                    | bigint(20)          | NO   | Server RPC_PORT     |
| LS_ID                       | bigint(20)          | NO   | ID of the log stream to which the request belongs     |
| CLIENT_IP                   | varchar(46)         | NO   | IP address of the machine where the process of the client is located     |
| CLIENT_PID                  | bigint(20)          | NO   | Process ID of the client     |
| CLIENT_TENANT_ID            | bigint(20)          | NO   | Tenant ID of the client     |
| CLIENT_TYPE                 | varchar(7)          | NO   | Type of the client:<ul><li>STANDBY: network standby database </li><li>CDC: OBCDC </li><li>UNKNOWN: unknown client type </li></ul>    |
| START_SERVE_TIME            | timestamp(6)        | NO   | Creation time of the record     |
| LAST_SERVE_TIME             | timestamp(6)        | NO   | Modification time of the record     |
| LAST_READ_SOURCE            | varchar(7)          | NO   | Source of the logs read during the last request:<ul><li>ONLINE: online logs </li><li>ARCHIVE: archived logs </li><li>UNKNOWN: unknown source </li></ul>     |
| LAST_REQUEST_TYPE           | varchar(24)         | NO   | RPC type of the last request:<ul><li>0: SEQUENTIAL_READ_SERIAL, which indicates that the serial log transfer protocol is used to sequentially read logs. </li><li>1: SEQUENTIAL_READ_PARALLEL, which indicates that the parallel log transfer protocol is used to sequentially read logs. </li><li>2: SCATTERED_READ, which indicates that the last request read discrete logs. </li><li>Other values: unknown RPC type. </li></ul>     |
| LAST_REQUEST_LOG_LSN        | bigint(20) unsigned | NO   | LSN of the last request logs     |
| LAST_REQUEST_LOG_SCN        | bigint(20) unsigned | NO   | SCN of the last request logs     |
| LAST_FAILED_REQUEST         | longtext            | NO   | Information about the last failed RPC, including the sending time, TRACE_ID, and error code.    |
| AVG_REQUEST_PROCESS_TIME    | bigint(20)          | NO   | Average RPC processing time     |
| AVG_REQUEST_QUEUE_TIME      | bigint(20)          | NO   | Average RPC queuing time     |
| AVG_REQUEST_READ_LOG_TIME   | bigint(20)          | NO   | Average RPC log reading time     |
| AVG_REQUEST_READ_LOG_SIZE   | bigint(20)          | NO   | Average RPC log reading size     |
| AVG_LOG_TRANSPORT_BANDWIDTH | varchar(29)         | NO   | Average log transfer bandwidth     |

## Sample query

The following example queries information about downstream log synchronization links (OBCDCs or network standby databases) of all tenants on the current OBServer node.
