# V$OB_GROUP_IO_STAT

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view was introduced in OceanBase Database V4.3.3. </p>
</main>

## Purpose

This view displays the real-time status of each resource group on the current OBServer node of the current tenant.

## Columns

| Column | Type | Nullable? | Description |
| --- | --- | --- | --- |
| TENANT_ID | bigint(20) | NO | The ID of the tenant. |
| SVR_IP | varchar(46) | NO | The IP address of the server. |
| SVR_PORT | bigint(20) | NO | The port of the server. |
| GROUP_ID | bigint(20) | NO | The ID of the resource group. |
| GROUP_NAME | varchar(128) | NO | The name of the resource group. |
| MODE | varchar(128) | NO | The read/write mode. Valid values include:<ul><li>`READ`: read mode</li> <li>`WRITE`: write mode</li></ul>     |
| MIN_IOPS | bigint(20) | NO | The minimum IOPS of the resource group. |
| MAX_IOPS | bigint(20) | NO | The maximum IOPS of the resource group. |
| NORM_IOPS                    | bigint(20)   | NO             | Current IOPS (displayed as the value converted to a 16 KB read equivalent)  <main id="notice" type='explain'><h4>Note</h4><p>For OceanBase Database V4.3.5, this field was introduced starting from V4.3.5 BP1.</p></main>  |
| REAL_IOPS | bigint(20) | NO | The current IOPS of the resource group. |
| MAX_NET_BANDWIDTH | bigint(20) | NO | The maximum network bandwidth of the resource group, in bytes. |
| MAX_NET_BANDWIDTH_DISPLAY | varchar(128) | NO | The maximum network bandwidth of the resource group after the format is converted, in the format of 10 GB/s or 10 MB/s.     |
| REAL_NET_BANDWIDTH | bigint(20) | NO | The current network bandwidth of the resource group, in bytes.    |
| REAL_NET_BANDWIDTH_DISPLAY | varchar(128) | NO | The current network bandwidth of the resource group after the format is converted, in the format of 10 GB/s or 10 MB/s.     |

## Sample query

The following example queries the real-time status of each resource group on the current OBServer node of the current user tenant.

```shell
obclient [oceanbase]> SELECT * FROM oceanbase.V$OB_GROUP_IO_STAT LIMIT 1\G
```

The query result is as follows:

