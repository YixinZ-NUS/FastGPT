# GV$OB_FUNCTION_IO_STAT

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view was introduced in OceanBase Database V4.3.4. </p>
</main>

## Purpose

This view displays the I/O statistics of functions at the tenant level on all OBServer nodes. When the resource group corresponding to a background task at the function level is not enabled for resource management, the displayed I/O information is only for reference and the corresponding resources cannot be limited.

## Columns

|             Column             |      Type    | Nullable? | Description                                            |
|------------------------------|--------------|----------|------------------------------------------------------------|
| SVR_IP                       | varchar(46)  | NO       | The IP address of the server.     |
| SVR_PORT                     | bigint(20)   | NO       | The port number of the server.       |
| TENANT_ID                    | bigint(20)   | NO       | The ID of the tenant.            |
| FUNCTION_NAME                | varchar(32)  | NO       | The name of the background task corresponding to the function.<ul><li>`COMPACTION_HIGH`: The names of mini merge and DDL KV merge tasks. </li> <li>`HA_HIGH`: The names of high-priority high-reliability tasks such as replication, rebuild, and recovery. </li> <li>`COMPACTION_MID`: The name of minor merge tasks. </li> <li>`HA_MID`: The names of medium-priority high-reliability tasks such as migration tasks. </li> <li>`COMPACTION_LOW`: The name of major merge tasks. </li> <li>`HA_LOW`: The names of low-priority high-reliability tasks such as backup and backup cleanup tasks. </li> <li>`DDL`: The names of tasks in scenarios such as unique index verification and column deletion and data supplementation. </li> <li>`DDL_HIGH`: The name of the DDL MemTable dump task. </li> <li>`CLOG_LOW`: The names of physical standby and CDC read tasks. </li> <li>`CLOG_MID`: The names of follower read shared storage, log archiving read local disk, and other tasks. </li> <li>`CLOG_HIGH`: The names of clog write local disk, clog file upload shared storage, and follower read local disk tasks. </li> <li>`GC_MACRO_BLOCK`: The names of macro block recycling and reuse tasks. </li> <li>`OPT_STATS`: The name of the statistics collection task. </li> <li>`IMPORT`: The name of the import task. </li> <li>`SQL_AUDIT`: The name of the SQL audit log write task. </li> <li>`DEFAULT_FUNCTION`: The names of all other background tasks. </li></ul>    |
| MODE                         | varchar(32)  | NO       | The read/write mode of I/O (remote requests for shared storage and local requests for other types of requests):<ul><li>`LOCAL READ`: Local read. </li> <li>`LOCAL WRITE`: Local write. </li> <li>`REMOTE READ`: Remote read. </li> <li>`REMOTE WRITE`: Remote write. </li></ul>    |
| SIZE                         | bigint(20)   | NO       | The average I/O size.     |
| REAL_IOPS                    | bigint(20)   | NO       | The actual IOPS.     |
| REAL_MBPS                    | bigint(20)   | NO       | The actual bandwidth in MB/s.     |
| SCHEDULE_US                  | bigint(20)   | NO       | The average time consumed for I/O scheduling in us.     |
| IO_DELAY_US                  | bigint(20)   | NO       | The average execution time of disk I/O in us.     |
| TOTAL_US                     | bigint(20)   | NO       | The overall average execution time of I/O in us.     |

## Sample query

The following example shows how to query the I/O statistics of functions at the tenant level on all OBServer nodes for the current user tenant.

```shell
obclient [oceanbase]> SELECT * FROM oceanbase.GV$OB_FUNCTION_IO_STAT LIMIT 5;
```
