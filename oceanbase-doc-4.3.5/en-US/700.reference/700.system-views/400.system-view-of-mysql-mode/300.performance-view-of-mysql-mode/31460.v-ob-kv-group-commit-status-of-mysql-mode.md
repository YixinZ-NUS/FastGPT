# V$OB_KV_GROUP_COMMIT_STATUS

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This view is available starting with V4.3.4. </p>
</main>

## Purpose

In a MySQL tenant, the `V$OB_KV_GROUP_COMMIT_STATUS` view provides information about the group commit status of the OBKV client connection on the current node.

## Columns

| **Column** | **Type** | **Nullable** | **Description** |
| --- | --- | --- | --- |
| SVR_IP | varchar(46) | NO | The IP address of the node where the group commit group is located. |
| SVR_PORT | bigint(20) | NO | The port number of the node where the group commit group is located. |
| TENANT_ID | bigint(20) | NO | The tenant ID: <li> 1: the ID of the system tenant <li>Other values: the ID of a user tenant or a meta tenant |
| TABLE_ID | bigint(20) | NO | The ID of the table to which the group commit group belongs. |
| LS_ID | bigint(20) | NO | The ID of the leader server to which the group commit group belongs. |
| SCHEMA_VERSION | bigint(20) | NO | The schema version of the table to which the group commit group belongs. |
| GROUP_TYPE | varchar(32) | NO | The type of the group commit group. Supported types: <ul><li> get: a queue for get requests </li><li>put: a queue for put requests </li><li>del: a queue for delete requests </li><li>replace: a queue for replace requests </li><li>insert: a queue for insert requests </li><li>hybird: a queue for mixed operations </li><li>fail: a queue for batch requests that failed to execute </li></ul>  |
| QUEUE_SIZE | bigint(20) | NO | The size of the request queue in the group commit group, representing the number of unprocessed requests. |
| BATCH_SIZE | bigint(20) | NO | The size of the batch in the group commit group. |
| CREATE_TIME | timestamp(6) | NO  | The time when the group was created in the group commit process.  |
| UPDATE_TIME | timestamp(6) | NO  | The time when the status information of the group was last updated in the group commit process.  |

## Sample query

Query the group commit status information of the OBKV client connection on the current node and display the records from the first day.

```shell
obclient > SELECT * FROM oceanbase.V$OB_KV_GROUP_COMMIT_STATUS limit 1;
```
