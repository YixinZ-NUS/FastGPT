# oceanbase.DBA_OB_TABLEGROUPS

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is introduced since OceanBase Database V4.0.0. </p>
</main>

## Purpose

This view displays metadata about table groups, including partition information.

<main id="notice" type='explain'>
<h4>Note</h4>
<p>In V4.2.0 and later, the partition attributes in this view are no longer meaningful and are uniformly displayed as <code>NONE</code> or <code>NULL</code>. </p>
</main>

## Columns

|           Column           |      Type      | Null? | Description |
|---------------------------|--------------|------|--------------------------------------------------------------------------------------------------------|
| TABLEGROUP_NAME           | varchar(128) | NO   | The name of the table group.          |
| PARTITIONING_TYPE         | varchar(13)  | NO   | The partitioning type of the table group. Valid values include:<li> `NONE`: a non-partitioned table group.   <li> `HASH`: a HASH-partitioned table group (with a single partitioning key).   <li> `KEY`: a KEY-partitioned table group.   <li> `RANGE`: a RANGE-partitioned table group (with a single partitioning key).   <li> `RANGE COLUMNS`: a RANGE COLUMNS-partitioned table group.   <li> `LIST`: a LIST-partitioned table group (with a single partitioning key).   <li> `LIST COLUMNS`: a LIST COLUMNS-partitioned table group.   <li> `UNKNOWN`: an abnormal partitioning type.<main id="notice" type='explain'><h4>Note</h4><p>This field is <code>NULL</code> in V4.2.0 and later. </p></main>            |
| SUBPARTITIONING_TYPE      | varchar(13)  | NO   | The partitioning type of the table group. Valid values include:<li> `NONE`: a non-partitioned table group or a partitioned table group.   <li> `HASH`: a HASH-partitioned table group (with a single partitioning key).   <li> `KEY`: a KEY-partitioned table group.   <li> `RANGE`: a RANGE-partitioned table group (with a single partitioning key).   <li> `RANGE COLUMNS`: a RANGE COLUMNS-partitioned table group.   <li> `LIST`: a LIST-partitioned table group (with a single partitioning key).   <li> `LIST COLUMNS`: a LIST COLUMNS-partitioned table group.   <li> `UNKNOWN`: an abnormal partitioning type.<main id="notice" type='explain'><h4>Note</h4><p>This field is <code>NULL</code> in V4.2.0 and later. </p></main>     |
| PARTITION_COUNT           | bigint(20)   | NO   | The value is `NULL`.            |
| DEF_SUBPARTITION_COUNT    | bigint(0)    | NO   | The value is `NULL`.|
| PARTITIONING_KEY_COUNT    | bigint(20)   | NO   | The value is `NULL`.         |
| SUBPARTITIONING_KEY_COUNT | bigint(20)   | NO   | The value is `NULL`.       |
| SHARDING                  | varchar(10)  | NO   | The SHARDING attribute of the table group. Valid values include:<ul><li>`NONE`: For a table group with this attribute, all partitions of all tables in the table group are located on the same server, and no restrictions are imposed on tables to be added to the table group. </li> <li>`PARTITION`: For a table group with this attribute, data of each table in the table group is distributed by partitioning. If a table is a subpartitioned table, all subpartitions of the table are aggregated together. </li> <li>`ADAPTIVE`: For a table group with this attribute, data of each table in the table group is distributed by using the adaptive method. That is, if a table in the table group is a partitioned table, data is distributed by partitioning; if a table in the table group is a subpartitioned table, data is distributed by subpartition. </li></ul>  <main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |

## Sample query

1. Create a table group named `tblgroup1`.

   ```shell
   obclient [test]> CREATE TABLEGROUP tblgroup1 SHARDING = 'PARTITION';
   ```

2. View the SHARDING attribute of the `tblgroup1` table group.

   ```shell
   obclient [test]> SELECT TABLEGROUP_NAME, SHARDING FROM oceanbase.DBA_OB_TABLEGROUPS WHERE tablegroup_name = 'tblgroup1';
   ```

   The query result is as follows:

   ```shell
   +-----------------+-----------+
   | TABLEGROUP_NAME | SHARDING  |
   +-----------------+-----------+
   | tblgroup1       | PARTITION |
   +-----------------+-----------+
   1 row in set
   ```
