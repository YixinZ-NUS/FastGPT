# information_schema.role_table_grants

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view was introduced in V4.3.3 for V4.3x and V4.2.4 for V4.2.x. </p>
</main>

## Purpose

The `role_table_grants` view displays the table-level privileges granted to activated roles of the user in the current session and the table-level privileges granted to other roles of the activated roles.

## Columns

| **Column** | **Type**     | **Nullable?** | **Description**        |
|----------------|--------------|----------------------|-----------------|
| GRANTOR        | varchar(97)  |  YES    | The username of the account that granted the role.      |
| GRANTOR_HOST   | varchar(256) |  YES    | The hostname of the account that granted the role.      |
| GRANTEE        | varchar(32)  |  NO     | The username of the account corresponding to the role.      |
| GRANTEE_HOST   | varchar(255) |  NO     | The hostname of the account corresponding to the role.      |
| TABLE_CATALOG  | varchar(3)   |  NO     | The name of the catalog to which the role belongs. The value is always def.      |
| TABLE_SCHEMA   | varchar(64)  |  NO     | The name of the database to which the table belongs.      |
| TABLE_NAME     | varchar(64)  |  NO     | The name of the table.      |
| PRIVILEGE_TYPE | varchar(90)  |  NO     | The privileges granted. This value can be any privilege granted at the table level.  |
| IS_GRANTABLE   | varchar(3)   |  NO     | Whether the privileges granted to the role can be granted to other users.     |

## Sample query

1. Create a role and a table.

   ```shell
   obclient [test]> CREATE ROLE employee;
   ```

   ```shell
   obclient [test]> CREATE TABLE tbl1(c1 int, c2 int, c3 int);
   ```

2. Grant table-level privileges to the role.

   ```shell
   obclient [test]> GRANT DROP,UPDATE,SELECT ON test.tbl1 TO employee WITH GRANT OPTION;
   ```

3. Grant the role to a user.

   ```shell
   obclient [test]> GRANT employee TO user1;
   ```

4. Connect to the database as the `user1` user.

5. Activate the `employee` role in the current session.

   ```shell
   obclient [test]> SET ROLE employee;
   ```

6. View the table-level privileges granted to activated roles of the user in the current session.

   ```shell
   obclient [test]> USE information_schema;
   ```

   ```shell
   obclient [test]> SELECT * FROM information_schema.role_table_grants;
   ```

   The query result is as follows:

   ```shell
   +---------+--------------+-----------+--------------+---------------+--------------+------------+--------------------+--------------+
   | GRANTOR | GRANTOR_HOST | GRANTEE   | GRANTEE_HOST | TABLE_CATALOG | TABLE_SCHEMA | TABLE_NAME | PRIVILEGE_TYPE     | IS_GRANTABLE |
   +---------+--------------+-----------+--------------+---------------+--------------+------------+--------------------+--------------+
   | root    | %            | employee  | %            | def           | test         | tbl1       | Drop,Update,Select | YES          |
   +---------+--------------+-----------+--------------+---------------+--------------+------------+--------------------+--------------+
   1 row in set
   ```
