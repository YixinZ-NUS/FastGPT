# oceanbase.CDB_OB_TABLE_LOCATIONS

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This view is introduced since OceanBase Database V4.0.0. </p>
</main>

## Purpose

The `CDB_OB_TABLE_LOCATIONS` view displays the locations of tables or partitions, including system tables, user tables, and index tables.

## Columns

| Column | Type | Nullable? | Description |
| --- | --- | --- | --- |
| TENANT_ID | bigint(20) | NO | Tenant ID |
| DATABASE_NAME | varchar(128) | NO | Database name |
| TABLE_NAME | varchar(256) | NO | Table name |
| TABLE_ID | bigint(20) | NO | Table ID |
| TABLE_TYPE | varchar(13) | NO | Table type. Valid values include:<ul><li>USER TABLE: user table </li><li>SYSTEM TABLE: system table, a table created by the system </li><li>INDEX: index table </li><li>LOB AUX TABLE: LOB auxiliary table, a table created by the system to support LOB columns</li></ul> |
| PARTITION_NAME | varchar(64) | NO | Name of the primary partition. For a non-partitioned table, this value is NULL. |
| SUBPARTITION_NAME | varchar(64) | NO | Name of the subpartition. If there is no subpartition, this value is NULL. |
| INDEX_NAME | varchar(256) | NO | If the table is an index table, the name of the index table is displayed. Otherwise, NULL is displayed. |
| DATA_TABLE_ID | bigint(20) | NO | If the table is an index table or a LOB auxiliary table, the ID of the corresponding primary table is displayed. Otherwise, NULL is displayed. |
| TABLET_ID | bigint(20) | NO | Tablet ID |
| LS_ID | bigint(20) | NO | Log stream ID of the object |
| ZONE | varchar(128) | NO | Zone where the replica is located |
| SVR_IP | varchar(46) | NO | IP address of the replica |
| SVR_PORT | bigint(20) | NO | Port of the replica |
| ROLE | varchar(8) | NO | Role of the replica |
| REPLICA_TYPE | varchar(18) | NO | Log stream replica type. Valid values include:<ul><li>`FULL`: full-featured replica</li> <li>READONLY: read-only replica</li> <li>`COLUMNSTORE`: columnstore replica</li></ul>  |
| DUPLICATE_SCOPE | varchar(7) | NO | Indicates whether the table is a replicated table. Valid values include:<ul><li>`CLUSTER`: replicated table  </li><li>`NONE`: ordinary table </li></ul> <main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
| DUPLICATE_READ_CONSISTENCY | varchar(6) | NO | Specifies the read consistency constraint for the standby replica of a replicated table. Valid values include:<ul><li>`NONE`: indicates that the table is not a replicated table. This attribute is meaningless. </li><li>`STRONG`: indicates that the standby replica of a replicated table supports strong consistency reads. </li></ul> <main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.3.4. </p></main>|
| OBJECT_ID | bigint(20) | NO | ID of the partition<main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
| TABLEGROUP_NAME | varchar(128) | NO | Name of the table group to which the table belongs<main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
| TABLEGROUP_ID | bigint(20) | NO | ID of the table group to which the table belongs<main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
| SHARDING | varchar(10) | NO | Sharding attribute (equilibrium mode) of the table group to which the table belongs<main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |

## Sample query

```sql
SELECT * FROM oceanbase.CDB_OB_TABLE_LOCATIONS WHERE TENANT_ID = 1004 AND TABLE_NAME='test1_tbl1'\G
```
