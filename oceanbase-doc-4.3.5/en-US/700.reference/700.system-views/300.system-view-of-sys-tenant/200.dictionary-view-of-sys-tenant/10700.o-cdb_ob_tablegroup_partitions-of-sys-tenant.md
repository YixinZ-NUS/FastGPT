# oceanbase.CDB_OB_TABLEGROUP_PARTITIONS

<main id="notice" type='explain'>
<h4>Note</h4>
<ul>
<li>Since OceanBase Database V4.2.0, table groups no longer have the concept of partitions. This view is no longer used. You can query this view, but the query result is empty. </li>
<li>For OceanBase Database V4.2.0 and later, you can query information about table groups by using the view <a href="11000.o-cdb_ob_tablegroups-of-sys-tenant.md">CDB_OB_TABLEGROUPS</a>. For more information about table groups, see <a href="../../../300.database-object-management/100.manage-object-of-mysql-mode/400.manage-table-groups-of-mysql-mode/100.about-table-groups-of-mysql-mode.md">Table groups</a>. </li>
</ul>
</main>

## Purpose

This view displays the information about the primary partitions of partitioned table groups in all tenants of the system tenant.

## Columns

|        Column        |      Type       | Nullable? | Description |
|--------------------|---------------|------------|---------------------------|
| TENANT_ID          | bigint(20)    | NO         | Tenant ID                                                    |
| TABLEGROUP_NAME    | varchar(128)  | NO         | Name of the table group                                            |
| COMPOSITE          | varchar(3)    | NO         | Whether the table group is a subpartitioned table group. Valid values: `YES` and `NO`                             |
| PARTITION_NAME     | varchar(64)   | NO         | Name of the primary partition of the table group                                              |
| SUBPARTITION_COUNT | bigint(20)    | NO         | This value is `NULL`.                  |
| HIGH_VALUE         | varchar(4096) | NO         | This value is `NULL`.            |
| HIGH_VALUE_LENGTH  | bigint(20)    | NO         | This value is `NULL`.                             |
| PARTITION_POSITION | bigint(20) unsigned | NO   | This value is `NULL`.    |
