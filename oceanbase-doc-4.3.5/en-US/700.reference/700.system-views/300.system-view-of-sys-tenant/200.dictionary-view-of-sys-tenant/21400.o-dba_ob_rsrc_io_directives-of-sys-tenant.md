# oceanbase.DBA_OB_RSRC_IO_DIRECTIVES

<main id="notice" type='explain'>
<h4>Note</h4>
<ul><li>This view was introduced in V4.1.0. </li>
<li>In the current version, we recommend that you use the <a href="21420.o-dba_ob_rsrc_directives-of-sys-tenant.md">DBA_OB_RSRC_DIRECTIVES</a> view to view the resource configuration parameters of a tenant in Resource Manager. </li></ul>
</main>

## Purpose

This view displays the IO resource configurations in Resource Manager.

## Columns

| **Column** | **Type** | **Nullable?** | **Description** |
| --- | --- | --- | --- |
| PLAN | varchar(128) | NO | The name of the resource plan to which the directive belongs. |
| GROUP_OR_SUBPLAN | varchar(128) | NO | The name of the resource group or subplan controlled by the directive. |
| COMMENTS    | varchar(2000) | YES | The remarks about the resource plan. |
| MIN_IOPS    | bigint(20) | NO | The IOPS resources reserved for the resource group when IO contention occurs. The sum of all `MIN_IOPS` values specified for resource groups cannot exceed 100. The default value is `0`.     |
| MAX_IOPS    | bigint(20) | NO | The maximum IOPS resources available for the resource group. The sum of all `MAX_IOPS` values specified for resource groups can exceed 100. The default value is `100`.     |
| WEIGHT_IOPS | bigint(20) | NO | The weight value of IOPS when IO contention occurs. The sum of all `WEIGHT_IOPS` values specified for resource groups can exceed 100. IO resources are allocated based on the normalized weight values, provided that the `MIN_IOPS` and `MAX_IOPS` values are set. The default value is `0`.     |
