# oceanbase.DBA_OB_STORAGE_IO_USAGE

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view was introduced in OceanBase Database V4.3.4. </p>
</main>

## Purpose

This view displays the I/O operations of the object storage and cloud disks of the current tenant.

## Columns

| Column |  Type  | Nullable? | Description |
|-----------|---------------|----------------|-------------------------------|
| TENANT_ID | bigint(20)    | NO   | Tenant ID     |
| PATH      | varchar(1024) | NO   | The path is in the format of `oss://`, `file://`, or `s3://`.    |
| ENDPOINT  | varchar(256)  | NO   | <ul><li>In shared-nothing (SN) mode, the endpoint of object storage used for backup and restore, data import, and data export. </li> <li>In shared-storage (SS) mode, the endpoint of object storage used for log (clog) or data storage, as well as backup and restore, data import, and data export. </li></ul> |
| TYPE      | varchar(128)  | NO   | The operation type. Valid values include:<ul><li>`READ`: read operations</li> <li>`WRITE`: write operations</li> <li>`READ_BYTES`: the total amount of traffic to the object storage network, in bytes</li> <li>`WRITE_BYTES`: the total amount of traffic from the object storage network, in bytes</li> <li>`TAG`: the total number of tags in object storage. This column is not displayed for local disks and cloud disks. </li></ul>     |
| TOTAL     | bigint(20)    | NO   | The cumulative value of the corresponding operation type (`TYPE`):<ul><li>`READ`: the number of read operations</li> <li>`WRITE`: the number of write operations</li> <li>`READ_BYTES`: the total amount of traffic to the object storage network, in bytes</li> <li>`WRITE_BYTES`: the total amount of traffic from the object storage network, in bytes</li> <li>`TAG`: the total number of tags in object storage. This column is not displayed for local disks and cloud disks. </li></ul>      |
