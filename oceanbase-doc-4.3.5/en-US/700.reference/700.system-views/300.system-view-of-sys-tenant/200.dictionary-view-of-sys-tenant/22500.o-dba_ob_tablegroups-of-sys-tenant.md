# oceanbase.DBA_OB_TABLEGROUPS

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is introduced since OceanBase Database V4.0.0. </p>
</main>

## Purpose

This view displays metadata about table groups, including partition information.

<main id="notice" type='explain'>
<h4>Note</h4>
<p>In V4.2.0 and later, the partition attributes in this view are not meaningful and are uniformly displayed as <code>NONE</code> or <code>NULL</code>. </p>
</main>

## Columns

|           Column           |      Type      | Nullable? | Description |
|---------------------------|--------------|------------|--------------------------------------------------------------------------------------------------------|
| TABLEGROUP_NAME           | varchar(128) | NO         | The name of the table group.          |
| PARTITIONING_TYPE         | varchar(13)  | NO         | The partitioning type of the table group. Valid values include:<li> `NONE`: a non-partitioned table group.   <li> `HASH`: a table group that uses hash partitioning (single partitioning key).   <li> `KEY`: a table group that uses key partitioning.   <li> `RANGE`: a table group that uses range partitioning (single partitioning key).   <li> `RANGE COLUMNS`: a table group that uses range columns partitioning.   <li> `LIST`: a table group that uses list partitioning (single partitioning key).   <li> `LIST COLUMNS`: a table group that uses list columns partitioning.   <li> `UNKNOWN`: an abnormal type. <main id="notice" type='explain'><h4>Note</h4><p>The value of this field is <code>NULL</code> in V4.2.0 and later. </p></main>         |
| SUBPARTITIONING_TYPE      | varchar(13)  | NO         | The partitioning type of the subpartitions in the table group. Valid values include:<li> `NONE`: a non-partitioned table group or a table group that uses single-level partitioning.   <li> `HASH`: a table group that uses hash partitioning (single partitioning key).   <li> `KEY`: a table group that uses key partitioning.   <li> `RANGE`: a table group that uses range partitioning (single partitioning key).   <li> `RANGE COLUMNS`: a table group that uses range columns partitioning.   <li> `LIST`: a table group that uses list partitioning (single partitioning key).   <li> `LIST COLUMNS`: a table group that uses list columns partitioning.   <li> `UNKNOWN`: an abnormal type. <main id="notice" type='explain'><h4>Note</h4><p>The value of this field is <code>NULL</code> in V4.2.0 and later. </p></main>    |
| PARTITION_COUNT           | bigint(20)   | NO         | The value is `NULL`.            |
| DEF_SUBPARTITION_COUNT    | bigint(0)    | NO         | The value is `NULL`.|
| PARTITIONING_KEY_COUNT    | bigint(20)   | NO         | The value is `NULL`.         |
| SUBPARTITIONING_KEY_COUNT | bigint(20)   | NO         | The value is `NULL`.       |
| SHARDING                  | varchar(10)  | NO         | The SHARDING attribute of the table group. Valid values include:<ul><li>`NONE`: for a table group with this attribute, all partitions of all tables in the table group are located on the same server, and no restrictions are imposed on tables that are added to the table group. </li> <li>`PARTITION`: for a table group with this attribute, data of each table in the table group is distributed to different servers by using single-level partitioning. If the table is a subpartitioned table, all subpartitions of the same single-level partition are located on the same server. </li> <li>`ADAPTIVE`: for a table group with this attribute, data of each table in the table group is distributed to different servers by using adaptive partitioning. That is, if the tables in the table group are single-level partitioned tables, data is distributed by using single-level partitioning; if the tables in the table group are subpartitioned tables, data is distributed by using subpartitioning for each single-level partition. </li></ul>  <main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |

## Sample query

1. Create a table group named `tblgroup1`.

   ```shell
   obclient [test]> CREATE TABLEGROUP tblgroup1 SHARDING = 'PARTITION';
   ```

2. View the SHARDING attribute of the `tblgroup1` table group.

   ```shell
   obclient [test]> SELECT TABLEGROUP_NAME, SHARDING FROM oceanbase.DBA_OB_TABLEGROUPS WHERE tablegroup_name = 'tblgroup1';
   ```

   The query result is as follows:

   ```shell
   +-----------------+-----------+
   | TABLEGROUP_NAME | SHARDING  |
   +-----------------+-----------+
   | tblgroup1       | PARTITION |
   +-----------------+-----------+
   1 row in set
   ```
