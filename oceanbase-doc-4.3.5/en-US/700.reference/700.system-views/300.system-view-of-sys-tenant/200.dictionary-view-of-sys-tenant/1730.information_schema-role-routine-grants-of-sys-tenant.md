# information_schema.role_routine_grants

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view was introduced in V4.3.3 for the V4.3.x series and in V4.2.4 for the V4.2.x series. </p>
</main>

## Purpose

The `role_routine_grants` view displays the routine privileges granted to activated roles and to other roles granted to activated roles in the current session.

## Columns

| **Column** | **Type** | **Nullable?** | **Description** |
|----------------|--------------|----------------------|-----------------|
| GRANTOR        | varchar(97)  | YES  | The username of the account that granted the role.    |
| GRANTOR_HOST   | varchar(256) | YES  | The host of the account that granted the role.      |
| GRANTEE        | varchar(32)  | NO   | The username of the account corresponding to the role.      |
| GRANTEE_HOST   | varchar(255) | NO   | The hostname of the account corresponding to the role.    |
| SPECIFIC_CATALOG | varchar(3) | NO   | The name of the directory where the granted routine is located. The value is always def.   |
| SPECIFIC_SCHEMA  | varchar(64)| NO   | The name of the database where the granted routine is located.   |
| SPECIFIC_NAME    | varchar(64)| NO   | The name of the granted routine.         |
| ROUTINE_CATALOG  | varchar(3) | NO   | The name of the directory where the granted routine is located. The value is always def.         |
| ROUTINE_SCHEMA   | varchar(64)| NO   | The name of the database where the granted routine is located.          |
| ROUTINE_NAME     | varchar(64)| NO   | The name of the granted routine.         |
| PRIVILEGE_TYPE   | varchar(90)| NO   | The granted privilege. This value can be any privilege granted at the routine level.          |
| IS_GRANTABLE     | varchar(3) | NO   | Whether the privilege granted to the role can be granted to other users.         |

## Sample query

1. Create a role and a function.

   Create a role named `employee`:

   ```shell
   obclient [test]> CREATE ROLE employee;
   ```

   Create a function named `add_numbers`:

   ```shell
   obclient [test]> DELIMITER //
   ```

   ```shell
   obclient [test]> CREATE FUNCTION test.add_numbers(a INT, b INT)
   RETURNS INT
   BEGIN
         RETURN a + b;
   END;//
   ```

   ```shell
   obclient [test]> DELIMITER ;
   ```

2. Grant routine privileges to the role.

   ```shell
   obclient [test]> GRANT EXECUTE,ALTER ROUTINE ON test.add_numbers TO employee;
   ```

3. Grant the role to a user.

   ```shell
   obclient [test]> GRANT employee TO user1;
   ```

4. Connect to the database as the user `user1`.

5. Activate the `employee` role in the current session.

   ```shell
   obclient [test]> SET ROLE employee;
   ```

6. View the routine privileges granted to activated roles and to other roles granted to activated roles in the current session.

   ```shell
   obclient [test]> USE information_schema;
   ```

   ```shell
   obclient [information_schema]> SELECT * FROM information_schema.role_routine_grants;
   ```

   The query result is as follows:

   ```shell
   +---------+--------------+-----------+--------------+------------------+-----------------+---------------+-----------------+----------------+--------------+-----------------------+--------------+
   | GRANTOR | GRANTOR_HOST | GRANTEE   | GRANTEE_HOST | SPECIFIC_CATALOG | SPECIFIC_SCHEMA | SPECIFIC_NAME | ROUTINE_CATALOG | ROUTINE_SCHEMA | ROUTINE_NAME | PRIVILEGE_TYPE        | IS_GRANTABLE |
   +---------+--------------+-----------+--------------+------------------+-----------------+---------------+-----------------+----------------+--------------+-----------------------+--------------+
   | root    | %            | employee  | %            | def              | test            | add_numbers   | def             | test           | add_numbers  | Execute,Alter Routine | NO           |
   +---------+--------------+-----------+--------------+------------------+-----------------+---------------+-----------------+----------------+--------------+-----------------------+--------------+
   1 row in set
   ```
