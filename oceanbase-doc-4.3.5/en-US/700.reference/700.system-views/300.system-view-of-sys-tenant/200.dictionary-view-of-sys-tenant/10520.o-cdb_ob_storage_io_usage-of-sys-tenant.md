# oceanbase.CDB_OB_STORAGE_IO_USAGE

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view was introduced in OceanBase Database V4.3.4. </p>
</main>

## Purpose

This view displays the I/O operations of all tenant objects in object storage and cloud disks.

## Columns

| Column | Type | Nullable? | Description |
| --- | --- | --- | --- |
| TENANT_ID | bigint(20) | NO | Tenant ID |
| PATH | varchar(1024) | NO | The path is in the format of `oss://`, `file://`, or `s3://`. |
| ENDPOINT | varchar(256) | NO | <ul><li>In shared-nothing (SN) mode, displays the endpoint of object storage used for backup and restore, data import, and data export. </li> <li>In shared-storage (SS) mode, displays the endpoint of object storage used for log (clog) or data (Data) storage and backup and restore, data import, and data export. </li></ul> |
| TYPE | varchar(128) | NO | Operation type:<ul><li>`READ`: read operation</li> <li>`WRITE`: write operation</li> <li>`READ_BYTES`: total amount of traffic entering the object storage network, in bytes</li> <li>`WRITE_BYTES`: total amount of traffic exiting the object storage network, in bytes</li> <li>`TAG`: tag type in object storage. This type is not displayed for local disks and cloud disks. </li></ul>     |
| TOTAL | bigint(20) | NO | The cumulative value of each type (`TYPE`):<ul><li>`READ`: the number of read operation-related operations</li> <li>`WRITE`: the number of write operation-related operations</li> <li>`READ_BYTES`: the total amount of traffic entering the object storage network, in bytes</li> <li>`WRITE_BYTES`: the total amount of traffic exiting the object storage network, in bytes</li> <li>`TAG`: the cumulative value of tag types in object storage. This statistic is not displayed for local disks and cloud disks. </li></ul>      |
