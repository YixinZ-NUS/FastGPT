# oceanbase.DBA_OB_TABLE_LOCATIONS

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is introduced since OceanBase Database V4.0.0. </p>
</main>

## Purpose

This view displays the locations of tables or partitions, including system tables, user tables, and index tables.

## Columns

| Column | Type | Nullable? | Description |
| --- | --- | --- | --- |
| DATABASE_NAME | varchar(128) | NO | Database name. |
| TABLE_NAME | varchar(256) | NO | Table name. |
| TABLE_ID | bigint(20) | NO | Table ID. |
| TABLE_TYPE | varchar(13) | NO | Table type. Valid values include:<ul><li>USER TABLE: user table </li><li>SYSTEM TABLE: system table, a table created by the system </li><li>INDEX: index table </li><li>LOB AUX TABLE: LOB auxiliary table, a table created by the system, used to implement LOB columns</li></ul> |
| PARTITION_NAME | varchar(64) | NO | Name of the primary partition. For a non-partitioned table, this field is NULL. |
| SUBPARTITION_NAME | varchar(64) | NO | Name of the subpartition. If there is no subpartition, this field is NULL. |
| INDEX_NAME | varchar(256) | NO | Name of the index table if the table is an index table. Otherwise, this field is NULL. |
| DATA_TABLE_ID | bigint(20) | NO | ID of the primary table if the table is an index table or a LOB auxiliary table. Otherwise, this field is NULL. |
| TABLET_ID | bigint(20) | NO | Tablet ID. |
| LS_ID | bigint(20) | NO | ID of the log stream where the object is located. |
| ZONE | varchar(128) | NO | Zone where the replica is located. |
| SVR_IP | varchar(46) | NO | IP address of the replica. |
| SVR_PORT | bigint(20) | NO | Port of the replica. |
| ROLE | varchar(8) | NO | Role of the replica. |
| REPLICA_TYPE | varchar(18) | NO | Log stream replica type. Valid values include:<ul><li>`FULL`: full-function replica</li> <li>READONLY: read-only replica</li> <li>`COLUMNSTORE`: columnstore replica</li></ul>  |
| DUPLICATE_SCOPE | varchar(7) | NO | Indicates whether the table is a replica table. Valid values include:<ul><li>`CLUSTER`: replica table  </li><li>`NONE`: ordinary table </li></ul> <main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
| DUPLICATE_READ_CONSISTENCY | varchar(6) | NO | Read consistency constraint of the replica of a replica table. Valid values include:<ul><li>`NONE`: indicates that the table is not a replica table. This attribute is meaningless. </li><li>`STRONG`: indicates that the replica of a replica table supports strong consistency read. </li></ul> <main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.3.4. </p></main>|
| OBJECT_ID | bigint(20) | NO | ID of the partition<main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
| TABLEGROUP_NAME | varchar(128) | NO | Name of the table group to which the table belongs<main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
| TABLEGROUP_ID | bigint(20) | NO | ID of the table group to which the table belongs<main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
| SHARDING | varchar(10) | NO | Sharding attribute (balanced mode) of the table group to which the table belongs<main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
