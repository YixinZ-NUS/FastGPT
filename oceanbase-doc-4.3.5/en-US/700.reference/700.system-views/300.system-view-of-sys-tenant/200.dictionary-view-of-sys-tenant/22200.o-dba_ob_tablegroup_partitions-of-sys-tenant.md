# oceanbase.DBA_OB_TABLEGROUP_PARTITIONS

<main id="notice" type='explain'>
<h4>Note</h4>
<ul>
<li>Starting from V4.2.0, table groups no longer have partitions. This view is no longer used. You can query this view, but the query result is empty. </li>
<li>For V4.2.0 and later, you can query table group information by using the <a href="22500.o-dba_ob_tablegroups-of-sys-tenant.md">DBA_OB_TABLEGROUPS</a> view. For more information about table groups, see <a href="../../../300.database-object-management/100.manage-object-of-mysql-mode/400.manage-table-groups-of-mysql-mode/100.about-table-groups-of-mysql-mode.md">About table groups</a>. </li>
</ul>
</main>

## Purpose

This view displays the information about primary partitions of table groups with partitions.

## Columns

|        Column        |      Type       | Nullable? | Description |
|--------------------|---------------|------------|---------------------------|
| TABLEGROUP_NAME    | varchar(128)  | NO         | Name of the table group                                            |
| COMPOSITE          | varchar(3)    | NO         | Whether the table group is a subpartition of a composite partition. Valid values: `YES` and `NO`                             |
| PARTITION_NAME     | varchar(64)   | NO         | Name of the primary partition of the table group                                              |
| SUBPARTITION_COUNT | bigint(20)    | NO         | This value is `NULL`.                  |
| HIGH_VALUE         | varchar(4096) | NO         | This value is `NULL`.            |
| HIGH_VALUE_LENGTH  | bigint(20)    | NO         | This value is `NULL`.                             |
| PARTITION_POSITION | bigint(20) unsigned | NO   | This value is `NULL`.    |
