# oceanbase.CDB_OB_TABLEGROUP_SUBPARTITIONS

<main id="notice" type='explain'>
<h4>Note</h4>
<ul>
<li>In V4.2.0 and later, table groups no longer support partitions. Therefore, this view is no longer used. You can query this view, but the query result is empty. </li>
<li>In V4.2.0 and later, you can query information about table groups by using the <a href="11000.o-cdb_ob_tablegroups-of-sys-tenant.md">CDB_OB_TABLEGROUPS</a> view. For more information about table groups, see <a href="../../../300.database-object-management/100.manage-object-of-mysql-mode/400.manage-table-groups-of-mysql-mode/100.about-table-groups-of-mysql-mode.md">About table groups</a>. </li>
</ul>
</main>

## Purpose

This view displays the information about subpartitions of partitioned table groups in all tenants in the system tenant.

## Columns

|         Column          |      Type       | Nullable? | Description |
|-----------------------|---------------|------------|----------------------------------------------------------------------|
| TENANT_ID             | bigint(20)    | NO         | Tenant ID            |
| TABLEGROUP_NAME       | varchar(128)  | NO         | Name of the table group    |
| PARTITION_NAME        | varchar(64)   | NO         | Name of the primary partition corresponding to the subpartition of the table group  |
| SUBPARTITION_NAME     | varchar(64)   | NO         | Name of the subpartition of the table group         |
| HIGH_VALUE            | varchar(4096) | NO         | This value is `NULL`.                                                       |
| HIGH_VALUE_LENGTH     | bigint(20)    | NO         | This value is `NULL`.                                                                           |
| PARTITION_POSITION    | bigint(20)    | NO         | This value is `NULL`.     |
| SUBPARTITION_POSITION | bigint(20)    | NO         | This value is `NULL`.            |
