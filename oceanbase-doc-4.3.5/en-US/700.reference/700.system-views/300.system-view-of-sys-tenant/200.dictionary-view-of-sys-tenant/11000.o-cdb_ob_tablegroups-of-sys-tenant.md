# oceanbase.CDB_OB_TABLEGROUPS

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is introduced since OceanBase Database V4.0.0. </p>
</main>

## Purpose

This view displays metadata about all table groups in all tenants in the system tenant, including the partitioning information of table groups.

<main id="notice" type='explain'>
<h4>Note</h4>
<p>Starting from V4.2.0, the values of the partitioning attributes in this view are displayed as <code>NONE</code> or <code>NULL</code>. </p>
</main>

## Columns

| Column |  Type  | Nullable? | Description |
|-----------|----------|---------|-------------------------------------|
| TENANT_ID | bigint(20) | NO | Tenant ID. |
| TABLEGROUP_NAME | varchar(128) | NO | Name of the table group. |
| PARTITIONING_TYPE | varchar(13) | NO | Partitioning type of the table group. Valid values include:<li>`NONE`: non-partitioned table group   <li>`HASH`: HASH partitioning (single partitioning key)   <li>`KEY`: KEY partitioning   <li>`RANGE`: RANGE partitioning (single partitioning key)   <li>`RANGE COLUMNS`: RANGE COLUMNS partitioning   <li>`LIST`: LIST partitioning (single partitioning key)   <li>`LIST COLUMNS`: LIST COLUMNS partitioning   <li>`UNKNOWN`: abnormal type<main id="notice" type='explain'><h4>Note</h4><p>This field has a value of <code>NULL</code> starting from V4.2.0. </p></main>            |
| SUBPARTITIONING_TYPE | varchar(13) | NO | Partitioning type of the table group. Valid values include:<li>`NONE`: non-partitioned table group or partitioned table group   <li>`HASH`: HASH partitioning (single partitioning key)   <li>`KEY`: KEY partitioning   <li>`RANGE`: RANGE partitioning (single partitioning key)   <li>`RANGE COLUMNS`: RANGE COLUMNS partitioning   <li>`LIST`: LIST partitioning (single partitioning key)   <li>`LIST COLUMNS`: LIST COLUMNS partitioning   <li>`UNKNOWN`: abnormal type<main id="notice" type='explain'><h4>Note</h4><p>This field has a value of <code>NULL</code> starting from V4.2.0. </p></main>     |
| PARTITION_COUNT | bigint(20) | NO | The value is `NULL`.            |
| DEF_SUBPARTITION_COUNT | bigint(0) | NO | The value is `NULL`.            |
| PARTITIONING_KEY_COUNT | bigint(20) | NO | The value is `NULL`.            |
| SUBPARTITIONING_KEY_COUNT | bigint(20) | NO | The value is `NULL`.            |
| SHARDING | varchar(10)  | NO | SHARDING attribute of the table group. Valid values include:<ul><li>`NONE`: For table groups with this attribute, all partitions of all tables in the table group are located on the same server, and there are no restrictions on tables added to the table group. </li> <li>`PARTITION`: For table groups with this attribute, data of each table in the table group is distributed across partitions. If the table is a subpartitioned table, all subpartitions under each partition are aggregated. </li> <li>`ADAPTIVE`: For table groups with this attribute, data of each table in the table group is distributed based on the adaptive method. That is, if the tables in the table group are partitioned tables, data is distributed based on partitions; if the tables in the table group are subpartitioned tables, data is distributed based on subpartitions under partitions. </li></ul>  <main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |

## Sample query

1. Create a table group named `tblgroup1`.

   ```shell
   obclient [test]> CREATE TABLEGROUP tblgroup1 SHARDING = 'PARTITION';
   ```

2. Query the SHARDING attribute of the `tblgroup1` table group.

   ```shell
   obclient [test]> SELECT TABLEGROUP_NAME, SHARDING FROM oceanbase.CDB_OB_TABLEGROUPS WHERE tablegroup_name = 'tblgroup1';
   ```

   The query result is as follows:

   ```shell
   +-----------------+-----------+
   | TABLEGROUP_NAME | SHARDING  |
   +-----------------+-----------+
   | tblgroup1       | PARTITION |
   +-----------------+-----------+
   1 row in set
   ```
