# GV$OB_TABLET_COMPACTION_PROGRESS

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is introduced in V4.0.0. </p>
</main>

## Purpose

This view displays the progress of tablet-level compactions across all tenants and nodes.

## Columns

| Column | Type | Nullable? | Description |
| --- | --- | --- | --- |
| SVR_IP                       | varchar(46)  | NO         | The IP address of the server.                                                                                |
| SVR_PORT                     | bigint(20)   | NO         | The port number of the server. |
| TENANT_ID                    | bigint(20)   | NO         | The tenant ID. |
| TYPE                         | varchar(64)  | NO         | The type of compaction. Valid values: <ul><li>`MDS_TABLE_MERGE`: persists the system metadata in the format of a SSTable to the disk. </li> <li>`MAJOR_MERGE`: tenant-level major compaction. </li> <li>`MEDIUM_MERGE`: partition-level major compaction. </li> <li>`MINI_MERGE`: mini compaction. It converts the MemTables into mini SSTables. </li> <li>`MINOR_MERGE`: minor compaction. It merges multiple mini SSTables into a new mini SSTable, or multiple mini SSTables and a minor SSTable into a new minor SSTable. </li> <li>`META_MAJOR_MERGE`: a special type of compaction that synthesizes data before a specified point in time into a meta major SSTable. The data format of this SSTable is the same as that of a major SSTable, but it does not contain multi-version data or uncommitted transaction data. </li></ul>    |
| LS_ID                        | bigint(20)   | NO         | The log stream ID. |
| TABLET_ID                    | bigint(20)   | NO         | The data shard ID.                                                                                  |
| COMPACTION_SCN                      | bigint(20) unsigned   | NO         | The version information of the compaction. |
| TASK_ID                      | varchar(64)  | NO         | The execution trace. |
| STATUS                       | varchar(15)  | NO         | The task status.  |
| DATA_SIZE                    | bigint(20)   | NO         | The total amount of data to be scanned.                                                                                |
| UNFINISHED_DATA_SIZE         | bigint(20)   | NO         | The amount of data that remains to be scanned.                                                                                 |
| PROGRESSIVE_COMPACTION_ROUND | bigint(20)   | NO         | The number of progressive compaction rounds. |
| CREATE_TIME                  | timestamp(6) | NO         | The task creation time. |
| START_TIME                   | timestamp(6) | NO         | The start time.  |
| ESTIMATED_FINISH_TIME        | timestamp(6) | NO         | The estimated completion time. |
| START_CG_ID                  | bigint(20)   | NO         | The start ID of the column group. <main id="notice" type='explain'><h4>Note</h4><p>This field is introduced in V4.3.0. </p></main>  |
| END_CG_ID                    | bigint(20)   | NO         | The end ID of the column group. <main id="notice" type='explain'><h4>Note</h4><p>This field is introduced in V4.3.0. </p></main>
