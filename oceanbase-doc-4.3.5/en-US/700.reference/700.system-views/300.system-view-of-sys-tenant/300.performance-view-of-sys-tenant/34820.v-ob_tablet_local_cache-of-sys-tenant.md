# V$OB_TABLET_LOCAL_CACHE

<main id="notice" type='explain'>
<h4>Note</h4>
<p>For V4.3.5, this view is introduced in V4.3.5 BP2. </p>
</main>

## Purpose

This view displays the overall cache usage of each tenant on the current OBServer node.

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view applies only to Shared-Storage (SS) mode. In Shared-Nothing (SN) mode, the query result of this view is empty. </p>
</main>

## Columns

|      Column      |         Type          | Nullable? |                       Description      |
|-----------------|-----------------------|------------------------|---------------------------------------|
| SVR_IP               | varchar(46)   | NO   | The IP address of the OBServer node.    |
| SVR_PORT             | bigint(20)    | NO   | The port number of the OBServer node.      |
| TENANT_ID            | bigint(20)    | NO   | The ID of the tenant.    |
| TABLET_ID            | bigint(20)    | NO   | The ID of the tablet.    |
| STORAGE_CACHE_POLICY | varchar(64)   | NO   | The cache status of the tablet. Valid values: <ul><li>`AUTO`: automatic cold and hot data recognition</li> <li>`HOT`: hot data</li></ul>   |
| CACHED_DATA_SIZE     | bigint(20)    | NO   | The amount of data cached locally by the tablet, in bytes.    |
| CACHE_HIT_COUNT      | bigint(20)    | NO   | The number of cache hits for the tablet. The value of this field is temporarily `-1`.    |
| CACHE_MISS_COUNT     | bigint(20)    | NO   | The number of cache misses for the tablet. The value of this field is temporarily `-1`.    |
| CACHE_HIT_SIZE       | bigint(20)    | NO   | The amount of data that hit the cache for the tablet, in bytes. The value of this field is temporarily `-1`.   |
| CACHE_MISS_SIZE      | bigint(20)    | NO   | The amount of data that missed the cache for the tablet, in bytes. The value of this field is temporarily `-1`.   |
| INFO                 | varchar(4096) | NO   | Other information.    |

## Sample query

In SS mode, query the cache usage of tenant ID `1002` in the current OBServer node in the sys tenant.

```shell
obclient> SELECT SVR_IP, SVR_PORT, TENANT_ID, TABLET_ID, STORAGE_CACHE_POLICY, INFO FROM oceanbase.V$OB_TABLET_LOCAL_CACHE WHERE TENANT_ID = 1002;
```
