# V$OB_SERVER_STORAGE

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view was introduced in OceanBase Database V4.3.4. </p>
</main>

## Purpose

This view displays the object storage configurations of the shared storage cluster on the current OBServer node.

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is applicable only to shared storage (SS) mode. If the database is in shared-nothing (SN) mode, the query result of this view is empty. </p>
</main>

## Columns

| Column | Type | Nullable? | Description |
|------|------|------|------|
| SVR_IP | varchar(46) | NO | The IP address of the OBServer node. |
| SVR_PORT | bigint(20) | NO | The port of the OBServer node. |
| PATH | varchar(1024) | NO | The root path used by object storage. This path can be of the following types: `oss://` or `s3://`. |
| ENDPOINT | varchar(1024) | NO | The endpoint of object storage. |
| USED_FOR | varchar(256) | NO | The users of object storage. Currently, only `ALL` is supported, indicating that object storage is used for clogs and data. |
| ZONE | varchar(128) | NO | The zone name. |
| STORAGE_ID | bigint(20) | NO | The ID of the object storage path. Paths with the same ID are the same. |
| MAX_IOPS | bigint(20) | NO | The maximum IOPS of object storage. |
| MAX_BANDWIDTH | bigint(20) | NO | The maximum bandwidth of object storage, in bytes. |
| CREATE_TIME | timestamp(6) | NO | The creation time of object storage. |
| AUTHORIZATION | varchar(1024) | NO | The authorization information of object storage. |
| STATE | varchar(256) | NO | The state of object storage. Valid values include:<ul><li>`ADDING`: indicates that object storage is being added. </li> <li>`ADDED`: indicates that object storage has been added. </li> <li>`DROPPING`: indicates that object storage is being dropped. </li> <li>`ROTATING`: indicates that object storage is being rotated. </li> <li>`ROTATED`: indicates that object storage has been rotated. </li></ul>      |
| STATE_INFO | varchar(1024) | NO | The state information of object storage. |
| LAST_CHECK_TIMESTAMP | timestamp(6) | NO | The time of the last check. |
| EXTENSION | varchar(4096) | NO | Other information about object storage. |
