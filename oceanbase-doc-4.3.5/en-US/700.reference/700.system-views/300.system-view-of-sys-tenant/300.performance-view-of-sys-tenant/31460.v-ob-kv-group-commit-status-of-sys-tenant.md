# V$OB_KV_GROUP_COMMIT_STATUS

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This view is available starting with V4.3.4. </p>
</main>

## Purpose

In the sys tenant, the `V$OB_KV_GROUP_COMMIT_STATUS` view can access the group commit status information for all OBKV tenant client connections on the current node.

## Columns

| **Column** | **Type** | **Nullable** | **Description** |
| --- | --- | --- | --- |
| SVR_IP | varchar(46) | NO | The IP address of the node where the group commit group is located. |
| SVR_PORT | bigint(20) | NO | The port number of the node where the group commit group is located. |
| TENANT_ID | bigint(20) | NO | The tenant ID: <li> 1: the sys tenant ID <li>Other values: the user tenant ID or the meta tenant ID |
| TABLE_ID | bigint(20) | NO | The ID of the table to which the group commit group belongs. |
| LS_ID | bigint(20) | NO | The ID of the leader server to which the group commit group belongs. |
| SCHEMA_VERSION | bigint(20) | NO | The schema version of the table to which the group commit group belongs. |
| GROUP_TYPE | varchar(32) | NO | The type of the group commit group. Supported types: <ul><li> get: get request queue </li><li>put: put request queue </li><li>del: delete request queue </li><li>replace: replace request queue </li><li>insert: insert request queue </li><li>hybird: hybrid operation request queue </li><li>fail: batch execution failure queue </li></ul>  |
| QUEUE_SIZE | bigint(20) | NO | The size of the request queue in the group commit group, representing the number of unprocessed requests. |
| BATCH_SIZE | bigint(20) | NO | The size of the batch in the group commit group. |
| CREATE_TIME | timestamp(6) | NO  | The time when the group was created in the group commit process.  |
| UPDATE_TIME | timestamp(6) | NO  | The time when the status information of the group was last refreshed in the group commit process.  |

## Sample query

Query the group commit status information for all OBKV tenant client connections on the current node and display the first day's records.

```shell
obclient > SELECT * FROM oceanbase.V$OB_KV_GROUP_COMMIT_STATUS limit 2;
```
