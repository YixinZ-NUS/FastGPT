# GV$OB_TENANT_MEMORY

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is introduced since OceanBase Database V4.0.0. </p>
</main>

## Purpose

The `GV$OB_TENANT_MEMORY` view displays the memory usage of all tenants in a cluster.

## Columns

| Column | Type | Nullable? | Description |
|-----------|-------------|------------|---------|
| TENANT_ID | bigint(20) | NO | The ID of the tenant. |
| SVR_IP | varchar(46) | NO | The IP address of the OBServer node. |
| SVR_PORT | bigint(20) | NO | The port number of the OBServer node. |
| HOLD | bigint(20) | NO | The memory used by the tenant. |
| FREE | bigint(21) | NO | The memory available for the tenant. |

## Sample query

Query the memory usage of the tenant with ID 1002 on all nodes in the system tenant.

```shell
obclient [oceanbase]> SELECT * FROM oceanbase.GV$OB_TENANT_MEMORY WHERE TENANT_ID = 1002;
```

The query result is as follows:

