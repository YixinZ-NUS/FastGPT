# V$OB_TABLET_COMPACTION_PROGRESS

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is introduced in V4.0.0. </p>
</main>

## Purpose

This view displays the progress of tablet-level compactions for all tenants on the current node.

## Columns

| Column | Type | Nullable? | Description |
| --- | --- | --- | --- |
| SVR_IP                       | varchar(46)  | NO         | The IP address of the server.                                                                                |
| SVR_PORT                     | bigint(20)   | NO         | The port number of the server. |
| TENANT_ID                    | bigint(20)   | NO         | The tenant ID. |
| TYPE                         | varchar(64)  | NO         | The type of compaction. <ul><li>`MDS_TABLE_MERGE`: persists the system metadata in the format of an SSTable to the disk. </li> <li>`MAJOR_MERGE`: tenant-level major compaction. </li> <li>`MEDIUM_MERGE`: partition-level major compaction. </li> <li>`MINI_MERGE`: minor compaction that converts the MemTables to Mini SSTables. </li> <li>`MINOR_MERGE`: major compaction that merges multiple Mini SSTables into one or merges multiple Mini SSTables and one Minor SSTable into one. </li> <li>`META_MAJOR_MERGE`: a special type of compaction that compacts data before a specified point in time into one Meta Major SSTable. The data in this SSTable is stored in the same format as that in a Major SSTable, but does not contain multi-version data or uncommitted transaction data. </li></ul>    |
| TABLE_ID                     | bigint(20)   | NO         | The ID of the primary table. |
| LS_ID                        | bigint(20)   | NO         | The log stream ID. |
| TABLET_ID                    | bigint(20)   | NO         | The ID of the data shard.                                                                                  |
| COMPACTION_SCN                      | bigint(20) unsigned   | NO         | The version information of the compaction. |
| TASK_ID                      | varchar(64)  | NO         | The executed trace.                                                                                |
| STATUS                       | varchar(15)  | NO         | The task status.  |
| DATA_SIZE                    | bigint(20)   | NO         | The total amount of data to be scanned.                                                                                |
| UNFINISHED_DATA_SIZE         | bigint(20)   | NO         | The amount of data that remains to be scanned.                                                                                 |
| PROGRESSIVE_COMPACTION_ROUND | bigint(20)   | NO         | The number of progressive compaction rounds. |
| CREATE_TIME                  | timestamp(6) | NO         | The task creation time. |
| START_TIME                   | timestamp(6) | NO         | The start time.  |
| ESTIMATED_FINISH_TIME        | timestamp(6) | NO         | The estimated completion time. |
| START_CG_ID                  | bigint(20)   | NO         | The start ID of the column group.<main id="notice" type='explain'><h4>Note</h4><p>This field is introduced in V4.3.0. </p></main>  |
| END_CG_ID                    | bigint(20)   | NO         | The end ID of the column group. <main id="notice" type='explain'><h4>Note</h4><p>This field is introduced in V4.3.0. </p></main>
