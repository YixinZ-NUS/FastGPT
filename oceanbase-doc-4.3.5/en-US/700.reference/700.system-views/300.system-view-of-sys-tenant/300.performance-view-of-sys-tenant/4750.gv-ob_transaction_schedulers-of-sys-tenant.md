# GV$OB_TRANSACTION_SCHEDULERS

<main id="notice" type='explain'>
 <h4>Note</h4>
 <p>This view is available starting with V4.2.1. </p>
</main>

## Purpose

`GV$OB_TRANSACTION_SCHEDULERS` displays the transaction scheduler information of all servers.

## Columns

| **Column** | **Type** | **Nullable** | **Description** |
| --- | --- | --- | --- |
| TENANT_ID | bigint(20) | NO | The tenant ID. |
| SVR_IP | varchar(46) | NO | The IP address of the server where the replica resides. |
| SVR_PORT | bigint(20) | NO | The port of the server where the replica resides. |
| SESSION_ID | bigint(20) | NO | For V4.3.x: <ul><li> V4.3.5 BP2 and earlier: <br> This field indicates the server session ID in all connection modes (direct mode or ODP mode). </li><li> V4.3.5 BP2 and later: <ul><li> In ODP mode: </li><ul><li> If the value of the <code>client_session_id_version</code> parameter in ODP is 2, this field indicates the client session ID. </li> <li> If the value of the <code>client_session_id_version</code> parameter in ODP is 1, this field indicates the server session ID. </li></ul></ul></li></ul> |
| TX_ID | bigint(20) | NO | The transaction ID. |
| STATE | varchar(18) | NO | The transaction state. |
| CLUSTER_ID | bigint(20) | NO | The unique ID of the cluster. |
| COORDINATOR | bigint(20) | NO | The log stream ID of the coordinator. |
| PARTICIPANTS | varchar(1024) | YES | The information about the participants of the transaction. |
| ISOLATION_LEVEL | varchar(16) | NO | The isolation level. |
| SNAPSHOT_VERSION | bigint(20) unsigned  | YES | The snapshot version when the RR or SERIAL isolation level is enabled. -1 is returned when the RR or SERIAL isolation level is not enabled. |
| ACCESS_MODE | varchar(10) | NO | The access mode (read-only or read/write). |
| TX_OP_SN | bigint(20)  | NO | The operation sequence number at the transaction level. |
| ACTIVE_TIME | timestamp(6) | YES | The timestamp when the transaction is explicitly or implicitly started. |
| EXPIRE_TIME | timestamp(6) | YES | The timestamp when the transaction expires. |
| CAN_EARLY_LOCK_RELEASE | varchar(7) | NO | Indicates whether the row lock can be released early. |
| FORMATID               | bigint(20)          | NO   | The format identifier of the global transaction. |
| GLOBALID               | varchar(257)        | YES  | The unique identifier of the global transaction. |
| BRANCHID               | varchar(257)        | YES  | The unique identifier of the branch transaction. |

## Sample query

```sql
obclient [oceanbase]> SELECT * FROM oceanbase.GV$OB_TRANSACTION_SCHEDULERS;
```
