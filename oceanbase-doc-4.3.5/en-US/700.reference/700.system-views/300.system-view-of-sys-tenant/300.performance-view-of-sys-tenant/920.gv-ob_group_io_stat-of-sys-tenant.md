# GV$OB_GROUP_IO_STAT

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view was introduced in OceanBase Database V4.3.3. </p>
</main>

## Purpose

This view displays the real-time status of all resource groups on all OBServer nodes of all tenants.

## Columns

| Column |  Type  | Nullable? | Description |
|------------|----------|----------|------------|
| TENANT_ID  | bigint(20) | NO       | Tenant ID. |
| SVR_IP     | varchar(46) | NO       | Server IP address. |
| SVR_PORT   | bigint(20) | NO       | Server port. |
| GROUP_ID   | bigint(20) | NO       | ID of the resource group. |
| GROUP_NAME | varchar(128) | NO       | Name of the resource group. |
| MODE       | varchar(128) | NO       | Read/write mode. Valid values include:<ul><li>`READ`: read mode</li> <li>`WRITE`: write mode</li></ul>     |
| MIN_IOPS   | bigint(20) | NO       | Minimum IOPS in the resource group. |
| MAX_IOPS   | bigint(20) | NO       | Maximum IOPS in the resource group. |
| NORM_IOPS                    | bigint(20)   | NO             | Current IOPS (displayed as the value converted to a 16 KB read equivalent)  <main id="notice" type='explain'><h4>Note</h4><p>For OceanBase Database V4.3.5, this field was introduced starting from V4.3.5 BP1.</p></main>  |
| REAL_IOPS  | bigint(20) | NO       | Current IOPS. |
| MAX_NET_BANDWIDTH | bigint(20) | NO       | Maximum network bandwidth in the resource group, in bytes.     |
| MAX_NET_BANDWIDTH_DISPLAY | varchar(128) | NO       | Converted and displayed maximum network bandwidth in the resource group, in the format of 10 GB/s, 10 MB/s, and so on.     |
| REAL_NET_BANDWIDTH | bigint(20) | NO       | Current network bandwidth, in bytes.    |
| REAL_NET_BANDWIDTH_DISPLAY   | varchar(128) | NO             | Converted and displayed current network bandwidth, in the format of 10 GB/s, 10 MB/s, and so on.     |

## Sample query

The following example queries the real-time status of all resource groups on all OBServer nodes of all tenants in the sys tenant.

```shell
obclient [oceanbase]> SELECT * FROM oceanbase.GV$OB_GROUP_IO_STAT LIMIT 1\G
```

The query result is as follows:

