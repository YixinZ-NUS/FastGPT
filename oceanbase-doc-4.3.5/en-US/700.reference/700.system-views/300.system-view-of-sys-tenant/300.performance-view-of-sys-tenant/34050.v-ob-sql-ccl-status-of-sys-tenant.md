# V$OB_SQL_CCL_STATUS

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is available starting with V4.3.5 BP3. </p>
</main>

## Purpose

This view records the number of SQL statements that hit the current throttling rules on all nodes for each tenant at the current moment. It is used to observe the impact of the throttling rules set in the current system.

## Columns

| Column | Type | Nullable | Description |
|---------|-------|-------------|--------|
| CON_ID              | bigint(20)  | NO   | The tenant ID.      |
| SVR_IP              | varchar(46) | NO   | The IP address of the OBServer node.     |
| SVR_PORT            | bigint(20)  | NO   | The port number of the BServer node.      |
| CCL_RULE_ID         | bigint(20)  | NO   | The ID of the effective throttling rule.     |
| FORMAT_SQLID        | varchar(32) | NO   | The throttling granularity:<li>Rule level: 0 </li><li>Format SQLID level:1</li>      |
| CURRENT_CONCURRENCY | bigint(20)  | NO   | The number of SQL statements that are currently executing and hit the throttling rule.     |
| MAX_CONCURRENCY     | bigint(20)  | NO   |  The maximum concurrency of the throttling rule.    |

## Sample query

```shell
obclient> SELECT * FROM oceanbase.GV$OB_SQL_CCL_STATUS;
```
