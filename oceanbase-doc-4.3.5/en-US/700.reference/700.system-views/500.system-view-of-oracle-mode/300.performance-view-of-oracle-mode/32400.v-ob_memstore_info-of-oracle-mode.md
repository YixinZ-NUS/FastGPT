# V$OB_MEMSTORE_INFO

<main id="notice" type='explain'>
<h4>Note</h4>
<p>The view <code>V$MEMSTORE_INFO</code> was renamed to <code>V$OB_MEMSTORE_INFO</code> in V4.0.0. You must use the new view name <code>V$OB_MEMSTORE_INFO</code> to query the view in V4.0.0 and later. Otherwise, an error will be returned. </p>
</main>

## Purpose

This view displays the detailed information about all MemTables in all partitions of the current tenant on the current server.

## Columns

| **Column** | **Type** | **Nullable?** | **Description** |
| --- | --- | --- | --- |
| SVR_IP | VARCHAR2(46) | NO | The IP address of the server. |
| SVR_PORT | NUMBER(38) | NO | The port of the server. |
| TENANT_ID | NUMBER(38) | NO | The ID of the tenant. |
| LS_ID | NUMBER(38) | NO | The ID of the log stream. |
| TABLE_ID | NUMBER(38) | NO | The ID of the data shard. |
| IS_ACTIVE | VARCHAR2(3) | NO | Whether the MemTable is active. |
| START_SCN | NUMBER(38) | NO | The starting SCN of the MemTable. |
| END_SCN | NUMBER(38) | NO | The ending SCN of the MemTable. |
| FREEZE_TS | NUMBER(38) | NO | The timestamp when the MemTable is frozen. |

## Sample query

The following example shows how to query the detailed information about all MemTables in all partitions of the current tenant on the current server for a user tenant.

```shell
obclient [SYS]> SELECT * FROM SYS.V$OB_MEMSTORE_INFO;
```
