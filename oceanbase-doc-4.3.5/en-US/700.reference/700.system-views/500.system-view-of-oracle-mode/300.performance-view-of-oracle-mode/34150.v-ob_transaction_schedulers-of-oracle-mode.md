# V$OB_TRANSACTION_SCHEDULERS

<main id="notice" type='explain'>
 <h4>Note</h4>
 <p>This view is available starting with V4.2.1. </p>
</main>

## Purpose

`V$OB_TRANSACTION_SCHEDULERS` is used to query the transaction scheduler information of all servers.

## Columns

| **Column** | **Type** | **Nullable** | **Description** |
| --- | --- | --- | --- |
| TENANT_ID | NUMBER(38) | NO | The tenant ID. |
| SVR_IP | VARCHAR2(46) | NO | The IP address of the server where the replica is located. |
| SVR_PORT | NUMBER(38) | NO | The port of the server where the replica is located. |
| SESSION_ID | NUMBER(38) | NO | For V4.3.x: <ul><li> For versions earlier than V4.3.5 BP2: <br> This field indicates the server session ID in all connection modes (direct mode or ODP mode). </li><li> For V4.3.5 BP2 and later: <ul><li> In ODP mode: <ul><li> If the <code>client_session_id_version</code> parameter is set to 2 in ODP, this field indicates the client session ID. </li> <li> If <code>client_session_id_version</code> is set to 1, this field indicates the server session ID. </li></ul></ul></li></ul> |
| TX_ID | NUMBER(38) | NO | The transaction ID. |
| STATE | VARCHAR2(18) | NO | The transaction state. |
| CLUSTER_ID | NUMBER(38) | NO | The unique ID of the cluster. |
| COORDINATOR | NUMBER(38) | NO | The log stream ID of the coordinator. |
| PARTICIPANTS | VARCHAR2(1024) | NO | The information about the participants of the transaction. |
| ISOLATION_LEVEL | VARCHAR2(16) | NO | The isolation level. |
| SNAPSHOT_VERSION | NUMBER(38)  | NO | The snapshot version when the RR or SERIAL isolation level is enabled. -1 is returned when the isolation level is not enabled. |
| ACCESS_MODE | VARCHAR2(10) | NO | The access mode (read-only or read/write). |
| TX_OP_SN | NUMBER(38)  | NO | The operation sequence number at the transaction level. |
| ACTIVE_TIME | TIMESTAMP(6) WITH LOCAL TIME ZONE | NO | The timestamp when the transaction was explicitly or implicitly started. |
| EXPIRE_TIME | TIMESTAMP(6) WITH LOCAL TIME ZONE | NO | The timestamp when the transaction expires. |
| CAN_EARLY_LOCK_RELEASE | VARCHAR2(7) | NO | Indicates whether row locks can be released early. |
| FORMATID               | NUMBER(38)          | NO   | The global transaction format identifier. |
| GLOBALID               | VARCHAR2(256)        | NO  | The unique identifier of the global transaction. |
| BRANCHID               | VARCHAR2(256)        | NO  | The unique identifier of the branch transaction. |

## Sample query

```sql
obclient [oceanbase]> SELECT * FROM oceanbase.V$OB_TRANSACTION_SCHEDULERS;
```
