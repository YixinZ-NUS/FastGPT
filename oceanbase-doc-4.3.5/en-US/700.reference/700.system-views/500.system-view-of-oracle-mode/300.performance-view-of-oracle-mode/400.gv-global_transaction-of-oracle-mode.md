# GV$GLOBAL_TRANSACTION

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is available starting with V2.2.77. </p>
</main>

## Purpose

This view displays information about the currently active global transactions.

  <main id="notice" >
    <h4>Applicability</h4>
    <p>This view is available starting with OceanBase Database in Oracle mode. </p>
  </main>

## Columns

|   **Column**   |     **Type**     | **Nullable?** |             **Description**              |
|--------------|----------------|-------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| FORMATID     | NUMBER(38)     | NO          | The format ID of the global transaction.                      |
| SVR_IP       | VARCHAR2(128)  | NO          | The IP address of the server.                          |
| SVR_PORT     | NUMBER(38)     | NO          | The port number of the server.                          |
| GLOBALID     | VARCHAR2(256 ) | NO          | The identifier of the global transaction.                        |
| BRANCHID     | VARCHAR2(256 ) | NO          | The branch qualifier of the global transaction.                       |
| BRANCHES     | NUMBER(38)     | NO          | The total number of branches in the global transaction.                    |
| REFCOUNT     | NUMBER(38)     | NO          | The number of peers of the global transaction, which must be the same as the number of branches.              |
| PREPARECOUNT | NUMBER         | NO          | The number of branches of the global transaction that have been prepared.                   |
| STATE        | CHAR(6)        | NO          | The status of the global transaction branch: <li> ACTIVE   <li> COLLECTING   <li> FINALIZED   <li> FAILED   <li> RECOVERING   <li> UNASSOCIATED   <li> FORGOTTEN   <li> READY FOR RECOVERY   <li> NO-READONLY FAILED   <li> SIBLING INFO WRITTEN   <li> \[ORACLE COORDINATED\]ACTIVE   <li> \[ORACLE COORDINATED\]COLLECTING   <li> \[ORACLE COORDINATED\]FINALIZED   <li> \[ORACLE COORDINATED\]FAILED   <li> \[ORACLE COORDINATED\]RECOVERING   <li> \[ORACLE COORDINATED\]UNASSOCIATED   <li> \[ORACLE COORDINATED\]FORGOTTEN   <li> \[ORACLE COORDINATED\]READY FOR RECOVERY   <li> \[ORACLE COORDINATED\]NO-READONLY FAILED   <li> \[MULTINODE\]ACTIVE   <li> \[MULTINODE\]COLLECTING   <li> \[MULTINODE\]FINALIZED   <li> \[MULTINODE\]FAILED   <li> \[MULTINODE\]RECOVERING   <li> \[MULTINODE\]UNASSOCIATED   <li> \[MULTINODE\]FORGOTTEN   <li> \[MULTINODE\]READY FOR RECOVERY   <li> \[MULTINODE\]NO-READONLY FAILED   <li> \[MULTINODE\]SIBLING INFO WRITTEN   <li> COMBINATION    |
| FLAGS        | NUMBER         | NO          | The numeric representation of the status.                         |
| COUPLING     | CHAR(15)       | NO          | Indicates whether the branch is free (FREE), loosely coupled (LOOSELY COUPLED), or tightly coupled (TIGHTLY COUPLED).                                                         |
| CON_ID       | VARCHAR2(256)  | NO          | The tenant ID.                           |
