# V$OB_SQL_CCL_STATUS

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is available starting with V4.3.5 BP3. </p>
</main>

## Purpose

This view records the number of SQL statements that hit the current throttling rules on the current node for the current tenant at the current moment. It is used to observe the impact of the throttling rules set in the system.

## Columns

| Column | Type | Nullable | Description |
|---------|-------|-------------|--------|
| CON_ID              | NUMBER(38)  | NO   | The tenant ID.      |
| SVR_IP              | VARCHAR2(46) | NO   | The IP address of the OBServer node.     |
| SVR_PORT            | NUMBER(38)  | NO   | The port number of the BServer node.      |
| CCL_RULE_ID         | NUMBER(38)  | NO   | The ID of the effective throttling rule.     |
| FORMAT_SQLID        | VARCHAR2(32) | NO   | The throttling granularity:<li>Rule level: 0 </li><li>Format SQLID level:1</li>      |
| CURRENT_CONCURRENCY | NUMBER(38)  | NO   | The number of SQL statements currently being executed that hit the throttling rule.     |
| MAX_CONCURRENCY     | NUMBER(38)  | NO   |  The maximum concurrency of the throttling rule.    |

## Sample query

```shell
obclient> SELECT * FROM V$OB_SQL_CCL_STATUS;
```
