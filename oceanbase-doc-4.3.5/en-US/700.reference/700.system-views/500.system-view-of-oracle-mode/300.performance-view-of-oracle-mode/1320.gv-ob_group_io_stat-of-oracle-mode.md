# GV$OB_GROUP_IO_STAT

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view was introduced in V4.3.3. </p>
</main>

## Purpose

This view displays the real-time status of each resource group on all OBServer nodes of the current tenant.

## Columns

| Column |  Type  | Nullable? | Description |
|------------|----------|----------|------------|
| TENANT_ID  | NUMBER(38) | NO       | The ID of the tenant. |
| SVR_IP     | VARCHAR2(46) | NO       | The IP address of the server. |
| SVR_PORT   | NUMBER(38) | NO       | The port number of the server. |
| GROUP_ID   | NUMBER(38) | NO       | The ID of the resource group. |
| GROUP_NAME | VARCHAR2(128)| NO       | The name of the resource group. |
| MODE       | VARCHAR2(128)| NO       | The read/write mode. Valid values include:<ul><li>`READ`: read mode</li> <li>`WRITE`: write mode</li></ul>     |
| MIN_IOPS   | NUMBER(38) | NO       | The minimum IOPS of the resource group. |
| MAX_IOPS   | NUMBER(38) | NO       | The maximum IOPS of the resource group. |
| NORM_IOPS                    | NUMBER(38)   | NO             | Current IOPS (displayed as the value converted to a 16 KB read equivalent)  <main id="notice" type='explain'><h4>Note</h4><p>For OceanBase Database V4.3.5, this field was introduced starting from V4.3.5 BP1.</p></main>  |
| REAL_IOPS  | NUMBER(38) | NO       | The current IOPS. |
| MAX_NET_BANDWIDTH| NUMBER(38) | NO       | The maximum network bandwidth of the resource group, in bytes. |
| MAX_NET_BANDWIDTH_DISPLAY| VARCHAR2(128)| NO       | The converted and displayed maximum network bandwidth of the resource group, in the format of 10 GB/s or 10 MB/s.     |
| REAL_NET_BANDWIDTH| NUMBER(38) | NO       | The current network bandwidth, in bytes.    |
| REAL_NET_BANDWIDTH_DISPLAY   | VARCHAR2(128)| NO             | The converted and displayed current network bandwidth, in the format of 10 GB/s or 10 MB/s.     |

## Sample query

The following example shows how to query the real-time status of each resource group on all OBServer nodes of the current tenant.

```shell
obclient [SYS]> SELECT * FROM SYS.GV$OB_GROUP_IO_STAT WHERE ROWNUM <= 2\G
```
