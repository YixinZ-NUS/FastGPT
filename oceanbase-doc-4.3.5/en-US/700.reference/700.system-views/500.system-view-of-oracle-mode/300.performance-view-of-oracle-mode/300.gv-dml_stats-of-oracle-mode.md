# GV$DML_STATS

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is introduced since OceanBase Database V4.0.0.</p>
</main>

## Purpose

This view displays the DML information of all user tables in the current tenant across all OBServer nodes.

## Columns

| Column | Type | Nullable? | Description |
| --- | --- | --- | --- |
| SVR_IP | VARCHAR2(46) | NO | The IP address of the server. |
| SVR_PORT | NUMBER | NO | The port number of the server. |
| INST_ID | NUMBER | NO | The ID of the tenant. |
| OBJN | NUMBER | NO | The ID of the object. |
| INS | NUMBER | NO | The approximate number of insertions since the statistics were last collected. |
| UPD | NUMBER | NO | The approximate number of updates since the statistics were last collected. |
| DEL | NUMBER | NO | The approximate number of deletions since the statistics were last collected. |
| DROPSEG | NUMBER | NO | Not supported. The default value is NULL. |
| CURROWS | NUMBER | NO | Not supported. The default value is NULL. |
| PAROBJN | NUMBER | NO | Not supported. The default value is NULL. |
| LASTUSED | NUMBER | NO | Not supported. The default value is NULL. |
| FLAGS | NUMBER | NO | Not supported. The default value is NULL. |
| CON_ID | NUMBER | NO | Not supported. The default value is NULL. |

## Sample query

View the DML information of user tables in this tenant on all OBServer nodes.

```shell
obclient [SYS]> SELECT * FROM SYS.GV$DML_STATS;
```
