# NLS_SESSION_PARAMETERS

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is available starting with V2.2.30. </p>
</main>

## Purpose

This view displays session-level NLS parameters.

  <main id="notice" >
    <h4>Applicability</h4>
    <p>This view is available starting with OceanBase Database. </p>
  </main>

## Columns

| **Column**  |     **Type**      | **Nullable** | **Description** |
|-----------|-----------------|----------------|--------|
| PARAMETER | VARCHAR2(128)   | NO             | The name of the parameter.   |
| VALUE     | VARCHAR2(65536) | YES            | The value of the parameter.   |

## Sample query

Query session-level NLS parameters in the current tenant.

```shell
obclient> SELECT * FROM SYS.NLS_SESSION_PARAMETERS;
```

The query result is as follows:

```shell
+-------------------------+----------------------------------+
| PARAMETER               | VALUE                            |
+-------------------------+----------------------------------+
| NLS_DATE_FORMAT         | YYYY-MM-DD HH24:MI:SS            |
| NLS_TIMESTAMP_FORMAT    | YYYY-MM-DD HH24:MI:SS.FF         |
| NLS_TIMESTAMP_TZ_FORMAT | YYYY-MM-DD HH24:MI:SS.FF TZR TZD |
| NLS_LANGUAGE            | AMERICAN                         |
| NLS_TERRITORY           | AMERICA                          |
| NLS_SORT                | BINARY                           |
| NLS_COMP                | BINARY                           |
| NLS_DATE_LANGUAGE       | AMERICAN                         |
| NLS_LENGTH_SEMANTICS    | BYTE                             |
| NLS_NCHAR_CONV_EXCP     | FALSE                            |
| NLS_CALENDAR            | GREGORIAN                        |
| NLS_NUMERIC_CHARACTERS  | .,                               |
+-------------------------+----------------------------------+
12 rows in set
```
