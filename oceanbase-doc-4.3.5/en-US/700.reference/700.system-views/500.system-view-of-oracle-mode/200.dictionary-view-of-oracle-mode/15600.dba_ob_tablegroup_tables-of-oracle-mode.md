# DBA_OB_TABLEGROUP_TABLES

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is introduced since OceanBase Database V4.0.0. </p>
</main>

## Purpose

The `DBA_OB_TABLEGROUP_TABLES` view displays associations between table groups and tables in the current tenant. Tables that do not belong to a table group are not displayed.

## Columns

| Column | Type | Nullable? | Description |
|-----------------|---------------|------------|---------------|
| TableGROUP_NAME | VARCHAR2(128) | NO | The name of the table group. |
| OWNER | VARCHAR2(128) | NO | The name of the database where the table belongs. |
| TABLE_NAME | VARCHAR2(256) | NO | The name of the table. |
| SHARDING        | VARCHAR2(10)  | NO         | The SHARDING attribute of the table group. Valid values: <ul><li>`NONE`: All partitions of all tables in the table group are aggregated on the same server, and no restrictions are imposed on tables to be added to the table group. </li> <li>`PARTITION`: The data of each table in the table group is scattered by partition. All subpartitions in a partition are aggregated together. </li> <li>`ADAPTIVE`: The data of each table in the table group is adaptively scattered. Specifically, tables partitioned by partition in the table group are partitioned by partition. Subpartitions in each partition are subpartitioned by subpartition. </li></ul>  |

## Sample query

A user tenant queries the associations of the table group named `TBLGROUP1` in the current tenant.

```shell
obclient [SYS]> SELECT * FROM SYS.DBA_OB_TABLEGROUP_TABLES WHERE tablegroup_name = 'TBLGROUP1';
```

The query result is as follows:

```shell
+-----------------+-------+------------+----------+
| TABLEGROUP_NAME | OWNER | TABLE_NAME | SHARDING |
+-----------------+-------+------------+----------+
| TBLGROUP1       | SYS   | TBL3       | NONE     |
+-----------------+-------+------------+----------+
1 row in set
```
