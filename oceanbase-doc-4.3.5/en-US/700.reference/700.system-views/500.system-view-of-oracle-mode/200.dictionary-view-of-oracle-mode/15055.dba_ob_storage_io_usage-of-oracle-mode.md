# oceanbase.DBA_OB_STORAGE_IO_USAGE

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view was introduced in OceanBase Database V4.3.4. </p>
</main>

## Purpose

This view displays the I/O operations on object storage and cloud disks in the current tenant.

## Columns

| Column | Type | Nullable? | Description |
|------|------|----------|-------------|
| PATH | VARCHAR2(1024) | NO | The path is in the format of `oss://`, `file://`, or `s3://`. |
| ENDPOINT | VARCHAR2(256) | NO | <ul><li>In shared-nothing (SN) mode, the endpoint of object storage used for backup and restore, data import, and data export. </li> <li>In shared-storage (SS) mode, the endpoint of object storage used for log (clog) or data (Data) storage, and backup and restore, data import, and data export. </li></ul> |
| TYPE | VARCHAR2(128) | NO | The type of operation:<ul><li>`READ`: read operations </li> <li>`WRITE`: write operations </li> <li>`READ_BYTES`: the total amount of network traffic from the object storage network entrance, in bytes. </li> <li>`WRITE_BYTES`: the total amount of network traffic from the object storage network exit, in bytes. </li> <li>`TAG`: the number of tags in object storage. This field is not displayed for local disks and cloud disks. </li></ul>     |
| TOTAL | NUMBER(38) | NO | The cumulative value of the type (`TYPE`):<ul><li>`READ`: the number of read operations. </li> <li>`WRITE`: the number of write operations. </li> <li>`READ_BYTES`: the total amount of network traffic from the object storage network entrance, in bytes. </li> <li>`WRITE_BYTES`: the total amount of network traffic from the object storage network exit, in bytes. </li> <li>`TAG`: the number of tags in object storage. This field is not displayed for local disks and cloud disks. </li></ul>      |
