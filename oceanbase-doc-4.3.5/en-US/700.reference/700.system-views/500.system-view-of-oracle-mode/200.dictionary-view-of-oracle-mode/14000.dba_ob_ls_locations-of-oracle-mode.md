# DBA_OB_LS_LOCATIONS

<main id="notice" type='explain'>
  <h4>Note</h4>
  <p>This view is introduced since OceanBase Database V4.0.0. </p>
</main>

## Purpose

The `DBA_OB_LS_LOCATIONS` view displays the distribution of the log stream replicas in the current tenant.

## Columns

| Column | Type | Nullable? | Description |
|--------------|---------------|------------|-------------------------------------|
| CREATE_TIME        | VARCHAR2(19)    | NO         | The time when the log stream was created. |
| MODIFY_TIME        | VARCHAR2(19)    | NO         | The time when the log stream was modified. |
| SVR_IP       | varchar2(46)   | NO         | The IP address of the OBServer node. |
| SVR_PORT     | NUMBER    | NO         | The port number of the OBServer node. |
| SQL_PORT     | NUMBER    | NO         | The SQL port number. |
| ZONE         | varchar2(128)  | NO         | The name of the zone. |
| ROLE         | varchar2(8)    | NO         | The role of the replica. Valid values:<li>`LEADER`<li>`FOLLOWER` |
| MEMBER_LIST  | varchar2(4480) | NO         | The list of replica members. Multiple replicas are separated with commas (,). <ul><li>This column is valid when `ROLE` is set to `LEADER`. A single replica is identified in the `svr_ip:svr_port:timestamp` format. The parameters are described as follows: <ul><li>`svr_ip`: the IP address of the OBServer node where the replica resides.</li> <li>`svr_port`: the port number of the OBServer node where the replica resides.</li> <li>`timestamp`: the time when the replica was added to the `MEMBER_LIST`. The default value is `1`, which changes after you add or migrate a replica.</li></ul></br>Here is an example: `172.xx.xxx.xxx:2882:1`.</li> <li>The value of this column is `NULL` when `ROLE` is not `LEADER`. </li></ul> |
| PAXOS_REPLICA_NUMBER       | NUMBER    | NO         | The quorum of a Paxos group. When `ROLE` is set to `LEADER`, this column is valid. In other cases, the value of this column is `NULL`. |
| REPLICA_TYPE | varchar2(18)   | NO         | The replica type. Valid values: <ul><li>`FULL`: indicates a full-featured replica.</li> <li>`READONLY`: indicates a read-only replica.</li> <li>`COLUMNSTORE`: indicates a columnstore replica.</li></ul> |
| LEARNER_LIST | CLOB           | YES  | The list of read-only replicas in the current log stream. Multiple replicas are separated with commas (,). A single replica is identified in the `svr_ip:svr_port:timestamp:flag` format. The parameters are described as follows: <ul><li>`svr_ip`: the IP address of the OBServer node where the replica resides.</li> <li>`svr_port`: the port number of the OBServer node where the replica resides.</li> <li>`timestamp`: the time when the replica was added to the `LEARNER_LIST`. The default value is `1`, which changes after you add or migrate a replica.</li> <li>`flag`: indicates whether the replica is being migrated. The value `0` indicates that the replica is not being migrated. The value `2` indicates that the replica is being migrated. </li></ul></br>Here is an example: `172.xx.xxx.xxx:2882:1:0`.<main id="notice" type='explain'><h4>Note</h4><p>This column is available in OceanBase Database V4.2.0 and later. </p></main> |
| REBUILD      | VARCHAR2(5)    | NO         | Indicates whether the replica is being rebuilt.<main id="notice" type='explain'><h4>Note</h4><p>This column is available in OceanBase Database V4.2.1 and later. </p></main> |
