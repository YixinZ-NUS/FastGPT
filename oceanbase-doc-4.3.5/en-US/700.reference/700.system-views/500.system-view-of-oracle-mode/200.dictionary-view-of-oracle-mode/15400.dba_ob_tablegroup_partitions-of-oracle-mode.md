# DBA_OB_TABLEGROUP_PARTITIONS

<main id="notice" type='explain'>
<h4>Note</h4>
<ul>
<li>Starting from V4.2.0, table groups no longer support partitions. Therefore, this view is no longer used. You can query this view, but the query result is empty. </li>
<li>For V4.2.0 and later, you can query table group information by using the view <a href="15700.dba_ob_tablegroups-of-oracle-mode.md">DBA_OB_TABLEGROUPS</a>. For more information about table groups, see <a href="../../../300.database-object-management/200.manage-object-of-oracle-mode/300.manage-table-groups-of-oracle-mode/100.about-table-groups-of-oracle-mode.md">About table groups</a>. </li>
</ul>
</main>

## Purpose

This view displays the information of primary partitions of table groups that contain partitions.

## Columns

|        Column        |       Type       | Nullable? | Description |
|--------------------|----------------|------------|---------------------------|
| TABLEGROUP_NAME    | VARCHAR2(128)  | NO         | The name of the table group.                                            |
| COMPOSITE          | VARCHAR2(3)    | NO         | Whether the table group is a subpartitioned table group. Valid values: `YES` and `NO`.                             |
| PARTITION_NAME     | VARCHAR2(64)   | NO         | The name of the primary partition of the table group.                                              |
| SUBPARTITION_COUNT | NUMBER         | NO         | `NULL`                  |
| HIGH_VALUE         | VARCHAR2(4096) | NO         | `NULL`            |
| HIGH_VALUE_LENGTH  | NUMBER         | NO         | `NULL`                             |
| PARTITION_POSITION | NUMBER         | NO         | `NULL`   |
