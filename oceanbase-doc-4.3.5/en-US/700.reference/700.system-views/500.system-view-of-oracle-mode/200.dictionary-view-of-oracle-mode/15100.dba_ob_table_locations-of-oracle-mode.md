# DBA_OB_TABLE_LOCATIONS

<main id="notice" type='explain'>
<h4>Note</h4>
<p>This view is introduced since OceanBase Database V4.0.0. </p>
</main>

## Purpose

This view displays the locations of tables or partitions, including system tables, user tables, and index tables.

## Columns

| Column |  Type  | Nullable? | Description |
| --- | --- | --- | --- |
| DATABASE_NAME | VARCHAR2(128) | NO | Database name. |
| TABLE_NAME | VARCHAR2(256) | NO | Table name. |
| TABLE_ID | NUMBER(38) | NO | Table ID. |
| TABLE_TYPE | VARCHAR2(13) | NO | Table type. Valid values include:<ul><li>USER TABLE: user table. </li><li>SYSTEM TABLE: system table, a table created by the system. </li><li>INDEX: index table. </li><li>LOB AUX TABLE: LOB auxiliary table, a table created by the system. It is used to support LOB columns. </li></ul> |
| PARTITION_NAME | VARCHAR2(64) | NO | Name of the primary partition. The value is NULL for a non-partitioned table. |
| SUBPARTITION_NAME | VARCHAR2(64) | NO | Name of the subpartition. The value is NULL if there is no subpartition. |
| INDEX_NAME | VARCHAR2(1024) | NO | If the table is an index table, the name of the index table is displayed. Otherwise, NULL is displayed. |
| DATA_TABLE_ID | NUMBER | NO | ID of the primary table if the table is an index table or a LOB auxiliary table. Otherwise, NULL is displayed. |
| TABLET_ID | NUMBER(38) | NO | TABLET ID. |
| LS_ID | NUMBER | NO | ID of the log stream where the object resides. |
| ZONE | VARCHAR2(128) | NO | Zone where the replica resides. |
| SVR_IP | VARCHAR2(46) | NO | IP address of the replica. |
| SVR_PORT | NUMBER | NO | Port of the replica. |
| ROLE | VARCHAR2(8) | NO | Role of the replica. |
| REPLICA_TYPE | VARCHAR2(18) | NO | Log stream replica type. Valid values include:<ul><li>`FULL`: full feature replica. </li> <li>READONLY: read-only replica. </li> <li>`COLUMNSTORE`: columnstore replica. </li></ul>  |
| DUPLICATE_SCOPE | VARCHAR2(7) | NO | Indicates whether the table is a replicated table. Valid values include:<ul><li>`CLUSTER`: replicated table. </li><li>`NONE`: ordinary table. </li></ul><main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
| DUPLICATE_READ_CONSISTENCY | varchar(6) | NO | Read consistency constraint of the standby replica of a replicated table. Valid values include:<ul><li>`NONE`: indicates that the table is not a replicated table. This attribute is meaningless in this case. </li><li>`STRONG`: indicates that the standby replica of a replicated table supports strong consistency read. </li></ul> <main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.3.4. </p></main>|
| OBJECT_ID | NUMBER(38) | NO | ID of the partition<main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
| TABLEGROUP_NAME | VARCHAR2(128) | NO | Name of the table group to which the table belongs<main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
| TABLEGROUP_ID | NUMBER(38) | NO | ID of the table group to which the table belongs<main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
| SHARDING | VARCHAR2(10) | NO | Sharding attribute (equilibrium mode) of the table group to which the table belongs<main id="notice" type='explain'><h4>Note</h4><p>This field was introduced in V4.2.0. </p></main> |
