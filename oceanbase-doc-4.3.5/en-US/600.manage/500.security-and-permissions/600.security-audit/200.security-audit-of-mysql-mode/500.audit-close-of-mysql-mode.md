# Disable security auditing

After a series of filters are used to auditing specific events, you can clean up the environment by clearing filter configurations, deleting filters, and clearing auditing logs.

<main id="notice">
  <h4>Applicability</h4>
  <p>This topic applies only to OceanBase Database Enterprise Edition. OceanBase Database Community Edition does not support the auditing feature.</p>
</main>

## Clear filter configurations

You can call the `AUDIT_LOG_FILTER_REMOVE_USER` function (expression) to remove a filter from a user.

### Syntax

```sql
AUDIT_LOG_FILTER_REMOVE_USER('user_name');
```

**The following table describes the fields in the statement.**

| Field      | Description |
|------------|-------------|
| user_name  | Specifies the user name.<main id="notice" type='explain'><h4>Note</h4><p>The <code>AUDIT_LOG_FILTER_REMOVE_USER</code> function has the same semantics as the <code>DROP IF EXISTS</code> statement. If the specified object does not exist, no error is returned.</p></main>|

### Example

Remove the filter from user `user001`.

```shell
obclient [test]> SELECT AUDIT_LOG_FILTER_REMOVE_USER('user001');
```

The result is as follows:

```shell
+-----------------------------------------+
| AUDIT_LOG_FILTER_REMOVE_USER('user001') |
+-----------------------------------------+
| OK                                      |
+-----------------------------------------+
1 row in set
```

## Delete a filter

You can call the `AUDIT_LOG_FILTER_REMOVE_FILTER` function (expression) to delete a filter.

### Syntax

```sql
AUDIT_LOG_FILTER_REMOVE_FILTER('filter_name');
```

**The following table describes the fields in the statement:**

| Field       | Description |
|-------------|-------------|
| filter_name | Specifies the filter name.<main id="notice" type='explain'><h4>Note</h4><p>The <code>AUDIT_LOG_FILTER_REMOVE_FILTER</code> function has the same semantics as the <code>DROP IF EXISTS</code> statement. If the specified object does not exist, no error is returned.</p></main>|

### Example

Delete the filter `log_all`.

```shell
obclient [test]> SELECT AUDIT_LOG_FILTER_REMOVE_FILTER('log_all');
```

The result is as follows:

```shell
+-------------------------------------------+
| AUDIT_LOG_FILTER_REMOVE_FILTER('log_all') |
+-------------------------------------------+
| OK                                        |
+-------------------------------------------+
1 row in set
```
