# DROP SYNONYM

## 描述

该语句用来删除一个同义词。

## 语法

```sql
DROP [PUBLIC] SYNONYM [schema_name.]synonym_name [FORCE]
```

<main id="notice" type='notice'>
<h4>注意</h4>
<p>删除同义词时，需要满足如下权限：</p>
<ul>
<li>删除私有同义词时：
  <ul>
    <li>要删除的 <code>SYNONYM</code> 在对应 Schema 下。</li>
    <li>要有 <code>DROP ANY SYNONYM</code> 权限。</li>
  </ul>
</li>
<li>删除公共同义词时：
  <ul>
    <li>必须指定 <code>PUBLIC</code> 关键字，且不能指定 <code>schema</code>。</li>
    <li>需要有 <code>DROP PUBLIC SYNONYM</code> 权限。</li>
  </ul>
</li>
</ul></main>

## 参数说明

|      参数      |                                        描述                                         |
|--------------|-----------------------------------------------------------------------------------|
| PUBLIC       | 指定 `PUBLIC` 来删除公共同义词。如果不指定 `PUBLIC`，则删除私有同义词。                                     |
| schema       | 指定同义词所在的 Schema。如果省略 `schema.`，则默认同义词在您自己的模式中。如果指定了 `PUBLIC`，则对同义词不需要指定 `schema`。 |
| synonym_name | `synonym_name` 表示同义词的名称。                                                          |
| FORCE        | 指定 `FORCE`，可删除具有依赖表或用户定义类型的同义词。                                                   |

## 示例

* 创建并删除一个私有同义词。

  ```sql
  -- 创建表
  obclient> CREATE TABLE employees (
    emp_id NUMBER PRIMARY KEY,
    emp_name VARCHAR2(50)
  );

  -- 创建并删除私有同义词
  obclient> CREATE SYNONYM syn1 FOR employees;
  obclient> DROP SYNONYM syn1;
  ```

* 创建并删除一个 `PUBLIC` 同义词。

  ```sql
  -- 创建表
  obclient> CREATE TABLE departments (
    dept_id NUMBER PRIMARY KEY,
    dept_name VARCHAR2(50)
  );

  -- 创建并删除公共同义词
  obclient> CREATE PUBLIC SYNONYM syn2 FOR departments;
  obclient> DROP PUBLIC SYNONYM syn2;
  ```
