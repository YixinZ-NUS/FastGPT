# oceanbase.DBA_OB_SENSITIVE_RULES

<main id="notice" type='explain'>
<h4>说明</h4>
<p>该视图从 V4.3.5 BP3 版本开始引入。</p>
</main>

## 功能描述

展示当前租户下所有敏感规则的定义和属性。

## 字段说明

|        字段名称         |      类型       | 是否可以为 NULL |                      描述                                |
|---------------------|---------------|------------|-----------------------------------------------------------------|
| RULE_NAME         | varchar(256)    | NO         | 敏感规则名称                                           |
| PROTECTION_POLICY      | varchar(16)    | NO         | 保护策略，取值为 `ENCRYPTION`/`INVALID`。`ENCRYPTION` 表示加密保护，`INVALID` 表示无效。                                        |
| METHOD        | varchar(4096)    | NO         | 指定加密方法。取值范围见下文                                      |
| ENABLED        | varchar(3)    | NO         | 是否启用，取值为 `YES` 或 `NO`。`YES` 表示启用，`NO` 表示禁用                                      |

`METHOD` 取值范围：

| 取值（在 SQL 语法中使用） | 实际使用的加密算法 | 密钥长度 | 模式 | 说明 |
|---|----------|----------|------|------|
| aes-256 | AES-256-ECB | 256 位 | ECB | 默认算法 |
| aes-128 | AES-128-ECB | 128 位 | ECB | 兼容性好 |
| aes-192 | AES-192-ECB | 192 位 | ECB | 中等安全级别 |
| aes-128-gcm | AES-128-GCM | 128 位 | GCM | 提供认证加密 |
| aes-256-gcm | AES-256-GCM | 256 位 | GCM | 高安全级别，提供认证加密 |
| sm4-cbc | SM4-CBC |  | CBC | 高安全级别 |
| sm4-GCM | SM4-GCM |  | GCM | 高安全级别 |

## 查询示例

查看当前租户下的所有敏感规则的定义和属性。

```shell
obclient [oceanbase]> SELECT * FROM oceanbase.DBA_OB_SENSITIVE_RULES;
```

查询结果如下：

```shell
+-----------+-------------------+---------+---------+
| RULE_NAME | PROTECTION_POLICY | METHOD  | ENABLED |
+-----------+-------------------+---------+---------+
| r1        | ENCRYPTION        | aes-256 | YES     |
+-----------+-------------------+---------+---------+
1 row in set
```
