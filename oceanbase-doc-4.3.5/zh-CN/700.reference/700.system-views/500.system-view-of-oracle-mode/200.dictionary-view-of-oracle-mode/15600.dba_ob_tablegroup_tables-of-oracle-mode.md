# DBA_OB_TABLEGROUP_TABLES

<main id="notice" type='explain'>
<h4>说明</h4>
<p>该视图从 V4.0.0 版本开始引入。</p>
</main>

## 功能描述

展示本租户表组、表的关联关系。不展示不在表组中的表。

## 字段说明

|      字段名称       |      类型       | 是否可以为 NULL |      描述       |
|-----------------|---------------|------------|---------------|
| TableGROUP_NAME | VARCHAR2(128) | NO         | 表组名           |
| OWNER           | VARCHAR2(128) | NO         | 表所属的数据库名 |
| TABLE_NAME      | VARCHAR2(256) | NO         | 表名       |
| SHARDING        | VARCHAR2(10)  | NO         | 表组的 SHARDING 属性：<ul><li>`NONE`：表组内的所有表的所有分区均聚集在同一台机器上，并且对加入表组的表没有任何限制。</li> <li>`PARTITION`：表组内每一张表的数据按一级分区打散；如果是二级分区表，则一级分区下的所有二级分区聚集在一起。</li> <li>`ADAPTIVE`：表组内每一张表的数据根据自适应方式打散。即，如果表组内的表是一级分区表，则按一级分区打散；如果表组内的表是二级分区表，则按每个一级分区下的二级分区打散。</li></ul>  |

## 查询示例

用户租户查询本租户名为 `TBLGROUP1` 的表组的关联关系。

```shell
obclient [SYS]> SELECT * FROM SYS.DBA_OB_TABLEGROUP_TABLES WHERE tablegroup_name = 'TBLGROUP1';
```

查询结果如下：

```shell
+-----------------+-------+------------+----------+
| TABLEGROUP_NAME | OWNER | TABLE_NAME | SHARDING |
+-----------------+-------+------------+----------+
| TBLGROUP1       | SYS   | TBL3       | NONE     |
+-----------------+-------+------------+----------+
1 row in set
```
