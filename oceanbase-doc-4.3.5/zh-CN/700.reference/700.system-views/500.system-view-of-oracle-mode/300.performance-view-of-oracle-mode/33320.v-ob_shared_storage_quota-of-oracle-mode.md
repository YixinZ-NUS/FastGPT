# V$OB_SHARED_STORAGE_QUOTA

<main id="notice" type='explain'>
<h4>说明</h4>
<p>该视图从 V4.3.4 版本开始引入。</p>
</main>

## 功能描述

展示当前 OBServer 节点上的 IO 访问资源信息。

## 字段说明

| 字段名称     | 类型     | 是否可以为 NULL | 描述                                                        |
|-------------|----------|----------------|-------------------------------------------------------------|
| SVR_IP      | VARCHAR2(46)  | NO   | OBServer 节点的 IP 地址     |
| SVR_PORT    | NUMBER(38)    | NO   | OBServer 节点的端口号     |
| ENDPOINT    | VARCHAR2(256) | NO   | 共享存储的 Endpoint。无共享（Shared-Nothing，SN）模式下，该字段值为空。     |
| PATH        | VARCHAR2(1024)| NO   | 对象存储的路径。无共享（Shared-Nothing，SN）模式下，展示为本地路径。</br>该路径通常使用的类型有：`local://`、`oss://`、`file://`、`s3://` 等。   |
| CLASS_ID    | NUMBER(38)    | NO   | 用于标注的 ID。根据对象存储的 Bucket 的用途不同，有不同的含义：<ul><li>当 Bucket 用于 clog 存储或数据存储时，该字段无意义，值为 `0`。</li> <li>当 Bucket 用于备份恢复、数据导入、导出等任务时，该字段的含义为租户 ID。</li></ul>      |
| TYPE        | VARCHAR2(32)  | NO   | 指标类型，主要包含以下类型：<ul><li>`IPS`：每秒的指令数</li> <li>`OPS`：每秒执行的操作次数</li> <li>`IOPS`：每秒输入或输出的操作数</li> <li>`IBW`：输入带宽</li> <li>`OBW`：输出带宽</li> <li>`IOBW`：输入或输出带宽</li> <li>`TAG`：用户设置的标签信息</li></ul>     |
| REQUIREMENT | NUMBER(38)    | NO   | 服务器所需的各项资源    |
| ASSIGN      | NUMBER(38)    | NO   | 服务器被限速算法所指定的各项资源的上限     |

## 查询示例

用户租户下，对于 clog 存储、数据存储等场景，查询当前 OBServer 节点上的对象存储的各项指标的申请配额及限制配额。对于备份恢复、数据导入、导出等场景，查询本租户当前 OBServer 节点上的对象存储的各项指标的申请配额及限制配额。

```shell
obclient> SELECT * FROM SYS.V$OB_SHARED_STORAGE_QUOTA;
```
