# V$OB_TRANSACTION_SCHEDULERS

<main id="notice" type='explain'>
 <h4>说明</h4>
 <p>该视图从 V4.2.1 版本开始引入。</p>
</main>

## 功能描述

`V$OB_TRANSACTION_SCHEDULERS` 用于查询所有 Server 的事务 Scheduler 信息。

## 字段说明

| **字段** | **类型** | **是否可以为 NULL** | **说明** |
| --- | --- | --- | --- |
| TENANT_ID | NUMBER(38) | NO | 租户 ID |
| SVR_IP | VARCHAR2(46) | NO | 副本所在的 Server IP 地址 |
| SVR_PORT | NUMBER(38) | NO | 副本所在的 Server 端口 |
| SESSION_ID | NUMBER(38) | NO | 对于 V4.3.x 版本：<ul><li> V4.3.5 BP2 之前版本：<br>在所有连接方式（直连模式/ODP 模式）下，该字段均表示 Server Session ID。</li><li> V4.3.5 BP2 及之后版本：<ul><li>在 ODP 模式下：</li><ul><li>当 ODP 中的配置项 <code>client_session_id_version = 2</code> 时，该字段表示 Client Session ID。 </li> <li>当 <code>client_session_id_version = 1</code> 时，该字段表示 Server Session ID。</li></ul></ul></li></ul> |
| TX_ID | NUMBER(38) | NO | 事务 ID |
| STATE | VARCHAR2(18) | NO | 事务状态 |
| CLUSTER_ID | NUMBER(38) | NO | 集群唯一 ID |
| COORDINATOR | NUMBER(38) | NO | 协调者所在的日志流 ID |
| PARTICIPANTS | VARCHAR2(1024) | NO | 事务的参与者信息 |
| ISOLATION_LEVEL | VARCHAR2(16) | NO | 隔离级别 |
| SNAPSHOT_VERSION | NUMBER(38)  | NO | 开启 RR  或 SERIAL 隔离级别的快照版本，没有开启则为 -1 |
| ACCESS_MODE | VARCHAR2(10) | NO | 访问模式（只读或者读写） |
| TX_OP_SN | NUMBER(38)  | NO | 事务级别的操作序列号 |
| ACTIVE_TIME | TIMESTAMP(6) WITH LOCAL TIME ZONE | NO | 显示或隐式开启事务的时间戳 |
| EXPIRE_TIME | TIMESTAMP(6) WITH LOCAL TIME ZONE | NO | 事务到期时间戳 |
| CAN_EARLY_LOCK_RELEASE | VARCHAR2(7) | NO | 是否能够提前解行锁 |
| FORMATID               | NUMBER(38)          | NO   | 全局事务的格式标识符 |
| GLOBALID               | VARCHAR2(256)        | NO  | 全局事务唯一标识符 |
| BRANCHID               | VARCHAR2(256)        | NO  | 分支事务的唯一标识符 |

## 查询示例

```sql
obclient [oceanbase]> SELECT * FROM oceanbase.V$OB_TRANSACTION_SCHEDULERS;
```
