# oceanbase.DBA_OB_ZONE_STORAGE

<main id="notice" type='explain'>
<h4>说明</h4>
<p>该视图从 V4.3.4 版本开始引入。</p>
</main>

## 功能描述

展示各个 Zone 上共享存储集群配置的对象存储的相关信息。

<main id="notice" type='explain'>
<h4>说明</h4>
<p>该视图仅适用于共享存储（Shared-Storage，SS） 模式。无共享（Shared-Nothing，SN） 模式下，该视图的查询结果为空。</p>
</main>

## 字段说明

| 字段名称       | 类型          | 是否可以为 NULL | 描述                          |
|---------------|---------------|----------------|-------------------------------|
| CREATE_TIME   | datetime      | YES  | 记录的创建时间     |
| MODIFY_TIME   | datetime      | YES  | 记录的修改时间     |
| ZONE          | varchar(128)  | NO   | Zone 名称     |
| PATH          | varchar(1024) | NO   | 对象存储所使用的根路径。该路径通常使用的类型有：`oss://`、`s3://` 等。     |
| ENDPOINT      | varchar(1024) | NO   | 对象存储的 Endpoint。    |
| USED_FOR      | varchar(256)  | NO   | 对象存储的使用者。当前仅支持 `ALL`，表示供 clog 和 data 使用。    |
| STORAGE_ID    | bigint(20)    | NO   | 对象存储路径的编号，相同路径有相同的 STORAGE_ID     |
| AUTHORIZATION | varchar(1024) | NO   | 对象存储的授权信息      |
| MAX_IOPS      | bigint(20)    | YES  | 对象存储的最大 IOPS     |
| MAX_BANDWIDTH | bigint(20)    | YES  | 对象存储的最大带宽，单位为字节     |
| STATE         | varchar(256)  | NO   | 对象存储的状态：<ul><li>`ADDING`：正在添加状态 </li> <li>`ADDED`：已添加成功</li> <li>`DROPPING`：正在删除状态</li> <li>`ROTATING`：正在轮转 AK/SK</li> <li>`ROTATED`：AK/SK 轮转已完成</li></ul>     |
| EXTENSION     | varchar(4096) | YES  | 对象存储的其它信息     |
