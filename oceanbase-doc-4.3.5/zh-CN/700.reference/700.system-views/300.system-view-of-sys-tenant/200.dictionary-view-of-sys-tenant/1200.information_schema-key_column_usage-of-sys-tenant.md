# information_schema.KEY_COLUMN_USAGE

<main id="notice" type='explain'>
<h4>说明</h4>
<p>该视图从 V1.4 版本开始引入。</p>
</main>

## 功能描述

展示 column 中 key 的约束信息。

## 字段说明

|           **字段名称**            |    **类型**    | **是否可以为空** |                                         **描述**                                          |
|-------------------------------|--------------|------------|---------------------|
| CONSTRAINT_CATALOG            | varchar(3)   | NO         | 固定值（def）                                                                                |
| CONSTRAINT_SCHEMA             | varchar(128) | NO         | 数据库名 |
| CONSTRAINT_NAME               | varchar(7)   | NO         | 约束名，为 PRIMARY 或列名或外键名                                                                   |
| TABLE_CATALOG                 | varchar(3)   | NO         | 表目录  |
| TABLE_SCHEMA                  | varchar(128) | NO         | 数据库名 |
| TABLE_NAME                    | varchar(256) | NO         | 表名   |
| COLUMN_NAME                   | varchar(128) | NO         | 列名   |
| ORDINAL_POSITION              | bigint(20)   | NO         | 该列在表中的编号                                                                                |
| POSITION_IN_UNIQUE_CONSTRAINT | null         | NO         | 对于唯一和主键约束，指当前列在约束定义中的顺序；对于外键约束，指所引用表内键中的顺序位置 |
| REFERENCED_TABLE_SCHEMA       | varchar(128) | NO         | 默认值为 `NULL`                      |
| REFERENCED_TABLE_NAME         | varchar(256) | NO         | 默认值为 `NULL` |
| REFERENCED_COLUMN_NAME        | varchar(128) | NO         | 默认值为 `NULL` |

## 查询示例

查询表 `tbl4` 的列中 key 的约束信息。

```shell
obclient [oceanbase]> SELECT * FROM information_schema.KEY_COLUMN_USAGE WHERE TABLE_NAME='tbl4';
```
