# 自动触发转储

本节主要介绍自动转储的触发机制。

当一个租户 Active MemTable 内存的使用量达到 `freeze_trigger_percentage * memstore_limit`（其中，`memstore_limit = 租户内存 * memstore_limit_percentage`）时，系统就会自动触发冻结（转储的前置动作），然后再调度转储。当冻结（minor freeze）的次数达到一定的阈值时，就会触发自动合并。

更多配置项 `memstore_limit_percentage` 和 `freeze_trigger_percentage` 的说明及设置方法，请参见 [修改转储配置](../100.dump-management/500.modify-dump-configuration.md)。
