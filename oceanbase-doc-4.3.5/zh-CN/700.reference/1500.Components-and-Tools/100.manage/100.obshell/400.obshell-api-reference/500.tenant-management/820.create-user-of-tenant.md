# 创建用户

本文介绍如何通过 obshell 为一个租户创建用户。

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>自 obshell V4.2.6 起支持该 API。</p>
</main>

## 调用说明

### 接口约束

obshell Server 会对该 API 进行安全校验，详细介绍可参见 [API 混合加密](../20.api-hybrid-encryption.md)。

### 请求路径

POST /api/v1/tenant/{name}/user

### 请求参数

| 参数 | 类型 | 必选 | 示例值 | 描述  |
|------|------|-----|------|--------|
| user_name | string | 是 | proxyro | 待创建的用户名称。  |
| password | string | 是 | ***** | 待创建用户的密码。  |
| root_password | string | 否 | ***** | 租户的 root 用户密码，当创建系统租户用户时，无需传入。  |
| global_privileges | []string | 否 | ["SELECT", "DELETE"] | 指定对所有 database 的权限。  |
| db_privileges | []DbPrivilegeParam | 否 | {</br>  "db_name": "oceanbase",</br>  "privileges": [</br>    "DROP"</br>  ]</br>} | 指定对特定 database 的权限，具体介绍可参见下文 **DbPrivilegeParam 信息的数据结构**。  |
| host_name | string | 否 | % | 设置访问白名单。  |

DbPrivilegeParam 信息的数据结构如下：

| 参数 | 类型 | 必选 | 示例值 | 描述 |
|------|-----|------|--------|------|
| db_name | string | 是 | oceanbase | 指定的 database 名称。 |
| privileges | []string | 是 | ["SELECT","DELETE"] | 用户可拥有的权限。 |

### 返回结果

| 参数 | 类型 | 说明  |
|-------|------|-------|
| successful | bool | 请求是否成功。  |
| timestamp | time.Time | 服务端完成请求的时间戳。  |
| duration | int | 服务端处理请求的时间（毫秒）。  |
| status | int | 符合 HTTP Status 规范的编码。  |
| traceId | string | 请求的 Trace ID。  |
| error | ApiError | 请求产生的 Error，包含如下信息：<ul><li><code>code</code>：错误码。</li><li><code>message</code>：错误详细信息。</li><li><code>subErrors</code>：子错误信息。</li></ul> |

## 示例

### 请求示例

POST 10.10.10.1:2886/api/v1/tenant/sys/user

```json
{
  "user_name": "proxyro",
  "password": "*****",
  "global_privileges": [
    "CREATE",
    "DELETE"
  ],
  "db_privileges": [
    {
      "db_name": "oceanbase",
      "privileges": [
        "DROP"
      ]
    }
  ],
  "host_name": "%"
}
```

### 返回示例

```json
{
    "successful": true,
    "timestamp": "2025-03-26T10:30:06.045600725+08:00",
    "duration": 76,
    "status": 200,
    "traceId": "cdb313dd77a2f56a"
}
```
