# 部署 OBProxy

本文介绍如何通过 obshell 部署 OBProxy（OceanBase 数据库代理，又称 ODP）。

<main id="notice" type='explain'>
  <h4>说明</h4>
  <ul>
  <li><p>目前仅支持部署 OBProxy V4.1.0.0 及之后版本。</p></li>
  <li><p>自 obshell V4.2.6 起支持该 API。</p></li>
  </ul>
</main>

## 调用说明

### 接口约束

* 该节点需已设置了 agent_password，详细介绍可参见 [设置节点密码](../100.obshell-management/1500.set-node-password.md)。
* obshell Server 会对该 API 进行安全校验，详细介绍可参见 [API 混合加密](../20.api-hybrid-encryption.md)。
* 要求该 obshell 并未管理任何 OBProxy 服务。
* 要求 OBProxy 安装包已解压安装到工作目录下。

### 请求路径

POST /api/v1/obproxy

### 请求参数

| 参数 | 类型 | 必选 | 示例值 | 描述 |
|------|------|------|-------|------|
| name | string | 否 | myobproxy | 部署的 OBProxy 服务名称。 |
| home_path | string | 是 | /home/admin/obproxy | OBProxy 的工作目录，需保证拥有该目录的读写权限。 |
| sql_port | int | 否 | 2883 | OBProxy SQL 端口。 |
| exporter_port | int | 否 | 2884 | OBProxy Exproter 端口。 |
| rpc_port | int | 否 | 2885 | OBProxy RPC 服务监听端口。 |
| proxyro_password | string | 否 | ******* | 所关联集群中 proxyro@sys 用户的密码。 |
| obproxy_sys_password | string | 否 | ******* | root@proxysys 密码。 |
| rs_list | string | 否 | 10.10.10.1:2881;10.10.10.2:2881 | 所关联集群的 Root Service 列表，配置后可使用 OBProxy 代理对应 OceanBase 集群，并且仅能代理这一个集群。 |
| config_url | string | 否 | http://10.10.10.1:8080/services?Action=GetObProxyConfig | obconfigserver 的 url，配置后可使用 OBproxy 代理 obconfigserver 中注册的所有集群。 |
| parameters | map[string]string | 否 | {</br>"server_tcp_keepidle": "5",</br>"enable_strict_kernel_release": "false"</br>} | 配置 OBProxy 的配置项。 |

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>请求参数中不可同时指定 <code>rs_list</code> 和 <code>config_url</code>，否则会报错。当 <code>rs_list</code> 和 <code>config_url</code> 均未指定时，若 obshell 节点已管理有 OceanBase 集群，部署的 OBProxy 会尝试自动关联该集群。</p>
</main>

### 返回结果

| 参数 | 类型 | 说明 |
|--------|-----|------|
| successful | bool | 请求是否成功。 |
| timestamp | time.Time | 服务端完成请求的时间戳。 |
| duration | int | 服务端处理请求的时间（毫秒）。 |
| status | int | 符合 HTTP Status 规范的编码。 |
| traceId | string | 请求的 Trace ID。 |
| data | DagDetailDTO | 详情参见下文 **DagDetailDTO 信息的数据结构**。 |
| error | ApiError | 请求产生的 Error，包含如下信息：<ul><li>`code`：错误码。</li><li>`message`：错误详细信息。</li><li>`subErrors`：子错误信息。</li></ul> |

DagDetailDTO 信息的数据结构如下：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| id | string | DAG 通用 ID。 |
| dag_id | int | DAG id，是记录在 OceanBase 数据库中的主键。 |
| name | string | DAG 名称。 |
| stage | int | DAG 的当前执行阶段。 |
| max_stage | int | DAG 执行过程中的总阶段数。 |
| state | string | DAG 执行状态。 |
| operator | string | DAG 执行操作类型。 |
| start_time | time.Time | DAG 执行起始时间。 |
| end_time | time.Time | DAG 执行结束时间。 |
| additional_data | map[string]any | DAG 其他数据。 |
| nodes | []NodeDetailDTO | DAG 内的所有 Node 信息，详见 [获取 Node 的详细信息](../1000.task-management/2100.get-node-detail.md)。 |

## 示例

### 请求示例

POST 10.10.10.1:2886/api/v1/obproxy

此处示例为：向 `10.10.10.1:2886` 节点发起部署 OBProxy 的请求，并且 OBProxy 以 rs_list 工作方式关联集群（Root Service 列表为 `10.10.10.1:2881;10.10.10.2:2881;10.10.10.3:2881`）。您可根据实际情况修改示例。

```json
{
    "home_path": "/data/obproxy",
    "name": "myobproxy",
    "sql_port": 2883,
    "exporter_port": 2884,
    "rpc_port": 2885,
    "rs_list": "10.10.10.1:2881;10.10.10.2:2881;10.10.10.3:2881",
    "proxyro_password": "********",
    "obproxy_sys_password": "********",
    "parameters": {
        "server_tcp_keepidle": "5",
        "enable_strict_kernel_release": "false"
    }
}
```
