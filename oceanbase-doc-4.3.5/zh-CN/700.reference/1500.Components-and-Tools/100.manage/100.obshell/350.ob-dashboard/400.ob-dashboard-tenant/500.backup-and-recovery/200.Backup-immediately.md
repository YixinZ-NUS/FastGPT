# 立即备份

## 前提条件

* 在进行备份前，请确认已完成 NFS 的部署，部署 NFS 相关操作请参见 [部署 NFS](https://www.oceanbase.com/docs/common-oceanbase-database-cn-1000000003976589)。

* 需提前创建数据存储目录和日志存储目录，并赋予相应读写权限，要求每个租户的备份路径需要配置为独立的空目录，不同租户不能配置相同的备份路径。

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>目前 obshell Dashboard 仅支持 NFS 备份介质类型。</p>
</main>

## 操作步骤

1. 登录 obshell Dashboard。

2. 在左侧导航栏中单击 **租户管理**。

3. 在 **租户管理** 页面单击 **租户列表** 中租户名。

4. 进入对应租户 **总览** 页面后单击左侧导航栏 **备份恢复** 进入 **备份恢复** 页面。

5. 点击页面右上方 **立即备份** 进入 **立即备份** 页面。

6. 填写或修改存储配置。

   * 若待备份的租户已有备份存储配置，则存储配置会自动填充，可通过编辑框进行修改。

   * 若待备份的租户无备份存储配置，则需手动填写存储配置。

7. 填写或修改完成后，单击 **测试并配置** ，测试并配置存储配置。

   <main id="notice" type='explain'>
     <h4>说明</h4>
     <p>单击 <b>测试并配置</b> 后，obshell 会创建对应的测试并配置存储配置的任务，可单击 <b>查看任务</b> 跳转查看任务。正常情况下，该任务在数秒内即可成功完成，如果失败，请前往任务中心查看并修正错误原因。待任务完成后才能单击 <b>立即备份</b> 执行备份。</p>
   </main>

   ![立即备份](https://obbusiness-private.oss-cn-shanghai.aliyuncs.com/doc/img/observer-enterprise/V4.4.1/obshell/V4.3.2/200.Backup-immediately.png)

8. 选择数据备份方式。

   <main id="notice" type='explain'>
     <h4>说明</h4>
     <p>物理备份下，增量备份之前必须有一次全量的数据备份。</p>
   </main>

9. 单击 **立即备份**，会生成备份任务，可单击 **查看任务** 跳转查看备份任务进度。
