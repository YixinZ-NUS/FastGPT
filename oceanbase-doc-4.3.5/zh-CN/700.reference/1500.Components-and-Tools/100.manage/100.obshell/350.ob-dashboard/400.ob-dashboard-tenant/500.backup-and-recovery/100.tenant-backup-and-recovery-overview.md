# 备份恢复概览

在左侧导航栏中单击 **租户管理**，在 **租户管理** 页面单击 **租户列表** 中租户名，进入对应租户 **总览** 页面后单击左侧导航栏 **备份恢复**，即可进入 **备份恢复** 页面，本文介绍 **备份恢复** 页面的信息及该页面可进行的操作。

您可在 **备份恢复** 页面查看当前 obshell Dashboard 备份恢复的基本信息、备份任务、恢复任务。并可跳转至对应页面进行发起恢复、立即备份操作。

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>不支持对系统租户（sys 租户）执行备份恢复操作。</p>
</main>

## 基本信息

您可在基本信息部分查看 **最近一次数据备份**、**日志备份位点**、**日志延时** 的数据，对当前备份恢复的时间位点信息进行了解。

![基本信息](https://obbusiness-private.oss-cn-shanghai.aliyuncs.com/doc/img/oceanbase-database-standalone/V4.3.5/obshell/V4.3.2/100.backup-and-recovery-overview-01.png)

## 备份任务

备份任务分为数据备份和日志备份。

### 数据备份

您可在 **数据备份** 页签查看数据的 **ID**、**数据备份方式**、**备份集 ID**、**开始时间**、**结束时间**、**状态** 和 **操作**，并根据不同的状态对备份任务进行筛选，详情请参见下表。您可以单击 **开始时间** 或 **结束时间** 后的排序图标，选择按照升序、降序方式展示数据备份信息；也可在该模块右上角选择查看近 24 小时、近 7 天、近 30 天的数据备份任务信息，或选择自定义时间，查看特定时间内的数据备份任务信息。

![备份任务](https://obbusiness-private.oss-cn-shanghai.aliyuncs.com/doc/img/oceanbase-database-standalone/V4.3.5/obshell/V4.3.2/100.tenant-backup-and-recovery-overview-06.png)

| 参数  | 说明  |
|-------|------|
| ID | 数据备份任务的唯一标识。单击 ID 左侧的 + 按钮，可展开查看 **备份方式**、**备份时长**、**数据版本** 及 **存储目录**，展开后再点击 ID 前的 - 按钮，即可取消展示。  |
| 数据备份方式 | 分为 **增量** 和 **全量** 两种方式。 |
| 备份集 ID | 备份集的 ID。  |
| 开始时间 | 备份开始时间点。  |
| 结束时间 | 备份结束时间点。  |
| 状态 | 数据备份状态。支持对备份状态进行筛选。 |
| 操作 | <ul><li>当数据备份任务失败时，可单击 **操作** 下的 **查看原因** 查看具体日志。</li><li>当任务进行中时，可单击 **操作** 下的 **取消**，取消数据备份任务。</li></ul> |

### 日志备份

您可在 **日志备份** 页签查看数据的 **备份方式**、**最新备份时间点**、**延时（s）**、**存储目录**、**开始时间**、**状态** 和 **操作**，详情请参见下表。您可以单击 **最新备份时间点** 或 **延时（s）** 后的排序图标，选择按照升序、降序方式展示数据备份信息；也可在该模块右上角选择查看近 24 小时、近 7 天、近 30 天的数据备份任务信息，或选择自定义时间，查看特定时间内的日志备份任务信息。

![日志备份](https://obbusiness-private.oss-cn-shanghai.aliyuncs.com/doc/img/oceanbase-database-standalone/V4.3.5/obshell/V4.3.2/100.backup-and-recovery-overview-03.png)

| 参数  | 说明  |
|-------|------|
| 备份方式 | 备份租户的备份方式，目前只支持 **物理备份**。  |
| 最新备份时间点 | 截至当前时间已完成的日志备份。  |
| 延时（s） | 备份数据与实际数据的时间差，单位为秒。  |
| 存储目录 | 备份存储目录。  |
| 开始时间 | 备份开始时间点。
| 状态 | 日志备份状态。  |
| 操作 | 可对 **进行中** 的租户进行 **停止** 和 **暂停** 操作，也可对 **暂停** 的租户进行 **恢复** 或 **停止** 操作。<main id="notice" type='explain'><h4>说明</h4><p>停止日志备份后，系统将不支持任意时间点恢复，无法保障数据安全，请谨慎操作。</p></main>|

## 恢复任务

您可在 **恢复任务** 页签查看任务的 **ID**、**备份集群**、**备份租户**、**备份方式**、**目标租户**、**恢复对象**、**开始时间**、**恢复时长**、**状态** 和 **操作**，并根据不同的状态对恢复任务进行查看。您可在该模块右上角选择查看近 24 小时、近 7 天、近 30 天的数据恢复任务信息，或选择自定义时间，查看特定时间内的数据恢复任务。

![恢复任务](https://obbusiness-private.oss-cn-shanghai.aliyuncs.com/doc/img/oceanbase-database-standalone/V4.3.5/obshell/V4.3.2/100.tenant-backup-and-recovery-overview-09.png)

| 参数  | 说明  |
|-------|------|
| ID | 恢复任务的唯一标识。单击 ID 左侧的 + 按钮，可展开查看 **恢复时间点**、**结束时间**、**日志存储目录** 及 **数据存储目录**，展开后再点击 ID 前的 - 按钮，即可取消展示。  |
| 备份集群 | 备份租户所在的集群。 |
| 备份租户 | 备份集群中已经完成备份的租户，可将该租户的数据恢复到同集群的其它租户中。|
| 备份方式 | 备份租户的备份方式，目前只支持物理备份。 |
| 目标租户 | 恢复数据的租户，将备份租户的数据恢复至目标租户。 |
| 恢复对象 | 恢复目标类型。 |
| 开始时间 | 恢复任务开始的时间。 |
| 恢复时长 | 恢复任务所用的时间。 |
| 状态 | 支持对任务状态进行筛选。物理备份的任务状态包括：**恢复中**、**创建租户**、**升级**、**恢复成功**、**恢复失败**、**等待租户恢复完成**、**已完成** 及 **失败**。 |
| 操作 | 可对状态为 **创建租户** 和 **等待租户恢复完成** 的租户进行 **取消** 操作，也可对状态为 **失败** 的租户进行 **查看原因** 操作。 |
