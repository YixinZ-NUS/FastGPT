# 查看告警事件

本章节为您介绍如何查看告警事件详情。

## 操作步骤

1. 登录 obshell Dashboard 控制台。

2. 在左侧导航栏中选择 **告警中心** 。

3. 单击 **告警事件** 页签。

4. 在查询框中设置查询条件，**告警事件** 列表中将返回所有符合查询条件的告警事件。

   * 支持对告警事件根据 **对象类型**、**告警对象**、**告警等级**、**关键词**、**开始时间**、**结束时间** 等条件进行精确查找。

     | 参数  | 说明  |
     |-------|------|
     |对象类型 | 对象类型分为 obcluster、obtenant、observer。<ul><li>obcluster 指告警对象是集群。</li><li>obtenant 指告警对象是租户。</li><li>observer 指告警对象是 observer 节点。</li></ul> |
     | 告警对象 | 根据 **对象类型** 的不同，可在此处下拉框中选择不同的告警对象。存在多个告警对象时，仅支持选择其中一个。<ul><li>当对象类型为 obcluster 时，告警对象是集群名。</li><li>当对象类型为 obtenant 时，告警对象是相应租户名。</li><li>当对象类型为 observer 时，告警对象是  observer 节点 IP。</li></ul> |
     | 告警等级 | 分为：严重、警告、注意、提醒 四个等级。<ul><li>严重：系统可用性下降，需要紧急修复，避免完全不可用， 如 `机器内存使用率大于 90% 持续 3 分钟`。</li><li>警告：系统仍然可用，但是即将达到不可用状态，需要采取措施防止可用性下降， 如 `OceanBase 租户连接数大于上限的 80%`。</li><li>注意：根据趋势判断，系统关键性能指标正在下降，但是还没达到触发警告的程度，此时可以通过排查发现潜在的问题，避免警告产生。</li><li>提醒：操作提醒，本质上不是告警，通常是管理员执行重要操作，如 `管理员执行集群下线操作`。</li></ul> |
     | 关键词 | 告警的关键字，可根据关键词检索对应告警事件。您可自定义输入。 |
     | 开始时间 | 指告警事件的开始时间。 |
     | 结束时间 | 指告警事件的结束时间。 |

   * 点击 **重置** 按钮，会将查询内容置空，可重新输入查询内容。

   * 点击 **收起** 按钮，将收起查询框，此时仅支持设置 **对象类型**、**告警对象** 和 **告警等级** 进行查询。

5. 支持对告警事件进行 **屏蔽** 操作。

   * 若告警事件无需关注，您可单击 **操作** 列的 **屏蔽** 对告警消息进行快速屏蔽。

     ![事件列表](https://obbusiness-private.oss-cn-shanghai.aliyuncs.com/doc/img/observer-enterprise/V4.4.1/obshell/V4.3.2/450.alarm-center/200.view-alarm-events-01.png)

   * 单击 **屏蔽** 后，页面会跳转到 **告警屏蔽** 页面，然后弹出 **屏蔽条件** 页面，并自动填充屏蔽对象类型、屏蔽对象、屏蔽告警规则、标签，您只需填写屏蔽结束时间和备注信息，单击 **提交**，即可屏蔽相应告警事件对应告警事件的告警状态将变为 **抑制**，且在设置的结束时间前，该告警不会被发送。具体配置说明可参考 [管理告警屏蔽](300.alert-shielding-management.md) 中的新建屏蔽。
