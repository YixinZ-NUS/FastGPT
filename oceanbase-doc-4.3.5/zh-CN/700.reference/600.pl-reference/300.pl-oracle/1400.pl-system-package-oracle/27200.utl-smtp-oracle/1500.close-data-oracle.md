# CLOSE_DATA

`CLOSE_DATA` 用于发送一个 `<CR><LF>.<CR><LF>` 标识邮件文本发送完成。

<main id="notice" >
    <h4>功能适用性</h4>
    <p>该内容仅适用于 OceanBase 数据库 Oracle 模式。</p>
  </main>

## 语法

```sql
UTL_SMTP.CLOSE_DATA (
   c     IN OUT NOCOPY connection)
RETURN reply;

UTL_SMTP.CLOSE_DATA (
   c     IN OUT NOCOPY connection);
```

## 参数解释

| **参数** |         **解释**         |
|--------|------------------------|
|c|SMTP 连接。|

## 返回值

| **返回值** |          **描述**          |
|---------|--------------------------|
|reply|command 的回复。如果有多个回复，则返回最后一个回复。|

## 使用说明

`OPEN_DATA`, `WRITE_DATA`, `WRITE_RAW_DATA` 和 `CLOSE_DATA` 需要严格按照顺序调用。`open_data` 发送 `data` 命令声明开始发送邮件内容，之后可以多个调用 `write_data` 和 `write_raw_data` 发送邮件正文，最后通过 `close_data` 发送终止符结束发送。
