# DATA

`DATA` 用于指定电子邮件正文。

<main id="notice" >
    <h4>功能适用性</h4>
    <p>该内容仅适用于 OceanBase 数据库 Oracle 模式。</p>
  </main>

## 语法

```sql
UTL_SMTP.DATA (
   c     IN OUT NOCOPY connection
   body  IN  VARCHAR2 CHARACTER SET ANY_CS)
RETURN reply;

UTL_SMTP.DATA (
   c     IN OUT NOCOPY connection
   body  IN VARCHAR2 CHARACTER SET ANY_CS);
```

## 参数解释

| **参数** |         **解释**         |
|--------|------------------------|
|c|SMTP 连接。|
|body|要发送的文本信息，包含headers，格式为[RFC822]。|

## 返回值

| **返回值** |          **描述**          |
|---------|--------------------------|
|reply|command 的回复。如果有多个回复，则返回最后一个回复。|

## 使用说明

* 应用程序必须确保 body 参数的内容符合 `MIME（RFC822）` 规范。使用 `<CR><LF>.<CR><LF>` 作为终止消息，文本内容中的 `<CR><LF>.<CR><LF>` 将会被转译为 `<CR><LF>..<CR><LF>` 需要在 `RCPT` 后调用。
* 预期返回是 `250` 开头的回复信息，初始 `data` 命令的返回码 `354` 不会被返回。

   * `354` 表示客户端发送了 `DATA` 命令给 SMTP 服务器并开始传输邮件正文，如果服务器准备好接收数据，则会回返回`354` 状态码。

   * `250` 表示请求被成功接收并处理。
