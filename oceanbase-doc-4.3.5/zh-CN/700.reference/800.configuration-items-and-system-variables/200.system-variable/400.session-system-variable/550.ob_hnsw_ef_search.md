# ob_hnsw_ef_search

<main id="notice" type='explain'>
<h4>说明</h4>
<p>该变量从 V4.3.3 版本开始引入，只支持 MySQL 租户。</p>
</main>

## 功能描述

`ob_hnsw_ef_search` 用于设置在进行向量索引查询时，HNSW 算法在每一层图中搜索的邻居点数量。该变量值越大，算法会探索更多的邻居点，增加找到真实最近邻的机会，从而提高搜索准确度，但同时搜索时间也会增加。

## 权限要求

* 查询变量

  `sys` 租户和所有用户租户均可以使用 `SHOW VARIABLES` 语句或视图 `information_schema.SESSION_VARIABLES`（MySQL 模式）查看 Session 系统变量的值。

* 修改变量

  `sys` 租户和所有 MySQL 用户租户均可以直接修改本租户的 Session 系统变量的值。

## 属性说明

| **属性**  | **描述**  |
|---------|---------|
| 参数类型    | Uint |
| 默认值     | 64    |
| 取值范围    | [1,160000] <main id="notice" type='explain'><h4>说明</h4><p>该变量从 V4.3.5 BP4 版本开始最大值从 1000 变更为 160000。</p></main>    |
| 生效范围    | Session  |
| 是否可修改 | 是，支持通过 `SET` 语句修改。 |

## 查询示例

设置在进行向量索引查询时，HNSW 算法在每一层图中搜索的邻居点数量为 50。

```shell
obclient> SET ob_hnsw_ef_search = 50;
```
