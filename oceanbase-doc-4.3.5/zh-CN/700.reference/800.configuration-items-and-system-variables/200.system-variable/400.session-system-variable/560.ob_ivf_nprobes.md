# ob_ivf_nprobes

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该变量从 V4.3.5 BP1 版本开始引入。</p>
</main>

## 功能描述

`ob_ivf_nprobes` 用于设置在进行向量索引查询时，本次查询搜索多少个最近的聚类中心。该变量影响 IVF 类查询性能和召回率：值越大，召回率越高，但同时搜索时间也会增加。

<main id="notice" type='explain'>
  <h4>功能适用性</h4>
  <p>该变量仅适用于 OceanBase 数据库 MySQL 模式。</p>
</main>

## 权限要求

* 查询变量

  `sys` 租户和所有用户租户均可以使用 `SHOW VARIABLES` 语句或视图 `information_schema.SESSION_VARIABLES`（MySQL 模式）查看 Session 系统变量的值。

* 修改变量

  `sys` 租户和所有 MySQL 用户租户均可以直接修改本租户的 Session 系统变量的值。

## 属性说明

| **属性**  | **描述**  |
|---------|---------|
| 参数类型    | Int |
| 默认值     | 8    |
| 取值范围    | [1,65536]    |
| 生效范围    | Session  |
| 是否可修改 | 是，支持通过 `SET` 语句修改。 |

## 配置示例

设置本次查询搜索 10 个最近的聚类中心。

```shell
obclient> SET ob_ivf_nprobes = 10;
```
