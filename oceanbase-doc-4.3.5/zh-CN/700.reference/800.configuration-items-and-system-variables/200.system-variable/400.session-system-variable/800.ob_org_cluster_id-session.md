# ob_org_cluster_id

<main id="notice" type='explain'>
<h4>说明</h4>
<p>该变量从 V1.4 版本开始引入。</p>
</main>

## 功能描述

`ob_org_cluster_id` 用于设置 OceanBase 数据库 Session 级别的 `cluster_id`，该值会持久化到事务日志中。

## 权限要求

* 查询变量

  `sys` 租户和所有用户租户均可以使用 `SHOW VARIABLES` 语句或视图 `SYS.TENANT_VIRTUAL_SESSION_VARIABLE`（Oracle 模式）及视图 `information_schema.SESSION_VARIABLES`（MySQL 模式）查看 Session 系统变量的值。

* 修改变量

  `sys` 租户和所有用户租户均可以直接修改本租户的 Session 系统变量的值。

## 属性说明

| **属性**  |      **描述**       |
|---------|-------------------|
| 参数类型    | Int               |
| 默认值     | 0，表示使用集群自身的 `cluster_id`，即集群创建时生成的唯一标识符    |
| 取值范围    | \[0, 4294967295\] |
| 生效范围    | Session          |
| 是否可修改  |  是，支持通过 `SET` 语句修改。 |

## 使用说明

通常情况下不需要设置 `ob_org_cluster_id` 变量，OceanBase 数据库生成日志（ clog ）时会使用缺省值。OMS 可以通过变量 `ob_org_cluster_id` 实现数据过滤功能。有关 OMS 数据过滤功能的详细介绍和配置，参见《OceanBase 迁移服务》中的 **创建 MySQL 至 OceanBase 数据库 MySQL 租户的数据迁移项目**。

例如：OMS obcdc 组件设置了 `cluster_id` 的黑名单为 `cluster_id_black_list=2147473648`，并且 OBServer 节点写入时设置 Session 变量为 `SET ob_org_cluster_id=2147473648;`。那么 obcdc 收到数据后，会过滤所有 `cluster_id` 等于 `2147473648` 的数据。

## 配置示例

设置 OceanBase 数据库 Session 级别的 `cluster_id` 为 `2147473648`。

```shell
obclient> SET ob_org_cluster_id = 2147473648;
```
