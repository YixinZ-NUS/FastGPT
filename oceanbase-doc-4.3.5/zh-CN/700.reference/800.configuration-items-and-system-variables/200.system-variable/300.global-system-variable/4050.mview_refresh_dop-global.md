# mview_refresh_dop

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>对于 OceanBase 数据库 V4.3.5 版本，该变量从 V4.3.5 BP1 版本开始引入。</p>
</main>

## 功能描述

`mview_refresh_dop` 用于控制物化视图的默认刷新并行度。

## 权限要求

* 查询变量

  * Global 级别

    `sys` 租户和所有用户租户均可以使用 `SHOW VARIABLES` 语句或视图 `information_schema.GLOBAL_VARIABLES`（MySQL 模式）查看 Global 系统变量的值。

  * Session 级别

    `sys` 租户和所有用户租户均可以使用 `SHOW VARIABLES` 语句或视图 `information_schema.SESSION_VARIABLES`（MySQL 模式）查看 Session 系统变量的值。

* 修改变量

  * Global 生效

    * `sys` 租户可以直接修改 Global 系统变量的值。

    * MySQL 用户租户需要拥有 `SUPER` 或 `ALTER SYSTEM` 权限才能修改 Global 系统变量的值。

  * Session 生效

    `sys` 租户和所有用户租户均可以直接修改本租户的 Session 系统变量的值。

## 属性说明

| **属性**  | **描述** |
|-----------|----------|
| 参数类型   | Int |
| 默认值     | 4 |
| 取值范围   | [0, uint64_max] |
| 生效范围   | <ul><li>Global</li><li>Session</li></ul>|
| 生效模式   | 立即生效 |
| 是否可修改 | 是，支持通过 `SET` 语句修改 |

## 使用说明

1. Session（会话）级别设置：

   在当前会话中设置 `mview_refresh_dop`，用于控制该会话下物化视图的默认刷新并行度。后续在此会话中发起的物化视图手动刷新，若未显式指定并行度，将使用此值。

2. Global（全局）级别设置：

   设置全局的 `mview_refresh_dop`，影响所有会话，包括手动和自动的物化视图刷新。后续所有会话在发起物化视图刷新时，若未显式指定并行度，将使用此值。

3. 当设置 `mview_refresh_dop` 为 0 或 1 时，表示不使用并行刷新能力。

## 使用示例

* 设置当前 Session 默认刷新并行度。

  ```shell
  obclient> SET mview_refresh_dop = 5;
  ```

* 设置全局默认刷新并行度。

  ```shell
  obclient> SET GLOBAL mview_refresh_dop = 5;
  ```
