# ddl_thread_score

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V4.2.1 版本开始引入。</p>
</main>

## 功能描述

`ddl_thread_score` 用于设置 DDL 任务使用的线程数。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  `sys` 租户和用户租户都可修改该配置项。

## 属性说明

|      **属性**      |   **描述**   |
|--------------------|--------------|
| 参数类型             | Int         |
| 默认值               | 0，表示默认线程数为 2        |
| 取值范围             | [0, 100] |
| 是否可修改           | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。     |

## 使用说明

* 该参数可用于控制 DDL 删列执行的并行度以及 **表级恢复跨租户导表阶段** 的并行度。

  * 有关表级恢复流程的详细介绍，参见 [恢复流程](../../../100.oceanbase-database-concepts/1000.high-data-reliability-and-availability/500.backup-and-recovery/400.recovery-architecture.md)。

  * 有关如何使用 `ddl_thread_score` 来控制删列执行的并行度的相关操作，参见 [重整数据 DDL 的执行速度优化](https://www.oceanbase.com/knowledge-base/oceanbase-database-1000000000217849?back=kb)

* 如果您发现低优先级的任务运行速度过慢，可以适当的提高该参数值。建议每次将数值翻倍。

## 配置示例

设置本租户下 DDL 任务的线程数为 4。

```shell
obclient> ALTER SYSTEM SET ddl_thread_score = 4;
```
