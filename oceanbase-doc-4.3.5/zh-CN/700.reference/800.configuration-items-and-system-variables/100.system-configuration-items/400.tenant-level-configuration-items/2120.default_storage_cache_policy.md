# default_storage_cache_policy

<main id="notice" type='explain'>
<h4>说明</h4>
<p>对于 V4.3.5 版本，该配置项从 V4.3.5 BP2 版本开始引入。</p>
</main>

## 功能描述

`default_storage_cache_policy` 用于配置默认的缓存策略，租户级 `storage_cache_policy` 的默认值，仅对新创建的表生效。

<main id="notice">
<h4>功能适用性</h4>
<p>该配置项仅适用于 OceanBase 数据库共享存储（Shared-Storage，SS）模式。</p>
</main>

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  `sys` 租户和用户租户均可修改该配置项。

  <main id="notice" type='notice'>
  <h4>注意</h4>
  <p>由于手动指定热数据缓存策略的功能仅在 MySQL 模式的用户租户下生效，Oracle 模式的用户租户修改该配置项，或者 <code>sys</code> 租户通过指定 Oracle 模式的用户租户或其他非用户租户（<code>sys</code> 租户 和 Meta 租户）修改该配置项时，配置项值可以修改成功，但功能不会生效。</p>
  </main>

## 属性说明

| **属性** | **描述** |
|----------|----------|
| 参数类型  | String  |
| 默认值    | AUTO |
| 取值范围   | <ul><li>`HOT`：指定表中所有数据作为热点数据缓存到本地磁盘。</li> <li> `AUTO`：指定表中所有数据为自动冷热识别。即由共享存储自行决定表中数据的存储方式。</li></ul> |
| 是否可修改 | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。 |

## 使用说明

* 当前租户下，未通过参数 `storage_cache_policy` 显式指定缓存策略的所有新建表默认使用 `default_storage_cache_policy` 中指定的缓存策略。
* 仅用户表和索引受配置项 `default_storage_cache_policy` 的影响，系统表不受配置项 `default_storage_cache_policy` 的影响，始终为 `AUTO` 配置。

## 配置示例

* 用户租户下，设置本租户默认的缓存策略为 `HOT`。

  ```shell
  obclient> ALTER SYSTEM SET default_storage_cache_policy='HOT';
  ```

* 系统租户下，指定租户 `mysqltenant` 默认的缓存策略为 `HOT`。

  ```shell
  obclient> ALTER SYSTEM SET default_storage_cache_policy='HOT' TENANT = mysqltenant;
  ```
