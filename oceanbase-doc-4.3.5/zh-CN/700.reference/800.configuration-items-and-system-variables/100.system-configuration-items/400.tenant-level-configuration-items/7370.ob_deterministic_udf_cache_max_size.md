# ob_deterministic_udf_cache_max_size

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>对于 V4.3.x 版本，该配置项从 V4.3.5 BP4 版本开始引入。</p>
</main>

## 功能描述

`ob_deterministic_udf_cache_max_size` 用于标记语句执行期，所有 `deterministic` 的 UDF 的缓存池总共可以使用的最大内存量(以字节为单位)。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  `sys` 租户和用户租户都可修改该配置项。

## 属性说明

| **属性**  |    **描述**  |
|-----------|---------------|
| 参数类型    |    Capacity        |
| 默认值      |  16M  |
| 取值范围    |    [0,+∞)<main id="notice" type='explain'><h4>说明</h4><p>如果为 0，则表示关闭执行期缓存功能。</p></main>       |
| 是否可修改 | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。 |

## 配置示例

* 执行如下命令关闭执行期缓存功能。

    ```shell
    obclient> ALTER SYSTEM SET ob_deterministic_udf_cache_max_size='0B';
    ```

* 执行如下命令将缓存池总共可以使用的最大内存量设置为 16M。

    ```shell
    obclient> ALTER SYSTEM SET ob_deterministic_udf_cache_max_size='16M';
    ```
