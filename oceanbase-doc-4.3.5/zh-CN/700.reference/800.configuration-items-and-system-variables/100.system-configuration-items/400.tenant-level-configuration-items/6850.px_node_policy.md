# px_node_policy

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V4.3.5 BP1 版本引入。</p>
</main>

## 功能描述

`px_node_policy` 用于支持 PX 计算节点资源的独立伸缩，表示计算节点的候选池。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  `sys` 租户和用户租户均可修改该配置项。

## 属性说明

|属性|描述|
|-----|----|
|参数类型              |String|
|默认值                |DATA|
|取值范围              |<ul><li>`DATA`：PX 计算节点可以选择当前 SQL 涉及的所有数据节点。</li> <li>`ZONE`：PX 计算节点可以选择当前 SQL 涉及的 zone 中属于该租户的所有节点。</li> <li>`CLUSTER`：PX 计算节点可以选择集群中当前租户涉及的所有节点。</li></ul> |
| 是否可修改           | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
|是否重启 OBServer 节点生效|否，设置后立即生效。|

## 配置示例

`sys` 租户指定 `mysql` 租户的 PX 计算节点可选择当前 SQL 涉及的所有数据节点。

```shell
obclient> ALTER SYSTEM SET px_node_policy = "DATA" tenant = 'mysql';
```
