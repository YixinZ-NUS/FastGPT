# dump_data_dictionary_to_log_interval

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V4.1.0 版本开始引入。</p>
</main>

## 功能描述

`dump_data_dictionary_to_log_interval` 用于控制租户数据字典持久化的周期。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  `sys` 租户和用户租户均可修改该配置项。

## 属性说明

| **属性** | **描述** |
| -------- | -------- |
| 参数类型   | Time |
| 默认值     | 24H，默认单位为秒 |
| 取值范围   | (0s, +∞)|
| 是否可修改 | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBSever 节点生效 | 否，设置后立即生效。 |

## 使用说明

数据字典仅服务于用户租户，通过 PALF 持久化在普通租户的系统日志流（SYS_LS）中。用户租户建立后数据字典服务自动开启，运行在租户 SYS_LS 的 LEADER 副本上。

日志归档服务开启时，会自动触发一次异步数据字典生成任务。OceanBase 数据库支持通过租户级配置项 `dump_data_dictionary_to_log_interval` 生成数据字典。

OceanBase 数据库自 V4.2.0 版本起支持通过 DBA/CDB 视图来记录数据字典的生成记录，可在系统租户下查询 [CDB_OB_DATA_DICTIONARY_IN_LOG](../../../../700.reference/700.system-views/300.system-view-of-sys-tenant/200.dictionary-view-of-sys-tenant/8200.o-cdb_ob_data_dictionary_in_log-of-sys-tenant.md) 视图，或普通租户下查询 [DBA_OB_DATA_DICTIONARY_IN_LOG](../../../../700.reference/700.system-views/400.system-view-of-mysql-mode/200.dictionary-view-of-mysql-mode/8300.o-dba_ob_data_dictionary_in_log-of-mysql-mode.md) 视图获取字典位置信息。

## 配置示例

设置租户数据字典持久化的周期为 24H。

```shell
obclient> ALTER SYSTEM SET dump_data_dictionary_to_log_interval = '24H';
```
