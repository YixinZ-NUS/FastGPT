# suspend_storage_cache_task

<main id="notice" type='explain'>
<h4>说明</h4>
<p>对于 V4.3.5 版本，该配置项从 V4.3.5 BP2 版本开始引入。</p>
</main>

## 功能描述

`suspend_storage_cache_task` 用于设置是否暂停后台缓存任务。

<main id="notice">
<h4>功能适用性</h4>
<p>该配置项仅适用于 OceanBase 数据库共享存储（Shared-Storage，SS）模式。</p>
</main>

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  `sys` 租户和用户租户均可修改该配置项。

  <main id="notice" type='notice'>
  <h4>注意</h4>
  <p>由于手动指定热数据缓存策略的功能仅在 MySQL 模式的用户租户下生效，Oracle 模式的用户租户修改该配置项，或者 <code>sys</code> 租户通过指定 Oracle 模式的用户租户或其他非用户租户（<code>sys</code> 租户 和 Meta 租户）修改该配置项时，配置项值可以修改成功，但功能不会生效。</p>
  </main>

## 属性说明

| **属性** | **描述** |
|----------|----------|
| 参数类型  | Bool  |
| 默认值    | False |
| 取值范围   | <ul><li>`True`：暂停当前正在进行以及新发起的缓存任务。</li> <li> `False`：不暂停后台的缓存任务，即正常进行缓存任务。</li></ul> |
| 是否可修改 | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。 |

## 配置示例

* 用户租户

  1. 设置暂停本租户下的缓存任务。

     ```shell
     obclient> ALTER SYSTEM SET suspend_storage_cache_task = True;
     ```

  2. 设置正常推进本租户下的缓存任务。

     ```shell
     obclient> ALTER SYSTEM SET suspend_storage_cache_task = False;
     ```

* 系统租户

  1. 设置暂停租户 `mysqltenant` 下的缓存任务。

     ```shell
     obclient> ALTER SYSTEM SET suspend_storage_cache_task = True TENANT = mysqltenant;
     ```

  2. 设置正常推进租户 `mysqltenant` 下的缓存任务。

     ```shell
     obclient> ALTER SYSTEM SET suspend_storage_cache_task = False TENANT = mysqltenant;
     ```
