# enable_manual_storage_cache_policy

<main id="notice" type='explain'>
<h4>说明</h4>
<p>对于 V4.3.5 版本，该配置项从 V4.3.5 BP2 版本开始引入。</p>
</main>

## 功能描述

`enable_manual_storage_cache_policy` 用于启用用户手动指定的热点缓存策略。

<main id="notice">
<h4>功能适用性</h4>
<p>该配置项仅适用于 OceanBase 数据库共享存储（Shared-Storage，SS）模式。</p>
</main>

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  `sys` 租户和用户租户均可修改该配置项。

  <main id="notice" type='notice'>
  <h4>注意</h4>
  <p>由于手动指定热数据缓存策略的功能仅在 MySQL 模式的用户租户下生效，Oracle 模式的用户租户修改该配置项，或者 <code>sys</code> 租户通过指定 Oracle 模式的用户租户或其他非用户租户（<code>sys</code> 租户 和 Meta 租户）修改该配置项时，配置项值可以修改成功，但功能不会生效。</p>
  </main>

## 属性说明

| **属性** | **描述** |
|----------|----------|
| 参数类型  | Bool  |
| 默认值    | True |
| 取值范围   | <ul><li>`True`：启用所有用户手动指定的热点缓存策略，系统采用表/分区/索引中指定的 `storage_cache_policy` 参数的值。</li> <li> `False`：禁用所有用户手动指定的热点缓存策略，系统不使用表/分区/索引中指定的 `storage_cache_policy` 参数的值，全部表/分区/索引的热点缓存策略均使用配置项 `default_storage_cache_policy` 的值。</li></ul> |
| 是否可修改 | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。 |

## 配置示例

* 用户租户下，启用用户手动指定的热点缓存策略。

  ```shell
  obclient> ALTER SYSTEM SET enable_manual_storage_cache_policy= True;
  ```

* 系统租户下，为租户 `mysqltenant` 启用用户手动指定的热点缓存策略。

  ```shell
  obclient> ALTER SYSTEM SET enable_manual_storage_cache_policy= True TENANT = mysqltenant;
  ```
