# default_load_mode

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V4.3.4 版本开始引入。</p>
</main>

## 功能描述

`default_load_mode` 用于在导数操作场景中，控制数据的导入方式。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  `sys` 租户和用户租户均可修改该配置项。

## 属性说明

| **属性** | **描述** |
| -------- | -------- |
| 参数类型  | String|
| 默认值    | DISABLED |
| 取值范围  | <ul><li>DISABLED：不使用旁路导入 </li><li>FULL_DIRECT_WRITE：使用全量旁路导入，以 SQL 的语义为准 </li><li>INC_DIRECT_WRITE：使用增量旁路导入的 INC 模式 </li><li>INC_REPLACE_DIRECT_WRITE：使用增量旁路导入的 INC_REPLACE 模式 </li></ul>|
| 是否可修改| 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。 |

## 使用说明

当通过 `LOAD DATA INFILE` 语句、`INSERT INTO SELECT` 语句以及 `CREATE TABLE AS SELECT` 语句导入数据，且没有指定旁路导入的相关 Hint 时，数据的默认导入方式由配置项  `default_load_mode` 控制，如果导数操作指定了 `APPEND/DIRECT/NO_DIRECT` Hint 时，数据的导数方式由 Hint 控制。

## 配置示例

* 在导数操作场景中，设置数据的导入方式为 `FULL_DIRECT_WRITE` 模式。

  ```shell
  obclient [test]> ALTER SYSTEM SET default_load_mode ='FULL_DIRECT_WRITE';
  ```

* 在导数操作场景中，设置数据的导入方式为 `INC_DIRECT_WRITE` 模式。

  ```shell
  obclient [test]> ALTER SYSTEM SET default_load_mode ='INC_DIRECT_WRITE';
  ```
