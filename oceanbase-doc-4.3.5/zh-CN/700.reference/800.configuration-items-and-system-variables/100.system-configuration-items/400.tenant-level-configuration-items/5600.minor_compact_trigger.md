# minor_compact_trigger

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V2.2.30 版本开始引入。</p>
</main>

## 功能描述

`minor_compact_trigger` 用于配置触发转储（Minor Compaction）的 SSTable 个数的阈值。

当 Mini SSTable 和 Minor SSTable 的个数总和超过配置项 `minor_compact_trigger` 所指定的值时，系统后台会尝试触发 Minor Compaction。但最终是否触发 Minor Compaction 还需要由系统内部其他逻辑来决定。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  `sys` 租户和用户租户均可修改该配置项。

## 属性说明

|      **属性**      |  **描述**   |
|------------------|-----------|
| 参数类型             | Int        |
| 默认值               | 2         |
| 取值范围             | \[0, 16\] |
| 是否可修改 | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。   |

## 配置示例

设置当 Mini SSTable 和 Minor SSTable 的个数总和超过 10 个时，尝试触发 Minor Compaction。

```shell
obclient> ALTER SYSTEM SET minor_compact_trigger = 10;
```
