# default_skip_index_level

<main id="notice" type='explain'>
<h4>说明</h4>
<p>对于 V4.3.5 版本，该配置项从 V4.3.5 BP5 版本开始引入。</p>
</main>

## 功能描述

`default_skip_index_level` 用于控制租户下增量 SSTable 生成 Skip Index 的机制。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  `sys` 租户和用户租户均可修改该配置项。

## 属性说明

| **属性** | **描述** |
|----------|----------|
| 参数类型  | INT |
| 默认值    | 0 |
| 取值范围   | <ul><li>0：表示增量 SSTable 不生成 Skip Index。</li> <li> 1：表示增量 SSTable 生成 Skip Index 的列与基线包含 Skip Index 的列保持一致，并且只对 `merge_engine = 'delete_insert'` 的表生效。</li></ul>|
| 是否可修改 | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。|

## 使用说明

配置项 `default_skip_index_level` 只会影响用户租户创建的表。

* `default_skip_index_level = 0` 表示增量 SSTable 不生成 Skip Index。
* `default_skip_index_level = 1` 表示增量 SSTable 生成 Skip Index 的列与基线包含 Skip Index 的列保持一致，并且只对 `merge_engine = 'delete_insert'` 的表生效。

## 配置示例

* MySQL 模式：设置用户租户创建表的 `skip_index_level` 默认为 1。

    ```shell
    obclient> ALTER SYSTEM SET default_skip_index_level = 1;
    ```

* Oracle 模式：设置用户租户创建表的 `skip_index_level` 默认为 1。

    ```shell
    obclient> ALTER SYSTEM SET "default_skip_index_level" = 1;
    ```
