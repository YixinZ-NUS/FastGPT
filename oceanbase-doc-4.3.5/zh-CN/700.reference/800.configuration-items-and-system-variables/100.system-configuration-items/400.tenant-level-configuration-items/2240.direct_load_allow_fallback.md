# direct_load_allow_fallback

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V4.3.4 版本开始引入。</p>
</main>

## 功能描述

`direct_load_allow_fallback` 用于控制在指定旁路导入的导数操作中，如果遇到旁路导入不支持的场景时，导数操作是否回退为普通导入的方式。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  `sys` 租户和用户租户均可修改该配置项。

## 属性说明

| **属性** | **描述** |
| -------- | -------- |
| 参数类型  | Bool |
| 默认值    | True <main id="notice" type='explain'><h4>说明</h4><p>该配置项从 V4.3.5 版本开始，默认值由 False 调整为 True。</p></main>|
| 取值范围  |<ul><li>False：旁路导入不支持时不允许回退为普通导入方式，导数操作报错。 </li><li>True：旁路导入不支持时允许回退为普通导入方式，导数操作不报错。 </li></ul> |
| 是否可修改 | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。 |

## 使用说明

* 不论是手动添加 Hint 控制数据导入方式为旁路导入，还是通过配置项 [default_load_mode](2050.default_load_mode.md) 全局配置数据导入方式为旁路导入，都会根据配置项 `direct_load_allow_fallback` 取值来判断旁路导入不支持时的行为。

## 配置示例

设置对于旁路导入不支持的场景进行导入操作时不报错，且允许回退为普通导入方式。

```shell
obclient> ALTER SYSTEM SET direct_load_allow_fallback = True;
```
