# dead_socket_detection_timeout

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V2.2.50 版本开始引入。</p>
</main>

## 功能描述

`dead_socket_detection_timeout` 用于设置检测失效 Socket 的时间间隔。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  仅 `sys` 租户可以修改该配置项，用户租户不能修改该配置项。

## 属性说明

|      **属性**    |   **描述**  |
|------------------|-------------|
| 参数类型             | Time       |
| 默认值               | 3s <main id="notice" type='explain'><h4>说明</h4><p>配置项从 V4.0.0 版本开始默认值由 10s 调整为 3s。</p></main>       |
| 取值范围             | [0s, 2h] |
| 是否可修改           | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。  |

## 使用说明

* 当 `dead_socket_detection_timeout` 的值为 0 时，表示禁用 Socket 级故障检测。

* 当 `dead_socket_detection_timeout` 的值大于 0 时，表示启用 Socket 级故障检测，且当发送端有数据超过设置的时间还未收到对方的应答，则认为当前 Socket 不可用。

## 配置示例

设置检测失效 Socket 的时间间隔为 3s。

```shell
obclient> ALTER SYSTEM SET dead_socket_detection_timeout= '3s';
```
