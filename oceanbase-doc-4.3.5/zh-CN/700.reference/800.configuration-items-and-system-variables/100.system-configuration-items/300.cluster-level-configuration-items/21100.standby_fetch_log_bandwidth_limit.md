# standby_fetch_log_bandwidth_limit

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V4.2.0 版本开始引入。</p>
</main>

## 功能描述

`standby_fetch_log_bandwidth_limit` 用于设置备租户所在集群中，可用于备租户从主租户或源端租户进行日志同步的所有带宽的总和。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  仅 `sys` 租户可以修改该配置项，用户租户不能修改该配置项。

## 属性说明

| **属性** | **描述** |
| -------- | -------- |
| 参数类型    | Capacity |
| 默认值      | 0MB，表示不对流量进行限制|
| 取值范围    | [0, 10000G] |
| 是否可修改  | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。 |

## 使用说明

设置该配置项时，必须带有单位（B/KB/MB/GB等），单位忽略大小写。

## 配置示例

设置备租户所在集群中，可用于备租户从主租户或源端租户进行日志同步的所有带宽的总和为 200M。

```shell
obclient> ALTER SYSTEM SET standby_fetch_log_bandwidth_limit = '200M';
```
