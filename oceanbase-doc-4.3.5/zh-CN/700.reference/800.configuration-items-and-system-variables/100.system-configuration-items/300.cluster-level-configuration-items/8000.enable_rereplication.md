# enable_rereplication

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V1.4 版本开始引入。</p>
</main>

## 功能描述

`enable_rereplication`是日志流副本自动迁移复制的总开关。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  仅 `sys` 租户可以修改该配置项，用户租户不能修改该配置项。

## 属性说明

|      **属性**    |   **描述**   |
|------------------|--------------|
| 参数类型             | Bool                |
| 默认值               | True                |
| 取值范围             | <li> True：开启   <li> False：关闭    |
| 是否可修改           | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。   |

## 使用说明

当关闭配置项 `enable_rereplication` （值为 `False`）时：

* 以下场景引起的日志流迁移复制将会无法发起：

  * 通过 Locality 变更来增加或减少副本

  * Unit 迁移（源端 OBServer 节点为 `INACTIVE` 状态的除外）

  * `UNIT_NUM` 变更引起的日志流迁移

* 以下场景仍然可以正常发起日志流的迁移复制：

  * 源端 OBServer 节点为 `INACTIVE` 状态的 Unit 的迁移

## 配置示例

开启日志流副本的自动迁移复制。

```shell
obclient> ALTER SYSTEM SET enable_rereplication = True;
```
