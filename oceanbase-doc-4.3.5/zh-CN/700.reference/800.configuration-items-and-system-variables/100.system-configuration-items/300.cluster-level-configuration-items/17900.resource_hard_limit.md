# resource_hard_limit

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V2.2.50 版本开始引入。</p>
</main>

## 功能描述

`resource_hard_limit` 用于定义 CPU 资源的超卖百分比，大于 100 表示允许超卖。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  仅 `sys` 租户可以修改该配置项，用户租户不能修改该配置项。

## 属性说明

|      **属性**      |    **描述**    |
|------------------|--------------|
| 参数类型             | Int             |
| 默认值               | 100            |
| 取值范围             | [100, 10000] |
| 是否可修改           | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。   |

## 使用说明

* OceanBase 数据库从 V4.0.0.0 版本开始，不再支持通过配置项 `resource_hard_limit` 进行内存资源的超卖。

* `resource_hard_limit` 用于调整 OBServer 节点的 `CPU_CAPACITY` 的超卖比例。在 OBServer 节点上，所有租户 Unit 的 `MAX_CPU` 的总和不能超过 CPU 总容量的超卖值：`CPU_CAPACITY * resource_hard_limit`。

* 如果设置了 `resource_hard_limit >100`，超卖生效后，只会影响对应资源单元的资源（CPU）上限，不影响其资源下限。但使用超卖机制需要格外小心，确认负载可以错峰占用资源，避免因负载‘过载’叠加资源超卖后导致的系统问题。

## 配置示例

设置 CPU 资源的超卖百分比为 100。

```shell
obclient> ALTER SYSTEM SET resource_hard_limit = 100;
```
