# px_task_size

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V2.2.50 版本开始引入。</p>
</main>

## 功能描述

`px_task_size` 用于设置 SQL 并行查询引擎每个任务处理的数据量大小，无默认单位，配置时必须指定单位。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  仅 `sys` 租户可以修改该配置项，用户租户不能修改该配置项。

## 属性说明

| **属性**  |    **描述**  |
|-----------|---------------|
| 参数类型    |   Capacity              |
| 默认值      |  2M  |
| 取值范围    |  [1K, +∞)   <main id="notice" type='explain'><h4>说明</h4><p>该配置项从 V4.3.5 版本开始最小取值改为 1K。</p></main>        |
| 是否可修改 | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。 |

## 配置示例

执行如下命令，将 SQL 并行查询引擎每个任务处理的数据量大小设置为 4K:

```shell
obclient> ALTER SYSTEM SET px_task_size='4k';
```
