# ob_startup_mode

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V4.1.0 版本开始引入。</p>
</main>

## 功能描述

`ob_startup_mode` 用于设置 OBServer 节点的启动模式，仅支持在第一次启动时修改。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  仅支持在集群第一次启动时修改。

## 属性说明

| **属性** | **描述** |
| -------- | -------- |
| 参数类型 | String |
| 默认值 | NORMAL |
| 取值范围 |<ul><li> NORMAL：表示以正常模式启动  </li><li>ARBITRATION：表示以仲裁模式启动 </li><li>SHARED_STORAGE：表示以共享存储（Shared-Storage，SS）模式启动 </li></ul><main id="notice" type='explain'><h4>说明</h4><p>`SHARED_STORAGE` 值从 V4.3.4 版本开始引入。</p></main>|
| 是否可修改 | 否，不支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，仅支持只读。 |

## 配置示例

集群第一次启动时，指定 OBServer 节点的启动模式为 `SHARED_STORAGE`。如果不指定，则默认以 SN 模式启动。

```shell
-bash-4.2$ cd /home/admin/oceanbase && /home/admin/oceanbase/bin/observer -I 10.10.10.1 -P 2882 -p 2881 -z zone1 -n obdemo -c 10001 -o "system_memory=400G,ob_startup_mode=shared_storage" -d /home/admin/oceanbase/store/obdemo -s "oss://cn-hangzhou-bucket-1?host=oss-cn-hangzhou-****.aliyuncs.com&ram_url=***&max_iops=10000&max_bandwidth=1GB"
```
