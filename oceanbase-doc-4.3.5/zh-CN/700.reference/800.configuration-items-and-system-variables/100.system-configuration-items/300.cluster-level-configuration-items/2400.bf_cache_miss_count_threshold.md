# bf_cache_miss_count_threshold

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V1.4 版本开始引入。</p>
</main>

## 功能描述

`bf_cache_miss_count_threshold` 用于控制 BloomFilter Cache 的触发阈值，当宏块未命中次数达到该值时，创建 BloomFilter 缓存。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句查询该配置项。

* 修改配置项

  仅 `sys` 租户可以修改该配置项，用户租户不能修改该配置项。

## 属性说明

|      **属性**    |  **描述**  |
|------------------|------------|
| 参数类型             | Int       |
| 默认值               | 100      |
| 取值范围             | [0, +∞) <br>取值为 0 时，表示关闭 BloomFilter 缓存的创建。|
| 是否可修改           | 是，支持通过 `ALTER SYSTEM SET` 语句修改。    |
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。   |

## 配置示例

设置宏块未命中次数为 100 时创建 BloomFilter Cache。

```shell
obclient> ALTER SYSTEM SET bf_cache_miss_count_threshold=100;
```
