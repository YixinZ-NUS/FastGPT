# data_storage_error_tolerance_time

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V2.2.77 版本开始引入。</p>
</main>

## 功能描述

`data_storage_error_tolerance_time` 用于控制数据盘状态设为 `ERROR` 的容忍时间。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  仅 `sys` 租户可以修改该配置项，用户租户不能修改该配置项。

## 属性说明

|        属性       |       描述       |
|------------------|----------------|
| 参数类型             | Time           |
| 默认值               | 300s           |
| 取值范围             | [10s, 7200s] |
| 是否可修改           | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。  |

## 使用说明

当出现非预期的 IO 失败时，OceanBase 数据库的 IO Manager 会通过探测线程尝试探测数据盘状态，如果超过该配置项设置的时间，探测发现这台机器的数据盘状态仍然未恢复正常，探测线程会将该数据盘状态设置为 `ERROR`，该状态会触发停机等事件以避免向该节点发送的请求失败。

## 配置示例

设置数据盘状态为 `ERROR` 时的容忍时间为 300s。

```shell
obclient> ALTER SYSTEM SET data_storage_error_tolerance_time = 300s;
```
