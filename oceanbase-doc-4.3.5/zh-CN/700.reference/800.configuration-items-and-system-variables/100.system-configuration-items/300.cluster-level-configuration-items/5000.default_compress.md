# default_compress

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V2.1 版本开始引入。</p>
</main>

## 功能描述

`default_compress` 用于设置在 Oracle 模式下指定建表时的默认压缩策略。

<main id="notice">
  <h4>功能适用性</h4>
  <p>该配置项仅适用于 OceanBase 数据库 Oracle 模式。</p>
</main>

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  仅 `sys` 租户可以修改该配置项，用户租户不能修改该配置项。

## 属性说明

|   **属性**    |  **描述**  |
|---------------|------------|
| 参数类型             | String     |
| 默认值               | archive                                                                  |
| 取值范围             | <ul><li>NOCOMPRESS：使用 Flat 格式，不进行压缩。</li><li>BASIC：使用 Flat 格式和 <code>lz4_1.0</code> 压缩方法。</li><li>OLTP：使用 Flat 格式和 <code>zstd_1.3.8</code> 压缩方法。</li><li>QUERY：使用 Encoding 格式和 <code>lz4_1.0</code> 压缩方法。</li> <li>ARCHIVE：使用 Encoding 格式和 <code>zstd_1.3.8</code> 压缩方法。</li></ul>     |
| 是否可修改           | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。  |

## 配置示例

设置在 Oracle 模式下指定建表时的默认压缩策略为 `NOCOMPRESS`。

```shell
obclient> ALTER SYSTEM SET default_compress= 'NOCOMPRESS';
```
