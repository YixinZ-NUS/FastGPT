# syslog_io_bandwidth_limit

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V1.4 版本开始引入。</p>
</main>

## 功能描述

`syslog_io_bandwidth_limit` 用于设置系统日志所能占用的磁盘 IO 带宽上限，超过带宽上限容量的系统日志将被丢弃。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句或视图 `GV$OB_PARAMETERS` 查询该配置项。

* 修改配置项

  仅 `sys` 租户可以修改该配置项，用户租户不能修改该配置项。

## 属性说明

| **属性** | **描述** |
| -------- | -------- |
| 参数类型             | Capacity  |
| 默认值               | 30MB，默认单位为 MB，配置时需说明单位。   |
| 是否可修改           | 是，支持通过 `ALTER SYSTEM SET` 语句修改。|
| 是否重启 OBSever 节点生效 | 否，设置后立即生效。 |

## 配置示例

设置系统日志所能占用的磁盘 IO 带宽为 30MB。

```shell
obclient> ALTER SYSTEM SET syslog_io_bandwidth_limit = '30MB';
```
