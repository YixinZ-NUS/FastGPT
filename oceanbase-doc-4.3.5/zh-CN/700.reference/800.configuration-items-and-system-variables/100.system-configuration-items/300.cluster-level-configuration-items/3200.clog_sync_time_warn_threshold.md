# clog_sync_time_warn_threshold

<main id="notice" type='explain'>
  <h4>说明</h4>
  <p>该配置项从 V1.4 版本开始引入。</p>
</main>

## 功能描述

`clog_sync_time_warn_threshold` 用于设置事务日志同步耗时告警阈值，同步耗时超过该值产生 WARN 日志。

## 权限要求

* 查询配置项

  `sys` 租户和所有用户租户均可以使用 `SHOW PARAMETERS` 语句查询该配置项。

* 修改配置项

  仅 `sys` 租户可以修改该配置项，用户租户不能修改该配置项。

## 属性说明

|      **属性**      |      **描述**      |
|------------------|------------------|
| 参数类型             | Time             |
| 默认值              | 100ms <main id="notice" type='explain'><h4>说明</h4><ul><li>从 V4.0.0 版本开始默认值由 1s 调整为 100ms。</li><li>从 V3.2.3 版本开始默认值由 100ms 调整为 1s。</li></ul></main>           |
| 取值范围             | [1ms, 10000ms] |
| 是否可修改          | 是，支持通过 `ALTER SYSTEM SET` 语句修改。    |
| 是否重启 OBServer 节点生效 | 否，设置后立即生效。  |

## 配置示例

设置事务日志同步耗时告警阈值为 100ms。

```shell
obclient> ALTER SYSTEM SET clog_sync_time_warn_threshold='100ms';
```
