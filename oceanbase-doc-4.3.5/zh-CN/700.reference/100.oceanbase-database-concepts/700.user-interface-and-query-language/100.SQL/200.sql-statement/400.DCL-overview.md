# DCL

数据控制语言（DCL）用于控制数据访问权限。DCL 可以管理指定账号对特定数据库资源的访问权限。

## 默认权限模型

不同数据库系统的默认权限模型不同：

* **Oracle 模式**：在 Oracle 数据库中，当创建一个用户时，系统会自动创建一个与该用户同名的 Schema。例如，用户 `SCOTT` 拥有 Schema `SCOTT`，该用户自动成为这个 Schema 中所有对象的所有者（Owner）。Schema 所有者无需额外授权即可访问自己 Schema 中的对象，而其他用户则需要获得授权。

* **MySQL 模式**：数据没有 Owner 概念。访问数据时通常需要授权。

## 常用 DCL 命令

DCL主要包括授权（GRANT）和回收权限（REVOKE）两种操作。

## 权限授予示例

将 Schema A 下的表 `ta` 的查询权限授予用户 B：

```sql
GRANT SELECT ON a.ta TO b;
```

回收用户 B 在 Schema A 下的表 ta 的查询权限：

```sql
REVOKE SELECT ON a.ta FROM b;
```
