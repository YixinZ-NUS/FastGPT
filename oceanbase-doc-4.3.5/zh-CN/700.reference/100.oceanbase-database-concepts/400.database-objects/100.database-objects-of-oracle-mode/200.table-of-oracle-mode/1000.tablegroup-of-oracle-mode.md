# 表组概述

表组（Table Group）是数据库管理中的一个重要逻辑概念，用于将一组表进行逻辑上的集合，以便于管理和优化数据存储和查询效率。表组在物理存储上控制着这些表之间的邻近关系，为确保数据的高效访问和处理创造了条件。

![tablegroup](https://obbusiness-private.oss-cn-shanghai.aliyuncs.com/doc/img/observer/kv/tablegroup.png)

在 V3.x 版本中，表组的设计注重于分区的概念。所有加入表组的表必须与表组的分区方式保持一致，这种强约束性虽保证了数据一致性，但对表的管理灵活性造成了一定限制。自 V4.2.0 版本起，表组的定义进一步简化。现在，表组只需定义 SHARDING 属性，即可允许不同分区方式的表加入同一表组，从而极大地增强了表组管理的灵活性。

## 表组 SHARDING 属性的分类

根据 SHARDING 属性值的不同，表组主要分为两大类，分别是 SHARDING = NONE 和 SHARDING 不为 NONE。

1. **SHARDING = NONE 的表组**：

   - 这种表组中的所有表的数据均聚集在同一台机器上，且不限制表组内表的分区类型。这种设计适用于对数据存取性能要求高，但表的规模相对较小的场景。

2. **SHARDING 不为 NONE 的表组**：

   - 在这种表组中，每一张表的数据被打散并分布在多台机器上。为了确保数据的均匀分布，要求表组内所有表的分区定义一致，包括分区类型、个数和分区值。系统会调度具有相同分区属性的分区在同一台机器上，从而支持高效的 Partition Wise Join。

### SHARDING 不为 NONE 的细分

在 SHARDING 不为 NONE 的表组中，又可细分为两种类型：

1. **SHARDING = PARTITION 的表组**：

   - 在这种表组内，数据按照一级分区打散，对于二级分区表，其对应一级分区下的所有二级分区会聚集在一起。所有表的一级分区的分区定义必须相同，从而允许一级分区表与二级分区表共存。

   - **分区对齐规则**：相同的一级分区值的分区会被聚集到同一台机器上。这包括一级分区表的一级分区，也包括对应一级分区下的二级分区。

2. **SHARDING = ADAPTIVE 的表组**：

   - 此类表组内的表根据自适应方式进行数据分散。如果表组内包含一级分区表，则按一级分区数据打散；如果包含二级分区表，则按每个一级分区下的二级分区进行打散。

   - **分区要求与对齐规则**：表组中的表需全部为同一分区类型（一级或二级），并要求分区定义一致。一级分区值相同的分区将聚集在一起，而二级表则需满足一级和二级分区值均相同的条件。
