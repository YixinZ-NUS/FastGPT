# 异构数据迁移

异构数据迁移指将数据从外部数据源（如 MySQL、Oracle、PostgreSQL 等关系型数据库，或各类 NoSQL 数据库）迁移至 OceanBase 数据库。这类迁移需要解决数据格式转换、结构适配和跨平台兼容性等问题。

## 迁移场景

异构数据迁移主要适用于以下场景：

+ **数据库升级或替换**：将现有数据库系统迁移到 OceanBase，以获得更好的性能、扩展性或成本效益。
+ **数据整合**：将多个不同数据库系统的数据整合到统一的 OceanBase 平台。
+ **架构升级**：将传统数据库迁移到分布式数据库架构。

在这类数据迁移场景中，需要关注：

+ **Schema 与数据类型映射**：需要精确定义源端和目标端（OceanBase）之间的数据结构和类型映射规则，例如，将 MySQL 的 `DATETIME` 转换为 OceanBase 的 `TIMESTAMP`。
+ **性能优化**：针对海量数据的迁移，采用高效的迁移策略，如并行处理、分片迁移和批量加载，以缩短迁移窗口。
+ **数据一致性校验**：在迁移完成后，需通过哈希校验、抽样对比或全量对比等方法，确保数据的完整性和准确性。
