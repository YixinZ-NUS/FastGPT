# 旁路导入概述

OceanBase 数据库支持 **旁路导入（Bypass Import）** 的方式高效插入数据，其核心是通过绕过 SQL 解析层，直接在底层数据文件中分配空间并写入数据。这一特性充分利用了 OceanBase 的 **分布式架构和 AP 能力**（Analytical Processing，分析型处理），在保证数据一致性和高可用的前提下，显著提升批量数据导入的吞吐量与效率。

## 旁路导入的两种模式

### 全量旁路导入（Full Bypass Import）

- **适用场景**：用于初始化数据库、数据迁移或在需要快速载入大量数据时使用。

- **AP 能力优势**：OceanBase 的 **自动分区机制** 可根据分区键均匀分配数据，避免热点。


### 增量旁路导入（Incremental Bypass Import）

- **适用场景**：用于高吞吐量的数据写入场景，例如大规模的实时数据采集、日志写入等场景。

- **AP 能力优势**：新增节点后，OceanBase 自动 **负载均衡**，动态调整数据分片分布。
